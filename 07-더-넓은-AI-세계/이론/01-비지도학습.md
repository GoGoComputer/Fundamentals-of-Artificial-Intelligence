# 7.1 비지도학습 — 정답 없는 데이터에서 배우기

지금까지 다룬 분류·회귀는 **정답이 있는** 데이터로 학습했습니다. 사진 → "고양이" 같은 라벨이 붙어 있었죠.

그런데 현실에서는 **정답이 없는 데이터** 가 훨씬 많습니다. 고객 1만 명의 구매 기록, 라벨 없는 이미지 100만 장, 로그 데이터 — 이런 데이터에서도 패턴을 찾고 싶을 때가 있죠.

이게 **비지도학습 (Unsupervised Learning)** 입니다.

---

## 비지도학습의 두 가지 큰 줄기

```
비지도학습
 ├─ 군집화 (Clustering)        — 비슷한 것끼리 묶기
 └─ 차원 축소 (Dim. Reduction) — 변수 수 줄이기
```

(이 외에 이상치 탐지, 생성 모델 등도 있지만 두 갈래가 가장 큼)

---

## 1. 군집화 (Clustering)

> "이 1만 명의 고객을 비슷한 사람끼리 묶어 줘."

**미리 정해진 라벨 없이** 데이터를 그룹으로 나눕니다. 군집의 개수와 모양은 알고리즘이 정해요.

### 1-1. K-Means — 가장 유명한 군집화

**아이디어:** 데이터를 K개의 그룹으로 나누되, 각 그룹의 **중심점에서 점들까지의 거리 합** 을 가장 작게.

#### 동작 (직관)

```
Step 1: 무작위로 K개 중심점 선택
Step 2: 각 데이터를 가장 가까운 중심점에 배정
Step 3: 각 그룹의 평균을 새 중심점으로
Step 4: 중심점이 거의 안 움직일 때까지 2~3 반복
```

#### 코드

```python
from sklearn.cluster import KMeans

kmeans = KMeans(n_clusters=3, random_state=42, n_init=10)
kmeans.fit(X)

labels = kmeans.labels_           # 각 점의 그룹 (0, 1, 2)
centers = kmeans.cluster_centers_  # 3개 중심점
```

#### 장단점

- ✅ 빠르고 단순, 큰 데이터에도 OK
- ❌ K를 미리 정해야 함
- ❌ 원형(球) 모양 군집만 잘 잡음 — 길쭉하거나 비뚤어진 모양엔 약함
- ❌ 초기 중심점에 영향 — `n_init` 으로 여러 번 시도

#### K를 어떻게 정해요? — 엘보 방법

K를 1, 2, 3, ... 으로 늘려가며 손실(inertia)을 봅니다. 어느 시점에 손실이 급격히 안 줄어들면, 거기가 적절한 K.

```python
import matplotlib.pyplot as plt

inertias = []
for k in range(1, 11):
    kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
    kmeans.fit(X)
    inertias.append(kmeans.inertia_)

plt.plot(range(1, 11), inertias, marker='o')
plt.xlabel('K')
plt.ylabel('Inertia (손실)')
plt.title('Elbow Method')
plt.show()
# 곡선이 꺾이는 지점이 적정 K
```

---

### 1-2. DBSCAN — 모양 자유로운 군집화

K-Means가 못 잡는 모양 (예: 초승달 모양, 도넛 모양) 을 잘 잡는 알고리즘.

**아이디어:** **밀도 기반.** 점이 빽빽한 곳을 한 군집으로, 빈 공간을 경계로.

```python
from sklearn.cluster import DBSCAN

dbscan = DBSCAN(eps=0.5, min_samples=5)
labels = dbscan.fit_predict(X)
# -1 은 어느 군집에도 안 속한 노이즈
```

#### 매개변수

- `eps` — "이 거리 안에 다른 점이 있으면 이웃" 의 거리 기준
- `min_samples` — 한 영역이 군집이 되려면 최소 몇 개의 이웃이 있어야 하는지

#### 장단점

- ✅ K 미리 정할 필요 없음 — 알아서 군집 개수 결정
- ✅ 모양 자유로움
- ✅ 노이즈 자동으로 분리
- ❌ 밀도가 다른 군집들엔 약함
- ❌ 매개변수 튜닝 까다로움

---

### 1-3. 계층적 군집화 (Hierarchical)

군집을 **트리 구조** 로 만듭니다. 큰 군집 안에 작은 군집이 있는 구조.

```python
from sklearn.cluster import AgglomerativeClustering

hac = AgglomerativeClustering(n_clusters=3)
labels = hac.fit_predict(X)
```

작은 데이터 (~수천 개) 에서 군집 구조를 시각화하고 싶을 때 유용합니다. 큰 데이터에는 느려요.

---

### 1-4. GMM (Gaussian Mixture Model)

각 군집이 **정규분포 모양** 이라고 가정. K-Means가 "어느 군집에 속함" 이라고 단정한다면, GMM은 "이 군집에 속할 확률" 을 줍니다 (소프트 군집화).

```python
from sklearn.mixture import GaussianMixture

gmm = GaussianMixture(n_components=3, random_state=42)
gmm.fit(X)
probs = gmm.predict_proba(X)    # 각 군집에 속할 확률
```

이상치 탐지, 음성 분리 등에 자주 씁니다.

---

### 군집화 — 어디에 써요?

- **고객 세분화** — 비슷한 구매 패턴의 고객끼리 묶어 마케팅
- **이상 탐지** — 어떤 군집에도 안 속한 점은 이상치 의심
- **이미지 분할 (segmentation)** — 픽셀을 색·위치 기준으로 묶기
- **문서 분류 (라벨 없을 때)** — 비슷한 주제의 문서 묶기
- **유전자 발현 분석**

## 2. 차원 축소 (Dimensionality Reduction)

> "변수가 1000개인 데이터를 2개로 줄여 보고 싶어요."

이미지 한 장이 28×28 = 784차원이고, 텍스트는 단어 수만큼 차원이 커집니다. 너무 많은 차원은:
- 시각화 불가능 (3차원 넘어가면 그릴 수 없음)
- 계산 비용 큼
- "차원의 저주" — 데이터가 희박해져서 모델이 못 배움

**핵심 정보를 유지하면서 차원만 줄이는 게 차원 축소.**

---

### 2-1. PCA (주성분 분석) — 가장 표준

수학적 기초는 [부록 4a 선형대수 심화](../../부록/수학/04a-선형대수-심화.md#4a6-pca-principal-component-analysis--주성분-분석) 에 자세히 있어요. 여기는 실용 위주.

**핵심:** 데이터의 **분산이 가장 큰 방향들** 을 찾아 그 방향으로 사영.

```python
from sklearn.decomposition import PCA

pca = PCA(n_components=2)
X_2d = pca.fit_transform(X_high_dim)

# 시각화
import matplotlib.pyplot as plt
plt.scatter(X_2d[:, 0], X_2d[:, 1])
plt.show()
```

#### 얼마나 많은 정보를 보존했는지 확인

```python
print(pca.explained_variance_ratio_)
# [0.45, 0.30] → PC1이 45%, PC2가 30% 정보 보존, 합쳐서 75%
```

#### 적정 차원 자동 선택

```python
# 분산의 95%를 보존하는 최소 차원
pca = PCA(n_components=0.95)
X_reduced = pca.fit_transform(X)
print(f"줄어든 차원: {X_reduced.shape[1]}")
```

#### 언제 PCA가 좋아요?

- 선형 관계가 충분한 데이터
- 노이즈 제거가 목적
- 시각화는 보너스 (하지만 비선형 구조엔 약함)

---

### 2-2. t-SNE — 시각화 전용

**비선형 차원 축소.** 시각화에 정말 강력하지만 느립니다.

```python
from sklearn.manifold import TSNE

tsne = TSNE(n_components=2, random_state=42, perplexity=30)
X_2d = tsne.fit_transform(X)
```

#### 주의사항

- ✅ 군집을 시각적으로 잘 분리해 보여줌
- ❌ 매번 결과가 다름 (random_state 고정 필수)
- ❌ 거리·크기에 의미 없음 — "두 군집이 멀어 보이는" 게 진짜 멀다는 의미가 아님
- ❌ 매우 느림 — 데이터 1만 개 이상이면 답답
- ❌ 새 데이터엔 못 씀 (재학습 필요)

**오직 시각화용으로만 쓰세요.** 모델 입력으로 쓰면 안 돼요.

---

### 2-3. UMAP — t-SNE의 빠른 형제

t-SNE 비슷한데 더 빠르고, 새 데이터에도 적용 가능합니다.

```python
# pip install umap-learn
import umap

reducer = umap.UMAP(n_components=2, random_state=42)
X_2d = reducer.fit_transform(X)
```

요즘은 t-SNE 대신 UMAP을 쓰는 경우가 많아요.

---

### 차원 축소 비교표

| | PCA | t-SNE | UMAP |
|---|---|---|---|
| 선형/비선형 | 선형 | 비선형 | 비선형 |
| 속도 | 빠름 | 매우 느림 | 보통 |
| 새 데이터 적용 | ✅ | ❌ | ✅ |
| 시각화 품질 | 보통 | 매우 좋음 | 매우 좋음 |
| 모델 입력으로 사용 | ✅ | ❌ | ✅ |
| 첫 시도 추천 | ✅ | (시각화만) | ✅ (UMAP) |

## 3. 이상치 탐지 (Anomaly Detection) — 짧게

비지도학습의 또 다른 용도. **"평소와 다른 것" 을 찾아내기.**

### Isolation Forest

```python
from sklearn.ensemble import IsolationForest

iso = IsolationForest(contamination=0.05, random_state=42)
labels = iso.fit_predict(X)
# 1 = 정상, -1 = 이상치
```

`contamination=0.05` 는 "데이터의 5% 정도가 이상치" 라는 가정.

### LOF (Local Outlier Factor)

밀도 기반. 이웃과 비교해 외로운 점을 찾음.

```python
from sklearn.neighbors import LocalOutlierFactor

lof = LocalOutlierFactor(n_neighbors=20)
labels = lof.fit_predict(X)
```

### 어디에 써요?

- 신용카드 사기 탐지
- 시스템 로그 이상 (장애 전조)
- 제조 공정 불량
- 보안 (침입 탐지)

## 4. 실전 예: 고객 세분화

```python
import pandas as pd
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

# 가상 고객 데이터 (구매 횟수, 평균 결제액, 최근 방문일)
df = pd.DataFrame({
    'purchases':  [...],
    'avg_amount': [...],
    'days_since': [...],
})

# 1. 정규화 (군집화에서 항상 필수)
scaler = StandardScaler()
X = scaler.fit_transform(df)

# 2. 적정 K 찾기 (엘보)
inertias = [KMeans(k, random_state=42, n_init=10).fit(X).inertia_ 
            for k in range(1, 8)]
plt.plot(range(1, 8), inertias, 'o-')
plt.show()

# 3. K=4로 군집화
km = KMeans(n_clusters=4, random_state=42, n_init=10)
df['cluster'] = km.fit_predict(X)

# 4. 각 군집의 특성 확인
print(df.groupby('cluster').mean())
# 0번 군집: 자주 구매, 큰 금액 → VIP
# 1번 군집: 가끔 구매, 작은 금액 → 일반
# 2번 군집: 옛날에 한 번, 큰 금액 → 휴면
# 3번 군집: 자주 구매, 작은 금액 → 단골

# 5. 시각화 (PCA로 2차원으로)
pca = PCA(n_components=2)
X_2d = pca.fit_transform(X)
plt.scatter(X_2d[:, 0], X_2d[:, 1], c=df['cluster'])
plt.show()
```

이게 **실무 고객 세분화의 표준 레시피** 입니다.

## 5. 정리

```python
# 군집화
from sklearn.cluster import KMeans, DBSCAN, AgglomerativeClustering
from sklearn.mixture import GaussianMixture

# 차원 축소
from sklearn.decomposition import PCA
from sklearn.manifold import TSNE
import umap

# 이상치 탐지
from sklearn.ensemble import IsolationForest
from sklearn.neighbors import LocalOutlierFactor
```

**핵심 가이드:**

| 상황 | 추천 |
|------|------|
| 일반 군집화 첫 시도 | **K-Means** |
| 모양이 자유로운 군집 | DBSCAN |
| 확률 필요 | GMM |
| 데이터 시각화 (1만 개 이하) | UMAP > t-SNE |
| 차원 축소 + 모델 입력 | PCA |
| 이상치 탐지 | Isolation Forest |
