# AI, 머신러닝, 딥러닝 — 무엇이 다른가요?

## 결론부터

세 개념의 관계를 한 그림으로 그리면 이렇습니다.

```
┌──────────────────────────────────────────────┐
│            인공지능 (AI)                       │
│   "사람처럼 행동하는 기계를 만들자"              │
│                                                │
│   ┌────────────────────────────────────────┐  │
│   │       머신러닝 (Machine Learning)         │  │
│   │  "기계에게 데이터를 주고 스스로 배우게 하자"  │  │
│   │                                          │  │
│   │   ┌──────────────────────────────────┐  │  │
│   │   │    딥러닝 (Deep Learning)         │  │  │
│   │   │  "신경망을 깊게 쌓아서 배우게 하자"   │  │  │
│   │   └──────────────────────────────────┘  │  │
│   └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

큰 우산 안에 작은 우산, 그 안에 또 작은 우산이 있는 모양이에요.

---

## AI: 가장 큰 우산

**AI(Artificial Intelligence, 인공지능)** 는 1950년대에 등장한 단어입니다. 그 시절의 정의는 이거예요.

> "사람의 지능을 흉내 내는 기계."

너무 넓죠? 그래서 AI는 한 가지 기술이 아니라 **방향을 가리키는 단어**라고 보는 게 맞아요. 누가 "AI를 만들었다"고 하면 "어떤 방식으로 만드셨어요?"가 다음 질문이 되어야 합니다.

AI를 만드는 방법은 크게 두 가지입니다.

### 방법 1. 규칙 기반 AI (옛날 방식)

사람이 모든 규칙을 일일이 적어 줍니다.

```
IF (눈이 두 개 있고)
   AND (코가 가운데 있고)
   AND (입이 아래에 있다면)
THEN "사람 얼굴"
```

이 방식의 한계는 명확해요. **세상의 모든 규칙을 적을 수가 없습니다.** 사람의 얼굴조차도 이렇게 단순한 규칙으로는 설명이 안 되는 거예요.

### 방법 2. 데이터로 배우는 AI (요즘 방식)

규칙을 적어 주는 게 아니라, **데이터를 잔뜩 보여주고 기계가 스스로 규칙을 찾게** 합니다.

```
사람 얼굴 사진 10만 장 + 사람 얼굴 아닌 사진 10만 장
                ↓
        기계에게 보여주기
                ↓
        "사람 얼굴"이 무엇인지
        기계가 스스로 학습
```

이 두 번째 방식이 바로 **머신러닝**입니다.

## 머신러닝: AI의 가장 인기 있는 갈래

**머신러닝(Machine Learning, ML)** 은 90년대에 본격적으로 떠오른 분야예요. 핵심 발상은 이겁니다.

> "인간이 규칙을 가르치는 게 어렵다면, 기계가 데이터에서 규칙을 찾게 하자."

머신러닝이라는 단어를 들으셨을 때 이 한 줄을 떠올리시면 됩니다. **"데이터로 패턴을 찾는 일."**

머신러닝 안에는 또 여러 갈래가 있어요. 가장 큰 세 갈래는 이렇습니다.

| 종류 | 무엇? | 예시 |
|------|------|------|
| **지도학습 (Supervised)** | 정답이 있는 데이터로 배우기 | 사진 → "고양이" 정답 / 집 정보 → "5억원" 정답 |
| **비지도학습 (Unsupervised)** | 정답 없이 패턴만 찾기 | 고객 1만 명을 비슷한 그룹으로 묶기 |
| **강화학습 (Reinforcement)** | 시행착오로 보상을 얻으며 배우기 | 알파고, 자율주행 |

**우리가 이 자료에서 다루는 건 지도학습입니다.** 가장 많이 쓰이고, 가장 결과가 좋고, 가장 배우기 좋거든요. 비지도학습과 강화학습은 일단 미뤄 두셔도 좋아요. 지도학습만 잘해도 회사에서 90%의 일을 할 수 있습니다.

지도학습은 다시 두 갈래로 나뉩니다.

- **분류(Classification)**: 정답이 "범주"인 경우. 이 사진은 강아지냐 고양이냐, 이 메일은 스팸이냐 아니냐.
- **회귀(Regression)**: 정답이 "숫자"인 경우. 이 집은 얼마짜리냐, 내일 기온은 몇 도냐.

이 두 가지가 머신러닝의 양 날개예요. **그래서 우리 자료의 2장이 분류, 3장이 회귀입니다.**

---

## 딥러닝: 머신러닝의 떠오르는 별

**딥러닝(Deep Learning, DL)** 은 2010년쯤부터 폭발적으로 떠올랐어요. 머신러닝의 한 갈래인데, **신경망(Neural Network)을 깊게(deep) 쌓아서** 학습하는 방식이에요.

신경망이라는 단어가 어렵게 들리지만, 사실 별 거 아닙니다. 그냥 **"입력을 받아서 → 곱하고 더한 다음 → 다음 층으로 넘기는"** 작업을 여러 층(layer) 쌓은 거예요.

```
입력 → [곱하기 더하기] → [곱하기 더하기] → [곱하기 더하기] → 출력
        layer 1            layer 2            layer 3
```

층이 얕으면 그냥 머신러닝이고, 층을 깊게 쌓으면 딥러닝입니다.

### "그럼 무조건 딥러닝이 더 좋은 거 아닌가요?"

이게 또 그렇지가 않아요. 딥러닝은 강력한 만큼 단점도 있습니다.

| | 머신러닝 (전통적) | 딥러닝 |
|---|---|---|
| **필요한 데이터** | 1천~1만 개로도 OK | 보통 10만 개 이상 |
| **계산 자원** | 노트북으로 충분 | GPU 필수 |
| **학습 시간** | 몇 분 | 몇 시간~며칠 |
| **결과 해석** | 어느 정도 가능 | 거의 불가능 (블랙박스) |
| **잘하는 분야** | 정형 데이터 (엑셀같은 표) | 비정형 데이터 (이미지, 음성, 텍스트) |

그래서 회사에서 실제로 일을 하실 땐 이렇게 됩니다.

> "데이터가 표 형태(엑셀)고 1만~10만 개면 → 머신러닝부터 시도. 잘 안 되면 그때 딥러닝."
>
> "데이터가 이미지/음성/텍스트라면 → 처음부터 딥러닝 (또는 사전학습된 딥러닝 모델 가져다 쓰기)."

이 자료가 머신러닝(2, 3장)을 먼저 가르치고 딥러닝(4, 5장)을 나중에 가르치는 이유가 그거예요. **현실에서 먼저 시도하는 순서**가 그렇습니다.

---

## "그럼 ChatGPT는 뭐예요?"

좋은 질문입니다. ChatGPT는 **딥러닝의 가장 화려한 응용 사례**예요. 정확히는 LLM(Large Language Model, 거대 언어 모델)이라고 부릅니다.

ChatGPT를 다시 우리 우산 그림에 넣어 보면 이렇게 돼요.

```
인공지능 (AI)
 └─ 머신러닝 (ML)
     └─ 딥러닝 (DL)
         └─ 신경망 중에서도 "Transformer"라는 구조
             └─ 그 중에서도 텍스트로만 학습한 모델
                 └─ 그것이 ChatGPT (GPT-4 같은 LLM들)
```

ChatGPT는 결국 **"다음에 올 단어를 분류하는 모델"** 입니다. 분류 모델이에요. 우리 자료의 2장에서 배우는 분류와 본질적으로 같습니다. 다만 분류해야 할 종류(클래스)가 5만 개쯤 되고(어휘 수만큼), 데이터가 인터넷 전체쯤 되고, 모델이 어마어마하게 클 뿐이에요.

그래서 이 자료를 다 보고 나시면 ChatGPT가 어떻게 돌아가는지에 대한 직관도 생기실 겁니다. 거짓말 아니에요.

## 요약: 한 문장씩 외우기

- **AI**: 사람처럼 행동하는 기계를 만들자는 큰 꿈.
- **머신러닝**: 데이터를 주고 기계가 스스로 패턴을 찾게 하자.
- **딥러닝**: 머신러닝 중에서 신경망을 깊게 쌓는 방식.
- **분류 vs 회귀**: 정답이 범주냐(분류), 숫자냐(회귀).
- **ML vs DL**: 데이터가 적고 표 형태면 ML, 많고 비정형이면 DL.

## 헷갈리기 쉬운 포인트

AI, 머신러닝, 딥러닝은 자주 같은 뜻처럼 쓰이지만, 학습할 때는 반드시 층위를 분리해야 합니다. 이 구분이 흐려지면 도구 선택과 기대치가 동시에 어긋납니다.

- AI는 목표(지능적 행동), ML은 방법(데이터 학습), DL은 ML의 특정 계열(신경망)입니다.
- "딥러닝이 항상 더 좋다"가 아니라 데이터량/문제형태/자원/해석가능성에 따라 선택이 달라집니다.
- 분류와 회귀는 모델 이름보다 먼저 결정해야 하는 문제 정의입니다.
- LLM도 본질적으로는 확률적 분류 문제의 대규모 버전이라는 점을 기억하면 이후 학습이 연결됩니다.

## 실전에서 보는 패턴

실무 팀은 보통 다음 순서로 판단합니다.

1. 문제의 출력 형태를 먼저 확정합니다. 범주면 분류, 연속값이면 회귀로 시작합니다.
2. 데이터 형태를 확인합니다. 표 데이터는 전통 ML, 이미지/텍스트/음성은 DL 우선 전략이 일반적입니다.
3. 베이스라인 모델을 빠르게 만듭니다. 단순 모델로 기준점을 잡아야 복잡 모델의 개선 폭이 의미를 가집니다.
4. 운영 제약을 반영합니다. 추론 시간, 비용, 설명 가능성 요구가 모델 선택을 바꿉니다.

## 자주 하는 실수와 해결

- 실수: 모델 종류부터 고른다.
        해결: 문제 정의 -> 데이터 점검 -> 평가 기준 설정 후 모델을 선택합니다.
- 실수: 용어를 외우지만 관계를 이해하지 못한다.
        해결: AI>ML>DL 포함 관계를 그림으로 직접 그리고, 예시를 각 층위에 배치해 봅니다.
- 실수: 한 번의 성능 수치로 결론을 낸다.
        해결: 분할/시드/지표를 고정해 재실험하고 평균 성능을 비교합니다.
- 실수: ChatGPT 같은 서비스와 모델 원리를 완전히 분리해서 본다.
        해결: 입력-출력-학습 목표 관점으로 분해해 기존 분류 학습과 연결합니다.

## 이해를 더 단단하게 만드는 보강 설명

이 단원은 처음 읽을 때는 "아, 이런 개념이 있구나" 정도로 넘어가기 쉽지만, 실제로는 작은 차이가 나중에 큰 성능 차이로 이어집니다. 특히 **AI, 머신러닝, 딥러닝 — 무엇이 다른가요?** 같은 주제는 한 번에 완벽히 이해하려고 하기보다, 다음 세 가지를 반복하면서 이해를 깊게 만드는 편이 훨씬 효과적입니다.

첫째, 용어를 정의로 외우지 말고 상황으로 연결해 보세요. 예를 들어 "왜 이 개념이 필요한가"를 스스로 설명할 수 있어야 다음 단원에서 응용이 됩니다. 둘째, 코드가 나오면 결과를 먼저 예측한 뒤 실행해 보세요. 예측과 결과가 다를 때가 진짜 학습이 일어나는 순간입니다. 셋째, 같은 코드를 숫자나 파라미터만 바꿔 여러 번 돌려 보세요. 머신러닝과 딥러닝은 정답을 암기하는 과목이 아니라 **변화를 읽는 과목**이기 때문입니다.

또 하나 중요한 점은, 이 단원의 내용이 단독으로 존재하지 않는다는 사실입니다. 실제 프로젝트에서는 데이터 준비, 모델링, 평가, 배포가 끊기지 않고 이어집니다. 그래서 지금 배우는 **AI, 머신러닝, 딥러닝 — 무엇이 다른가요?**도 나중에 다른 단계와 반드시 연결됩니다. 이 연결을 의식하면서 읽으면, 단편 지식이 아니라 시스템 관점으로 이해하게 됩니다.

## 자주 하는 오해와 바로잡기

### 1) "이론만 이해하면 실습은 저절로 된다"
아닙니다. 이론 이해와 코드 실행 능력은 별개의 근육입니다. 둘을 같이 써야 합니다.

### 2) "예제 코드가 돌아갔으니 완전히 이해했다"
동작 확인은 시작일 뿐입니다. 파라미터를 바꿨을 때 결과가 어떻게 달라지는지까지 확인해야 이해가 완성됩니다.

### 3) "좋은 모델은 항상 하나다"
데이터 특성과 제약(시간, 메모리, 응답 속도)에 따라 최선의 선택은 달라집니다.

### 4) "지표 숫자만 높으면 끝이다"
지표는 해석이 필요합니다. 어떤 데이터에서, 어떤 분포에서, 어떤 기준으로 얻은 점수인지 함께 보아야 합니다.

### 5) "지금은 초반이니까 배포는 나중에 생각해도 된다"
초반부터 재현성과 저장 구조를 습관화해야 나중에 배포 단계에서 무너지지 않습니다.

## 실전 연결 시나리오

아래는 이 단원 내용을 실제 업무로 연결할 때 자주 쓰는 흐름입니다.

1. 문제를 한 문장으로 정의한다.  
예: "AI, 머신러닝, 딥러닝 — 무엇이 다른가요? 관점에서 입력으로부터 어떤 출력을 만들 것인가?"

2. 성공 기준을 지표로 정한다.  
예: 정확도/RMSE/F1/지연시간 같은 운영 가능한 숫자로 정의.

3. 가장 단순한 베이스라인을 만든다.  
복잡한 모델보다 먼저, 빠르게 재현 가능한 기본 모델을 세웁니다.

4. 한 번에 하나씩 개선한다.  
전처리, 모델, 튜닝, 임계값, 배치 크기 등은 동시에 바꾸지 않고 순차적으로 바꿉니다.

5. 결과를 로그와 함께 저장한다.  
실험 이름, 파라미터, 점수, 데이터 버전을 함께 남겨야 재현이 됩니다.

6. 실패 케이스를 분석한다.  
점수 평균보다 오분류/고오차 샘플을 분석할 때 개선 아이디어가 나옵니다.

7. 배포 가능 형태로 정리한다.  
모델 파일, 메타데이터, 입력 스키마, 테스트 스크립트까지 묶습니다.

## 복습 체크리스트

다음 질문에 답할 수 있으면 이 단원을 실전 수준으로 이해한 것입니다.

- 이 단원의 핵심 개념을 중학생에게 설명하듯 말할 수 있는가?
- 코드의 각 블록이 왜 필요한지 한 줄씩 설명할 수 있는가?
- 이 단원에서 가장 자주 틀리는 지점을 알고 있는가?
- 지표를 볼 때 어떤 함정이 있는지 알고 있는가?
- 베이스라인과 개선 모델의 차이를 표로 정리할 수 있는가?
- 재현 가능하게 실험을 저장하는 습관이 있는가?
- 실패 사례를 보고 다음 실험 가설을 만들 수 있는가?
- 이 단원 내용을 다음 장과 어떻게 연결할지 설명할 수 있는가?

## 확장 연습 과제

### 과제 A: 파라미터 감각 만들기
현재 예제에서 핵심 파라미터 1~2개만 바꿔 5회 실험하고, 결과를 표로 정리해 보세요.

### 과제 B: 실패 사례 분석
오분류 또는 오차가 큰 샘플 20개를 모아 공통 패턴을 찾아 보세요.

### 과제 C: 설명 가능 리포트 작성
"무엇을 했고, 왜 했고, 결과가 어땠고, 다음엔 무엇을 할지"를 1페이지로 정리해 보세요.

이 3개를 직접 해 보면 **AI, 머신러닝, 딥러닝 — 무엇이 다른가요?**의 이해도는 읽기만 했을 때보다 훨씬 빠르게 올라갑니다.

## 한 번 더 정리

AI, 머신러닝, 딥러닝 — 무엇이 다른가요?는 단순히 시험용 개념이 아니라, 실제 문제를 풀 때 반복해서 쓰게 되는 실전 도구입니다. 처음에는 낯설 수 있지만, 작은 실험을 반복하면 감각이 생기고, 감각이 생기면 코드와 지표를 읽는 속도가 급격히 빨라집니다. 지금 단계에서 중요한 것은 완벽한 암기가 아니라, **작게 실행하고, 결과를 읽고, 다시 수정하는 루프**를 몸에 익히는 것입니다. 이 루프를 익히면 다음 장으로 넘어갈수록 학습 속도가 오히려 빨라집니다.
