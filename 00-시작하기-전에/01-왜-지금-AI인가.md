# 왜 지금 AI인가요?

## 한 장면으로 시작해 볼게요

90년대 어느 골목에는 **컴퓨터 학원**이 김밥천국만큼 많았습니다. 부모님은 자식이 컴퓨터를 잘 다루는 어른이 되길 바랐고, 자식들은 그저 게임을 만들고 싶었어요. 두 욕망이 절묘하게 만나는 지점에 컴퓨터 학원이 있었던 거죠.

그리고 30년 가까이 흐른 지금, 그 자리를 차지한 게 무엇일까요?

**AI 학원입니다.** 길에 나가 보면 "초등학생 AI 코딩", "AI 자격증반", "AI 활용 1일 특강" 간판이 줄지어 있어요. 다시 한 번, 부모님과 자식의 욕망이 절묘하게 만나고 있는 겁니다.

이 풍경을 보면서 한 번쯤 이런 의심이 들지 않으세요?

> "이거 그냥 유행 아닐까? 몇 년 지나면 다 사그라드는 거 아닐까?"

저도 처음엔 그랬어요. 그래서 한참을 들여다봤습니다. 그리고 지금은 이렇게 답합니다. **"이번엔 진짜입니다."**

왜 그렇게 생각하는지, 잠깐만 같이 따라와 주세요.

## 한 사람이 만들 수 있는 것이 너무 커졌습니다

이 이야기부터 시작해 볼게요.

**2012년 4월 9일.** 페이스북이 인스타그램이라는 회사를 인수합니다. 인수 금액은 **약 10억 달러** (당시 환율 기준 약 1조 1천억 원). 현금 3억 달러와 페이스북 주식의 결합이었어요.

당시 인스타그램은:
- 창립한 지 약 **18개월** 된 회사 (2010년 10월 창립)
- 월간 활성 사용자 **약 3천만 명**
- 매일 수백만 장의 사진이 업로드되는 서비스
- 직원 수: **13명**

**13명입니다.** 디자이너 1~2명을 빼고, 경영진을 빼고, 코딩 안 하는 분들을 빼면 — 실제로 그 인프라를 굴리던 엔지니어는 **5명 안팎**으로 추정됩니다.

같은 시점(2012년)의 다른 회사들과 한 번 비교해 보세요.

```
회사         직원 수      창립    한 줄 평
─────────────────────────────────────────────────────
IBM          ~43만 명     1911   컴퓨터의 시조, 100년 기업
Microsoft    ~9만 4천 명  1975   Windows의 회사
Google       ~5만 4천 명  1998   검색의 황제
Facebook     ~3천 5백 명  2004   IPO 직전, 폭풍 성장 중
Instagram    13명          2010   인수 직전
```

(가로 막대로 그리면 인스타그램의 13은 점 하나 찍기도 어려워요.)

```
IBM        ████████████████████████████████  43만
MS         ███████                            9만 4천
Google     ████                               5만 4천
Facebook   ▏                                  3천 5백
Instagram  ·                                  13
```

처음 이 비교를 봤을 때 저는 좀 무서웠어요. **수천만 명이 쓰는 서비스를 13명이 만들 수 있다고?**

이 사람들이 천재라서? 그렇지 않아요. 인스타그램 창립자 케빈 시스트롬도 "정상적인 엔지니어"였어요. **이 사람들이 천재라기보다는, 이 사람들이 일할 수 있는 도구가 천재가 된 거예요.**

지난 20년 동안 소프트웨어를 만드는 일에서 일어난 가장 큰 변화 4가지를 같이 봅시다.

## 도구 1: API — 남이 만든 걸 빌려 쓰기

**API**는 **A**pplication **P**rogramming **I**nterface의 약자예요. 말은 어려운데, 직관은 단순해요. **"다른 회사가 만들어 둔 기능을 내 코드에서 호출해서 쓰는 창구."**

예시 하나만. 여러분이 "근처 음식점 찾기 앱"을 만들고 싶다고 해 봐요. 90년대였다면?

- 위성 사진 사기 (수억 원)
- 도로망 데이터 사기 (수십억 원)
- 지리 정보 가공 (인력 수십 명, 수년)
- 그 다음에 비로소 앱 개발 시작

지도는 사실 **군사 정보**였어요. 일반인이 접근 자체를 못 했어요.

오늘날에는?

```python
# 5분이면 끝
import googlemaps
gmaps = googlemaps.Client(key='YOUR_KEY')
results = gmaps.places_nearby(location=(37.5, 127.0), radius=500)
```

구글 맵스, 카카오맵, 네이버 지도가 만들어 둔 API를 호출하면 끝이에요. **그들이 수십억을 들여 만든 지도를 우리는 한 줄의 코드로 빌려 쓸 수 있어요.**

지도뿐만이 아닙니다.

| 분야 | 대표 API |
|------|---------|
| 결제 | 토스페이먼츠, 카카오페이, Stripe |
| 인증 | 구글/네이버/카카오 로그인 |
| 메일 발송 | SendGrid, AWS SES |
| 문자 발송 | Twilio, NHN Cloud |
| 음성 인식 | OpenAI Whisper, 구글 Speech |
| 번역 | DeepL, 파파고 |
| 결제 정산 | 토스 |

**남이 어렵게 만들어 둔 걸 우리는 빌려 쓰면 그만**입니다. 인스타그램도 푸시 알림은 Apple/Google에, 위치 정보는 Apple/Google 지도 API에 의존했어요.

## 도구 2: 클라우드 컴퓨팅 — 컴퓨터를 빌려 쓰기

여러분이 만든 웹서비스를 오픈했어요. 일주일 만에 100만 명이 들어왔어요. 대박이죠?

전통적인 관점에서는 **쪽박**입니다. 슬픈 쪽박이에요.

왜냐하면 100만 명을 감당할 서버를 며칠 만에 사고, 데이터센터에 갖다 놓고, 네트워크 깔고, OS 깔고, 세팅하는 게 **물리적으로 불가능**하거든요. 사용자가 와도 다 튕겨 나가요. 입소문은 "에이, 그 서비스 안 들어가지더라"가 됩니다.

오늘날에는?

```bash
# AWS 콘솔 또는 CLI로
aws ec2 run-instances --instance-type m5.large --count 1000
# → 1분 안에 1000대의 컴퓨터가 만들어짐
```

**AWS** (Amazon Web Services, 2006년 출시), **Google Cloud** (2008년), **Azure** (2010년) — 이런 클라우드 서비스가 있어서, 마우스 클릭 또는 명령어 하나로 **1분 만에 1000대의 컴퓨터를 만들 수 있어요.**

이용한 만큼만 시간 단위로 돈을 내고, 사용자가 줄면 다시 클릭으로 없애요. **사라진 컴퓨터를 어떻게 처분할지 걱정이 없어요.** 과거에는 한 번 컴퓨터를 사면 5년은 써야 했죠. 지금은 시간 단위로 빌리고 끝낼 수 있어요.

이게 가능해진 게 **2006년 AWS S3와 EC2** 출시 이후예요. 인스타그램은 2010년 처음부터 AWS 위에 만들어졌고, 사용자가 폭증하던 2011~2012년에도 **물리 서버를 한 대도 사지 않고** 사용자가 늘어난 만큼 클라우드 서버만 더 빌렸어요. 그래서 13명이 수천만 사용자를 감당할 수 있었던 거예요.

## 도구 3: 오픈소스 — 설계도를 모두에게 공개하기

소프트웨어의 "**소스 코드**"는 그 소프트웨어의 **설계도**예요. 소스가 있으면 같은 소프트웨어를 다시 만들 수 있어요.

여기서 건축의 설계도와 다른 결정적인 점이 있어요. **건축은 설계도가 있어도 짓는 데 수년이 걸리지만, 소프트웨어는 설계도(소스)만 있으면 그게 곧 제품**이에요. 컴파일하면 바로 동작하니까요.

**오픈소스(Open Source)** 는 이 설계도를 **무료로, 누구나 보고, 고치고, 가져다 쓸 수 있게 공개**하자는 운동이에요. 1980~90년대에는 일부 선각자들의 문화 운동에 가까웠어요. 지금은요?

오늘날 거의 모든 소프트웨어가 오픈소스 위에 만들어집니다.

| 분야 | 오픈소스 | 점유율/위상 |
|------|---------|-------------|
| 운영체제 | **Linux** (1991년) | 서버의 약 95% |
| 모바일 OS | **Android** (Linux 기반) | 전세계 모바일의 ~70% |
| 프로그래밍 언어 | Python, JavaScript, Go, Rust | 거의 다 |
| 웹 프레임워크 | React, Vue, Django, Spring | 거의 다 |
| 데이터베이스 | PostgreSQL, MySQL, Redis | 거의 다 |
| 머신러닝 | **PyTorch, TensorFlow, scikit-learn** | 거의 다 (이 자료에서 쓸 도구!) |
| 버전 관리 | **Git** (2005년) | 사실상 표준 |
| 컨테이너 | Docker, Kubernetes | 사실상 표준 |

세계 최대 IT 회사들이 오히려 오픈소스에 가장 많이 기여합니다.
- **메타**(페이스북) → React, PyTorch
- **구글** → TensorFlow, Kubernetes, Go, Android
- **마이크로소프트** → VS Code, TypeScript, .NET
- **OpenAI** → Whisper, 일부 모델

수억을 들여 만든 걸 그냥 줘요. 왜? **공개하면 더 많은 개발자가 쓰고, 더 많은 버그가 잡히고, 더 빨리 좋아져요.** 결과적으로 회사에도 이득이거든요. 이게 21세기 소프트웨어 비즈니스의 근본 논리예요.

지나가는 개발자 아무나 붙잡고 "당신의 생계에 특허와 오픈소스 중 무엇이 더 큰 공헌을 했나요?" 라고 물어보세요. **거의 모두 오픈소스라고 답할 거예요.** 이 자료에서 우리가 쓸 도구 — Python, NumPy, scikit-learn, PyTorch — 도 **전부 오픈소스이고, 한 푼도 내지 않습니다.**

인스타그램의 코드 절반 이상이 오픈소스 라이브러리였어요.

## 도구 4: 앱스토어 — 유통/결제/홍보를 통째로 위임하기

90년대에 소프트웨어를 만들면 어떻게 팔았을까요?

1. CD에 구워서
2. 박스에 담아서
3. 매장에 깔고
4. 신문에 광고하고
5. 결제 시스템 직접 만들고
6. 고객 지원 전화 직접 받고
7. 환불 처리도 직접

이 모든 게 **만드는 일**보다 더 큰 일이었어요. 그래서 작은 팀이 소프트웨어를 팔기는 거의 불가능했어요.

오늘날에는?

```
1. 앱 스토어에 등록
2. 끝.
```

**iOS App Store** (2008년 7월 출시), **Google Play** (2008년), **Steam** (게임), **npm/PyPI** (개발 라이브러리) — 이런 플랫폼들이 **유통/결제/평점/리뷰/환불을 다 처리해 줍니다.** 우리는 만드는 데만 집중하면 돼요. 물론 30%(애플) 또는 15%(구글의 일부 카테고리)는 떼이지만요.

인스타그램도 마찬가지였어요. 만든 직후 App Store에 올렸고, 결제·다운로드·평점·환불·전세계 유통을 다 Apple이 처리해 줬어요. **13명이 직접 했다면 불가능했을 일이에요.**

여기서 한 가지 중요한 효과 — App Store에 한 번 올리면 **전 세계 수억 명에게 동시에 도달**합니다. 90년대였다면 한 나라씩 유통 계약을 맺었어야 할 일이죠.

## 이 4가지가 합쳐지면

다시 인스타그램의 13명 이야기로 돌아가 봅시다. 그들이 직접 한 일과 외부에 의존한 일을 나눠 보면 이래요.

**직접 한 것:**
- 사진 필터 알고리즘 (창업의 핵심 기능)
- iOS 앱 코드
- 백엔드 비즈니스 로직 (좋아요, 팔로우 등)

**의존한 것:**
- 컴퓨터 자원 → **AWS 클라우드**
- 사진 저장소 → **AWS S3**
- 데이터베이스 → **PostgreSQL** (오픈소스)
- 코드의 절반 이상 → **오픈소스 라이브러리**
- 푸시/위치/카메라 → **Apple iOS API**
- 유통/결제/홍보 → **App Store**
- 페이스북 친구 가져오기 → **Facebook API**

이 4가지(API, 클라우드, 오픈소스, 앱스토어)가 없었다면, 인스타그램은 같은 서비스를 만들기 위해 **수천 명짜리 회사**여야 했을 거예요. 그런데 13명으로 가능해진 거죠.

이게 지난 20년의 변화입니다. **한 명이 만들 수 있는 것이 너무 커졌어요.**

## 그리고 지금, 다섯 번째가 추가되었습니다

여러분 짐작하셨죠. **AI** 입니다.

2022년 11월 30일, **OpenAI가 ChatGPT를 공개**했습니다. 출시 5일 만에 100만 사용자, 2개월 만에 1억 사용자 — 역사상 가장 빠르게 성장한 소비자 서비스가 됐어요. 그리고 이 시점부터 소프트웨어를 만드는 방식 자체가 또 한 번 변하기 시작했습니다.

| 과거에는... | 지금은... |
|-----------|----------|
| 디자이너 고용 | AI에게 "로고 만들어 줘" |
| 카피라이터 고용 | AI에게 "광고 문구 써줘" |
| 데이터 분석가 고용 | AI에게 "이 csv 분석해 줘" |
| 코드 작성에 1주 | AI 도움 받아 1일 |
| 번역 외주 | AI 즉시 번역 |
| 고객 지원 팀 | AI 챗봇 + 사람 1명 |

이제는 1인 개발자가, 디자이너 없이도, 마케팅 팀 없이도, 며칠 만에 서비스를 만들고 출시할 수 있는 시대가 됐어요.

13명짜리 인스타그램이 충격이었다면, 앞으로는 **3명짜리 — 어쩌면 1명짜리 — 유니콘 회사**도 나올 거예요. 진짜로요. 이미 사례들이 나오고 있어요.

그리고 그 한가운데에 있는 것이 **AI** — 다섯 번째 도구 — 입니다.

## "쓰는 사람"과 "아는 사람"의 차이

ChatGPT가 처음 나왔을 때 많은 분들이 "오, 챗봇이 좋아졌네" 하고 넘어갔어요. 그런데 어떤 분들은 그걸로 1인 기업을 차렸습니다. 어떤 분들은 회사의 고객 지원 비용을 90% 줄였고요. 어떤 분들은 한 달이면 못 만들 어플을 일주일 만에 만들었습니다.

같은 도구인데, 누군가에게는 그저 "신기한 챗봇"이고 누군가에게는 "회사를 바꾼 도구"였습니다. 차이가 뭘까요?

**그 도구의 작동 원리를 이해하고 있느냐**입니다.

ChatGPT를 그냥 쓰는 사람과, "이건 다음 단어를 확률적으로 예측하는 모델이고, 이래서 이런 걸 잘하고 저래서 저런 건 못해" 하고 아는 사람은 똑같은 ChatGPT를 들고도 만들 수 있는 게 100배쯤 차이가 납니다.

앞에서 본 4가지 도구도 마찬가지였어요. **API가 뭔지 아는 개발자**가 모르는 개발자보다 10배 빨리 일을 끝냈고, **클라우드를 잘 쓰는 개발자**가 그렇지 못한 개발자보다 100배 큰 서비스를 운영했어요. AI도 똑같은 길을 가고 있어요. 지금부터 5~10년 안에, **AI를 도구로 쓸 줄 아는 사람과 그렇지 못한 사람의 격차**가 그 어떤 시대보다 크게 벌어질 거예요.

그래서 이 자료의 목표가 그겁니다. "AI를 쓰는 사람"이 아니라 **"AI가 어떻게 돌아가는지 아는 사람"** 으로 만들어 드리는 것.

---

## 그런데 잠깐, 뭐부터 알아야 하죠?

AI라는 단어 안에는 너무 많은 게 들어 있어요.

- 머신러닝, 딥러닝, 강화학습, 생성형 AI, LLM, 멀티모달, RAG, 에이전트, 파인튜닝...

이 단어들을 처음 접하시면 머리가 빙빙 돌 거예요. 다 안 알아도 됩니다. **AI 분야의 90%는 "머신러닝"이고, 머신러닝의 가장 큰 두 갈래가 "분류"와 "회귀"입니다.** 그리고 머신러닝을 더 깊게 만든 게 "딥러닝"이에요.

그래서 우리는 이 순서로 갑니다.

```
파이썬(말하는 법) →  분류(범주 맞히기) →  회귀(숫자 맞히기)
                    └────────┬─────────┘
                          머신러닝
                              ↓
                        같은 두 문제를
                        신경망(=딥러닝)으로
                        다시 풀어 보기
```

이 다섯 단계를 다 거치고 나면, 세상에 나오는 거의 모든 AI 뉴스를 읽고 "아, 그게 그런 거구나" 하고 이해하실 수 있게 됩니다. ChatGPT 같은 LLM도 결국 거대한 분류 모델(다음 단어 분류)이기 때문이에요.

---

## "지금 시작하면 늦지 않을까요?"

가장 많이 듣는 질문입니다. 그래서 솔직하게 답해 드릴게요.

**박사 학위를 받고 AI 연구자가 되고 싶으시다면**, 솔직히 어렵습니다. 그건 5~10년 정도의 정통 트랙이 필요하고, 지금 시작하시는 분이 그 길을 따라잡기는 쉽지 않아요.

하지만 **"AI를 도구로 쓰면서 본인의 분야에 적용하고 싶으시다면"**, 지금이 인생에서 시작하기 가장 좋은 때입니다. 그리고 이 두 번째가 압도적으로 많은 사람의 정답이에요.

본인의 분야에서 데이터를 가장 잘 아는 사람이 가장 좋은 모델을 만듭니다. 의사가 의료 AI를 가장 잘 만들고, 변호사가 법률 AI를 가장 잘 만들어요. 이미 가지고 계신 그 도메인 지식 위에 AI라는 날개를 다는 것 — 그게 이 자료가 도와드리려는 일입니다.

## 요약

- AI는 잠깐의 유행이 아니라, **30년에 한 번 올 만한 도구의 변화**입니다.
- AI를 "쓰는 사람"과 "이해하는 사람"은 만들 수 있는 것이 100배쯤 다릅니다.
- AI 분야의 90%는 머신러닝이고, 우리는 분류 → 회귀 → 딥러닝 순서로 갑니다.
- 본인 분야에 AI를 더하면 가장 강력합니다. 지금이 시작하기 좋은 때입니다.
