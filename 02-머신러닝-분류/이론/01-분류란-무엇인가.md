# 2.1 분류란 무엇인가

## 비유로 시작합시다

여러분이 과수원의 직원이라고 해 봅시다. 사장님이 이런 일을 시키셨어요.

> "오늘 들어온 사과 박스 중에 **상품(上品)** 만 골라내."

여러분은 어떻게 사과를 분류하시겠어요?

- 색이 빨간 게 상품일까?
- 크기가 큰 게 상품일까?
- 표면이 매끈한 게 상품일까?

아마 **여러 가지를 다 봐서** 결정하실 거예요. 빨갛고, 크고, 매끈한 것 → 상품. 하지만 빨갛지만 작거나, 크지만 표면이 우둘투둘하면? 음... 좀 애매하네요.

그런데 여러분이 1년쯤 이 일을 하고 나면, **거의 사진만 봐도** 상품인지 아닌지 척 알게 돼요. 손으로 만져 보지 않아도, 어떤 자가 있는 것도 아닌데, 그냥 **경험으로 익힌 감**이 생기는 거죠.

**이게 바로 머신러닝 분류입니다.**

기계에게 "상품 사과 사진 1만 장"과 "비상품 사과 사진 1만 장"을 보여줘요. 기계는 이걸 보고 **상품과 비상품을 구분하는 패턴**을 스스로 찾아냅니다. 그 다음에 새로운 사과 사진을 줬을 때, "상품!" 또는 "비상품!" 하고 답해 주는 거예요.

---

## 분류의 정의

> **분류(Classification)** 는 입력을 **미리 정해진 몇 개의 카테고리** 중 하나로 분류하는 문제이다.

핵심 단어는 **"미리 정해진 몇 개"** 입니다. 카테고리가 끝없이 나오는 게 아니라, **유한한 개수**예요.

### 카테고리 개수에 따라

- **이진 분류 (Binary Classification)**: 2개 중 하나
  - 스팸 ↔ 햄
  - 합격 ↔ 불합격
  - 심장병 ↔ 정상
  - True ↔ False

- **다중 분류 (Multi-class Classification)**: 3개 이상 중 하나
  - 손글씨 0~9 (10개)
  - 강아지 / 고양이 / 토끼 / 햄스터
  - A학점 / B / C / D / F (5개)

이 장에서는 **다중 분류**(MNIST 0~9)부터 시작할 거예요. 사실 이진 분류는 다중 분류의 특수한 경우라서, 다중을 할 줄 알면 이진은 자동으로 됩니다.

### 분류가 아닌 것

- **회귀 (Regression)**: 숫자 예측. "이 집 값은 5억 원" 처럼 카테고리가 아니라 **연속된 숫자**를 예측. (3장에서 다룹니다)
- **클러스터링 (Clustering)**: 정답 없이 비슷한 것끼리 묶기. (이 자료에서는 안 다룹니다)
- **생성 (Generation)**: 새로운 것을 만들기. ChatGPT가 새 문장을 만드는 것 같은. (이 자료에서는 안 다룹니다)

## 분류 문제의 진짜 예시

### 1. 메일 스팸 분류
- **입력**: 메일 한 통의 텍스트
- **출력**: 0 (정상) 또는 1 (스팸)
- **활용**: Gmail의 자동 스팸 분류

### 2. 의료 진단
- **입력**: 흉부 X-ray 사진
- **출력**: 정상 / 폐렴 / 결핵 / 폐암 ...
- **활용**: 영상의학과 보조

### 3. 금융 사기 탐지
- **입력**: 신용카드 거래 한 건의 정보 (금액, 시간, 위치 등)
- **출력**: 정상 / 사기 의심
- **활용**: 카드사의 실시간 차단

### 4. 손글씨 인식
- **입력**: 손으로 쓴 숫자 사진
- **출력**: 0 ~ 9 중 하나
- **활용**: 우편번호 자동 인식, 우리가 다음 글에서 다룰 MNIST

### 5. 고객 이탈 예측
- **입력**: 고객의 가입 기간, 결제 횟수, 마지막 접속일 등
- **출력**: 다음 달에 떠날 가능성 높음 / 낮음
- **활용**: 사전 마케팅

이걸 다 같은 도구(`sklearn`)와 같은 코드 패턴으로 풀 수 있어요. 신기하죠?

## 분류는 어떻게 작동하나요?

직관적으로 한 번 그려 봅시다. 사과의 "지름(cm)"과 "무게(g)" 두 가지만 본다고 해 봐요.

```
무게(g)
   ↑
350│         ●●  ← 상품
   │       ●●●●
300│      ●●●●●
   │     ●●●●●
250│    ●●●●  ← 비상품
   │   ××●●
200│  ××××
   │ ×××××
150│××××
   └────────────────→ 지름(cm)
       6  7  8  9  10
```

- ● : 상품
- × : 비상품

이런 데이터가 있을 때, 분류 모델이 하는 일은 **두 그룹을 가르는 선을 찾는 것**이에요.

```
무게(g)
   ↑
350│         ●●
   │       ●●●●
300│      ●●●●●
   │ ━━━━━━━━━━━━━━━━ ← 이런 선!
250│    ●●●●
   │   ××●●
200│  ××××
   │ ×××××
150│××××
   └────────────────→
       6  7  8  9  10
```

이 선 위쪽에 있으면 "상품"이라고 답하고, 아래쪽에 있으면 "비상품"이라고 답하는 거예요.

새로운 사과가 들어오면, 그 사과의 지름과 무게가 선의 어느 쪽에 있는지를 보고 "상품/비상품"을 결정합니다.

### 알고리즘마다 그리는 "선"이 달라요

이게 분류 알고리즘들의 차이입니다. 우리가 이 장에서 배울 4가지 모델은 각각 다른 방식으로 선을 그어요.

| 알고리즘 | 선의 모양 | 비유 |
|---------|---------|------|
| **로지스틱 회귀** | 직선 | "두 그룹을 직선 하나로 가르자" |
| **SVM (선형)** | 직선이지만, "두 그룹에서 가장 멀리 떨어진" 직선 | "안전 거리를 최대로 두자" |
| **SVM (RBF)** | 곡선 | "꾸불꾸불해도 상관없어, 잘 나누면 돼" |
| **랜덤 포레스트** | 계단 모양의 여러 선 (트리들의 다수결) | "여러 단순한 결정 규칙들의 합의" |
| **KNN** | 선이 없음. 새 데이터에 가까운 K개를 보고 다수결 | "친구를 보면 그 사람을 안다" |

이 다섯 가지를 어떻게 골라야 하는지는 [2.6 여러 모델 비교하기](06-여러-모델.md)에서 자세히 다룹니다. 지금은 "다 다르구나" 정도면 됩니다.

---

## 그럼 우리는 무엇을 하면 되나요?

좋은 소식 하나 드릴게요. 머신러닝의 가장 어려운 부분(어떻게 선을 그을지 결정하는 수학)은 **이미 다 만들어져 있어요.** `scikit-learn`이라는 라이브러리에 들어 있습니다.

우리가 할 일은 이거예요.

1. **데이터를 잘 준비한다** (이게 사실상 일의 80%)
2. **적절한 모델을 고른다**
3. **학습시킨다 (`.fit`)**
4. **예측한다 (`.predict`)**
5. **잘했는지 평가한다**
6. **잘 못했으면 1~5를 반복한다**

이 6단계를 머신러닝 작업의 **"표준 사이클"** 이라고 부릅니다. 우리가 이 장에서 만나게 될 모든 게 다 이 사이클의 어느 한 부분이에요.

## 헷갈리기 쉬운 포인트

분류 문제에서 중요한 건 "정답 맞히기"보다 "어떤 종류의 오답을 허용할지"를 정하는 일입니다.

- 같은 정확도라도 오탐/미탐 비용이 다르면 좋은 모델이 달라집니다.
- 클래스 정의가 애매하면 모델 성능보다 라벨 품질 문제가 먼저 터집니다.
- 배포 환경의 임계값 정책까지 포함해 문제를 정의해야 실전 성능이 맞습니다.

## 이해를 더 단단하게 만드는 보강 설명

이 단원은 처음 읽을 때는 "아, 이런 개념이 있구나" 정도로 넘어가기 쉽지만, 실제로는 작은 차이가 나중에 큰 성능 차이로 이어집니다. 특히 **2.1 분류란 무엇인가** 같은 주제는 한 번에 완벽히 이해하려고 하기보다, 다음 세 가지를 반복하면서 이해를 깊게 만드는 편이 훨씬 효과적입니다.

첫째, 용어를 정의로 외우지 말고 상황으로 연결해 보세요. 예를 들어 "왜 이 개념이 필요한가"를 스스로 설명할 수 있어야 다음 단원에서 응용이 됩니다. 둘째, 코드가 나오면 결과를 먼저 예측한 뒤 실행해 보세요. 예측과 결과가 다를 때가 진짜 학습이 일어나는 순간입니다. 셋째, 같은 코드를 숫자나 파라미터만 바꿔 여러 번 돌려 보세요. 머신러닝과 딥러닝은 정답을 암기하는 과목이 아니라 **변화를 읽는 과목**이기 때문입니다.

또 하나 중요한 점은, 이 단원의 내용이 단독으로 존재하지 않는다는 사실입니다. 실제 프로젝트에서는 데이터 준비, 모델링, 평가, 배포가 끊기지 않고 이어집니다. 그래서 지금 배우는 **2.1 분류란 무엇인가**도 나중에 다른 단계와 반드시 연결됩니다. 이 연결을 의식하면서 읽으면, 단편 지식이 아니라 시스템 관점으로 이해하게 됩니다.

## 자주 하는 오해와 바로잡기

### 1) "이론만 이해하면 실습은 저절로 된다"
아닙니다. 이론 이해와 코드 실행 능력은 별개의 근육입니다. 둘을 같이 써야 합니다.

### 2) "예제 코드가 돌아갔으니 완전히 이해했다"
동작 확인은 시작일 뿐입니다. 파라미터를 바꿨을 때 결과가 어떻게 달라지는지까지 확인해야 이해가 완성됩니다.

### 3) "좋은 모델은 항상 하나다"
데이터 특성과 제약(시간, 메모리, 응답 속도)에 따라 최선의 선택은 달라집니다.

### 4) "지표 숫자만 높으면 끝이다"
지표는 해석이 필요합니다. 어떤 데이터에서, 어떤 분포에서, 어떤 기준으로 얻은 점수인지 함께 보아야 합니다.

### 5) "지금은 초반이니까 배포는 나중에 생각해도 된다"
초반부터 재현성과 저장 구조를 습관화해야 나중에 배포 단계에서 무너지지 않습니다.

## 실전 연결 시나리오

아래는 이 단원 내용을 실제 업무로 연결할 때 자주 쓰는 흐름입니다.

1. 문제를 한 문장으로 정의한다.  
예: "2.1 분류란 무엇인가 관점에서 입력으로부터 어떤 출력을 만들 것인가?"

2. 성공 기준을 지표로 정한다.  
예: 정확도/RMSE/F1/지연시간 같은 운영 가능한 숫자로 정의.

3. 가장 단순한 베이스라인을 만든다.  
복잡한 모델보다 먼저, 빠르게 재현 가능한 기본 모델을 세웁니다.

4. 한 번에 하나씩 개선한다.  
전처리, 모델, 튜닝, 임계값, 배치 크기 등은 동시에 바꾸지 않고 순차적으로 바꿉니다.

5. 결과를 로그와 함께 저장한다.  
실험 이름, 파라미터, 점수, 데이터 버전을 함께 남겨야 재현이 됩니다.

6. 실패 케이스를 분석한다.  
점수 평균보다 오분류/고오차 샘플을 분석할 때 개선 아이디어가 나옵니다.

7. 배포 가능 형태로 정리한다.  
모델 파일, 메타데이터, 입력 스키마, 테스트 스크립트까지 묶습니다.

## 한 번 더 정리

2.1 분류란 무엇인가는 단순히 시험용 개념이 아니라, 실제 문제를 풀 때 반복해서 쓰게 되는 실전 도구입니다. 처음에는 낯설 수 있지만, 작은 실험을 반복하면 감각이 생기고, 감각이 생기면 코드와 지표를 읽는 속도가 급격히 빨라집니다. 지금 단계에서 중요한 것은 완벽한 암기가 아니라, **작게 실행하고, 결과를 읽고, 다시 수정하는 루프**를 몸에 익히는 것입니다. 이 루프를 익히면 다음 장으로 넘어갈수록 학습 속도가 오히려 빨라집니다.
