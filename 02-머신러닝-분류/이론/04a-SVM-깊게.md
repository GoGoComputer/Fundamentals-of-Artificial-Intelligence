# 2.4a (보강) SVM 깊게 — 마진과 커널의 직관

## "왜 SVM이 그렇게 강력하다고 하는 거예요?"

[2.4 첫 모델 SVM](04-첫-모델-SVM.md)에서 SVM의 기본을 봤어요. 이 글에서는:

1. **하드 마진 vs 소프트 마진** — 현실 데이터를 어떻게 다루는가
2. **C 파라미터** — 가장 중요한 하이퍼파라미터
3. **커널 트릭** — 비선형 데이터를 다루는 마법
4. **다중 분류 전략** — One-vs-Rest, One-vs-One

이론 좀 들어가지만, 직관 위주로 갑니다.

---

## 1. 마진 (Margin)이 진짜 뭐예요?

기본 그림 다시:

```
       ●  ●               
    ●     ●                ← 그룹 A (●)
     ●  ●                  
                              
    ━━━━━━━━━━━━━━━━━━━━     ← 결정 경계 (분리선)
                              
       ×  ×              
      × ×                    ← 그룹 B (×)
        × × ×              
```

**마진**이란 결정 경계와 가장 가까운 데이터 점까지의 거리예요.

```
       ●  ●               
    ●     ●                
     ●  ●                  ← 가장 가까운 ● (서포트 벡터)
       ─ ─ ─ ─ ─ ─ ─ ─    ← 마진 위쪽 경계
                              
       ━━━━━━━━━━━━━━━━     ← 결정 경계
                              
       ─ ─ ─ ─ ─ ─ ─ ─    ← 마진 아래쪽 경계
       × ×              ← 가장 가까운 × (서포트 벡터)
        × × ×              
```

SVM의 목표: **이 마진을 최대로 하는 결정 경계 찾기.** 이걸 **최대 마진 분류기 (Maximum Margin Classifier)** 라고 부릅니다.

### 왜 마진이 크면 좋아요?

새 데이터가 왔을 때 결정 경계 근처일 가능성이 줄어들어요. 즉, **일반화가 잘 됨.**

비유: 횡단보도 가운데로 걸으면 자전거에 안 부딪힘. 가장자리로 걸으면 위험.

---

## 2. 하드 마진 (Hard Margin) — 이상적

**모든 데이터를 정확히 분리할 수 있어야** 작동.

```
       ●  ●               
    ●     ●  
     ●  ●  
       ─ ─ ─ ─ ─ ─ ─ ─
       
       ━━━━━━━━━━━━━━━━     ← 어느 ●도 마진 침범 없음
       
       ─ ─ ─ ─ ─ ─ ─ ─
       × ×            ← 어느 ×도 마진 침범 없음
        × × ×              
```

문제: **현실 데이터는 거의 항상 노이즈가 있어요.** 100% 분리되는 일이 없죠.

```
       ●  ●          ●×    ← !! 데이터 안에 ●가 ×사이에 있음
    ●     ● 
     ●  ●        ×● 
                              ← 직선으로 100% 분리 불가능
       
        ×  ×              
       × ×                
         × ×              
```

이런 데이터에 하드 마진을 적용하면 **해를 찾을 수 없어요.** SVC가 답을 못 줍니다.

---

## 3. 소프트 마진 (Soft Margin) — 현실적

> **"몇 개의 잘못 분류된 점은 봐줄게."**

이게 소프트 마진이에요. 일부 데이터가 마진을 침범하거나 잘못 분류되는 걸 허용해요.

대신 **얼마나 봐줄지를 C 파라미터로 조절.**

### C 파라미터의 의미

```
손실 = (1/마진) + C × (잘못 분류된 정도)
        ↑                  ↑
    크게 하고 싶음      작게 하고 싶음
```

`C`가 둘 사이의 균형을 조절해요.

#### C가 작으면 (예: C=0.01)

> "마진을 크게 만드는 게 더 중요해. 잘못 분류 좀 해도 OK."

```
       ●  ●          ●×       
    ●     ●            ←
     ●  ●        ×●       ← 일부 ●가 잘못 분류돼도
       ─ ─ ─ ─ ─ ─ ─ ─        ← 마진은 크게
       
       ━━━━━━━━━━━━━━━━
       
       ─ ─ ─ ─ ─ ─ ─ ─
        ×  ×              
       × ×                
         × ×              
```

→ 더 단순한 결정 경계 (정규화 강함)
→ 학습 데이터 정확도 ↓, 일반화 ↑
→ **과소적합 위험**

#### C가 크면 (예: C=100)

> "잘못 분류 절대 안 돼. 마진이 작아져도 좋아."

```
        ●  ●         ●×
     ●     ●         ╲  ╱  ←  복잡하게라도
      ●  ●            \\X     모든 점을 정확히
        ─ ─ ─ ─ ─ ─ ─ ─       분류하려 함
       
        ━━━━━━━━━━━━━━━━
        
        ─ ─ ─ ─ ─ ─ ─ ─
         ×  ×              
        × ×                
          × ×              
```

→ 더 복잡한 결정 경계 (정규화 약함)
→ 학습 데이터 정확도 ↑, 일반화 ↓
→ **과적합 위험**

### C 튜닝 가이드

```python
# 일반적인 시도 범위
C_values = [0.001, 0.01, 0.1, 1, 10, 100]

# RandomizedSearch로 자동 튜닝
from sklearn.model_selection import GridSearchCV
from scipy.stats import loguniform

param = {'C': loguniform(0.01, 100)}
search = RandomizedSearchCV(SVC(), param, n_iter=20, cv=5)
search.fit(X_train, y_train)
```

**일반 가이드:**
- 데이터가 노이즈 많음 → C 작게 (0.1~1)
- 데이터가 깨끗함 → C 크게 (10~100)
- 모르겠음 → C=1 (기본값) 부터

## 4. 커널 트릭 — 비선형의 마법

### 문제: 직선으로 못 나누는 데이터

```
       ●               ●
   ●       ×       ●
       ×   ×   ×        
   ●       ×       ●
       ●               ●
```

× 가 가운데, ●는 외곽. 직선으로는 절대 못 나눠요.

### 직관: 차원을 올려서 풀자

만약 우리가 데이터를 **3D로 옮길** 수 있다면 어떨까요?

```
2D (못 나뉨):                3D (나뉨!):
                              
●     ×     ●                 ●  ←━━━━━━━━━━━ 평면 ━━━━━━━━━━
                                  ↑
×  ×  ×  ×  ×                     │ (× 는 아래)
                                  │ × × × × ×
●     ×     ●                  ●
```

3D에서는 **평면 하나로** × 그룹과 ● 그룹을 나눌 수 있어요!

### 커널 트릭 — 직접 차원 안 올리고도

3D로 직접 옮기면 계산이 폭발해요. 변수 100개면 3D는 어떻게 할 거예요?

여기서 **커널 트릭**이 등장:

> **고차원으로 옮긴 척만 하고, 실제로는 안 옮긴다.**
> **수학적으로 동등한 결과를 더 빨리 계산.**

이게 천재적인 아이디어예요. 수학적 깊이는 복잡하지만, 결과적으로:

- **원본 데이터에 어떤 함수**(커널)**를 적용**
- **그 함수만으로 고차원 분리 효과**

### 주요 커널 종류

#### Linear Kernel
```python
SVC(kernel='linear')
```
직선으로 나눔 (커널 트릭 안 씀). 빠르지만 비선형 데이터 약함.

#### Polynomial Kernel
```python
SVC(kernel='poly', degree=3)
```
다항식 곡선. 특정 패턴에 강함.

#### RBF (Radial Basis Function) — 가장 자주 씀
```python
SVC(kernel='rbf', gamma='scale')
```
가우시안 함수 기반. **거의 모든 비선형 패턴 잡음.** SVM의 기본값.

#### Sigmoid
```python
SVC(kernel='sigmoid')
```
신경망의 활성화 함수처럼. 현대에서는 거의 안 씀.

### gamma 파라미터 (RBF, Polynomial)

`C`처럼 중요한 또 하나의 파라미터.

#### gamma 작으면 (예: 0.001)

> "각 데이터의 영향이 멀리까지 미침"

→ 부드러운 결정 경계, 단순한 모델, 과소적합 위험

#### gamma 크면 (예: 10)

> "각 데이터의 영향이 자기 근처에만"

→ 매우 복잡한 결정 경계, 과적합 위험

```python
# 보통 시도 범위
gamma_values = ['scale', 'auto', 0.001, 0.01, 0.1, 1, 10]
```

`'scale'` 이 기본값이고, 보통 잘 동작해요.

### C와 gamma — 한 그림으로

```
        gamma 작음         gamma 큼
       (단순한 경계)      (복잡한 경계)
        ┌──────────────┬──────────────┐
   C    │    과소적합    │    중간       │
  작음   │ (너무 단순)    │              │
        ├──────────────┼──────────────┤
   C    │    중간        │   과적합      │
  큼    │              │ (너무 복잡)    │
        └──────────────┴──────────────┘
```

**좋은 조합 찾기:** GridSearchCV 또는 RandomizedSearchCV.

## 5. 다중 분류 (Multi-class)

SVM은 원래 **이진 분류** 알고리즘이에요. 클래스가 3개 이상이면 어떻게 하죠?

### 전략 1: One-vs-Rest (OvR)

각 클래스마다 "이 클래스 vs 나머지" 분류기를 만들기.

```
3개 클래스 (A, B, C):
- 분류기 1: A vs (B+C)
- 분류기 2: B vs (A+C)
- 분류기 3: C vs (A+B)

새 데이터 → 각 분류기 점수 → 가장 높은 점수의 클래스
```

**장점:** 분류기가 N개 (적음)
**단점:** 클래스 불균형 발생 가능

### 전략 2: One-vs-One (OvO)

모든 쌍마다 분류기를 만들기.

```
3개 클래스 (A, B, C):
- 분류기 1: A vs B
- 분류기 2: A vs C
- 분류기 3: B vs C

새 데이터 → 모든 분류기 → 다수결로 결정
```

**장점:** 각 분류기가 작은 데이터로 학습 가능
**단점:** 분류기가 N(N-1)/2개 (많음)

### sklearn의 SVC는?

기본은 **OvO**. 변경 가능:

```python
SVC(decision_function_shape='ovr')   # OvR
SVC(decision_function_shape='ovo')   # OvO (기본)
```

10개 클래스 분류라면:
- OvR: 10개 분류기
- OvO: 45개 분류기

데이터 양과 클래스 수에 따라 다르지만, **클래스 적으면(<10) OvO, 많으면 OvR이 보통**.

## 6. SVM의 한계

SVM은 강력하지만 만능은 아니에요.

### 한계 1: 큰 데이터에 매우 느림

학습 시간이 O(n²) ~ O(n³). 데이터 10만 개 이상이면 답이 안 나옴.

→ 큰 데이터엔 **로지스틱 회귀, XGBoost, 신경망**이 더 빠름.

### 한계 2: 확률 출력이 느림

```python
SVC(probability=True)   # ← 매우 느려짐
```

→ 확률이 필요하면 **로지스틱 회귀**가 자연스러움.

### 한계 3: 정규화 필수

거리 기반이라 특성 스케일에 민감. **항상 StandardScaler 사용.**

### 한계 4: 결과 해석이 어려움

선형 SVM은 가중치 해석 가능. RBF/Polynomial 커널은 거의 블랙박스.

→ 해석이 중요하면 **로지스틱 회귀** 또는 **결정 트리**.

### 한계 5: 노이즈에 민감

소프트 마진이 있어도 노이즈가 매우 많으면 흔들림.

→ 노이즈 많으면 **랜덤 포레스트**가 더 안정적.

---

## 정리: SVM 사용 가이드

### 언제 SVM이 좋아요?

- **작거나 중간 크기의 데이터** (~1만 개)
- **고차원 특성** (텍스트 분류, 유전자 데이터)
- **정확도가 매우 중요**
- **특성보다 샘플이 적은 경우** (기존 ML이 약한 곳)

### 언제 다른 걸?

- 큰 데이터 → **XGBoost / LightGBM**
- 확률 출력 필요 → **로지스틱 회귀**
- 해석이 중요 → **결정 트리 / 로지스틱 회귀**
- 매우 큰 데이터 + 비정형 → **신경망**

### 실용 팁

```python
from sklearn.svm import SVC
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import loguniform

# 항상 정규화와 함께
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('svm', SVC(kernel='rbf')),
])

# 자동 튜닝
param_dist = {
    'svm__C': loguniform(0.01, 100),
    'svm__gamma': loguniform(0.001, 10),
}
search = RandomizedSearchCV(pipe, param_dist, n_iter=30, cv=5, n_jobs=-1)
search.fit(X_train, y_train)

print(f"최고 점수: {search.best_score_:.4f}")
print(f"최고 조합: {search.best_params_}")
```

## 한 줄 요약

```
- 마진을 최대로 하는 결정 경계 찾기
- C 작음 = 단순/일반화↑, C 큼 = 복잡/정확도↑
- 비선형은 RBF kernel + gamma 튜닝
- 작은 데이터의 강자, 큰 데이터에는 다른 거 쓰기
```
