# 4.3 첫 신경망 만들기

## 목표: MNIST를 신경망으로

지금까지 배운 걸 다 합쳐서, MNIST 손글씨를 분류하는 첫 신경망을 만들어 봅시다.

2장에서 SVM으로 만든 모델이 약 91% 정확도였죠. 신경망으로 **97% 이상** 만들어 봅시다.

---

## 전체 흐름

```
1. 데이터 준비 (DataLoader)
   ↓
2. 모델 정의 (nn.Module)
   ↓
3. 손실 함수와 옵티마이저 정의
   ↓
4. 학습 루프 (epoch을 반복하며)
   ├── forward
   ├── loss 계산
   ├── backward
   └── step
   ↓
5. 평가
```

---

## 1. 데이터 준비

PyTorch는 `torchvision`이라는 패키지로 MNIST를 쉽게 가져올 수 있어요.

```python
import torch
from torch.utils.data import DataLoader
from torchvision import datasets, transforms

# 데이터 변환: 텐서로 + 0~1 범위 정규화
transform = transforms.Compose([
    transforms.ToTensor(),   # PIL 이미지 → 텐서, 0~255 → 0~1
])

# MNIST 다운로드
train_dataset = datasets.MNIST(
    root='./data', train=True, download=True, transform=transform,
)
test_dataset = datasets.MNIST(
    root='./data', train=False, download=True, transform=transform,
)

print(f"학습 데이터: {len(train_dataset)}개")    # 60000
print(f"평가 데이터: {len(test_dataset)}개")    # 10000

# 한 샘플 보기
img, label = train_dataset[0]
print(f"이미지 모양: {img.shape}")    # torch.Size([1, 28, 28])
print(f"라벨: {label}")
```

### DataLoader: 미니배치를 자동으로 만들어 주는 도구

신경망은 보통 **미니배치(mini-batch)** 단위로 학습합니다. 32개나 64개씩 묶어서요. DataLoader가 이걸 자동으로 해 줘요.

```python
batch_size = 64

train_loader = DataLoader(
    train_dataset,
    batch_size=batch_size,
    shuffle=True,        # 매 epoch마다 섞기 (중요!)
)

test_loader = DataLoader(
    test_dataset,
    batch_size=batch_size,
    shuffle=False,
)

# 한 배치 보기
images, labels = next(iter(train_loader))
print(f"배치 모양: {images.shape}")    # torch.Size([64, 1, 28, 28])
print(f"라벨 모양: {labels.shape}")    # torch.Size([64])
```

### 배치 크기 (batch_size) 어떻게 정해요?

| batch_size | 영향 |
|-----------|------|
| 작음 (16, 32) | 더 다양한 업데이트, 더 안정적이지만 느림 |
| 큼 (256, 512) | 빠른 계산, 일반화 약간 약함 |

**시작점: 32 또는 64.** 그리고 메모리 부족하면 줄이고, 더 빠르게 하고 싶으면 늘리세요.

## 2. 모델 정의

```python
import torch.nn as nn

class MNISTClassifier(nn.Module):
    def __init__(self):
        super().__init__()
        self.flatten = nn.Flatten()              # 2D → 1D
        self.fc1 = nn.Linear(784, 256)           # 784 → 256
        self.fc2 = nn.Linear(256, 128)           # 256 → 128
        self.fc3 = nn.Linear(128, 10)            # 128 → 10 (숫자 0~9)

    def forward(self, x):
        x = self.flatten(x)                      # (B, 1, 28, 28) → (B, 784)
        x = torch.relu(self.fc1(x))
        x = torch.relu(self.fc2(x))
        x = self.fc3(x)                          # 마지막은 활성화 X
        return x

device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
model = MNISTClassifier().to(device)
print(model)
```

```
MNISTClassifier(
  (flatten): Flatten(start_dim=1, end_dim=-1)
  (fc1): Linear(in_features=784, out_features=256, bias=True)
  (fc2): Linear(in_features=256, out_features=128, bias=True)
  (fc3): Linear(in_features=128, out_features=10, bias=True)
)
```

### 왜 마지막에 활성화 함수를 안 줘요?

분류 모델의 마지막 출력은 **각 클래스에 대한 점수(logit)** 예요. 우리가 쓸 손실 함수(`CrossEntropyLoss`)가 **자동으로 softmax를 적용**해서 확률로 바꿔 줘요. 그래서 우리가 따로 안 해도 됩니다.

---

## 3. 손실 함수와 옵티마이저

### 손실 함수 (Loss Function)

**얼마나 틀렸는지를 점수로 나타내는 함수**예요. 분류에서는 보통 `CrossEntropyLoss`를 씁니다.

```python
loss_fn = nn.CrossEntropyLoss()
```

이 손실 함수는:
- 입력: 모델의 출력 (logit, [-∞, ∞])
- 타겟: 정답 라벨 (정수, 0~9)
- 출력: 손실 값 (낮을수록 좋음)

내부적으로 softmax를 적용한 다음, 정답 클래스의 확률을 -log 한 값을 계산해요. 이론적으로는 복잡하지만, 우리는 그냥 한 줄로 쓰면 돼요.

### 옵티마이저 (Optimizer)

**가중치를 업데이트하는 알고리즘**이에요. 가장 자주 쓰는 게 **Adam**입니다.

```python
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
```

- `model.parameters()`: 모델의 모든 가중치를 옵티마이저에게 알려줌
- `lr=0.001`: 학습률 (한 번에 가중치를 얼마나 바꿀지)

옵티마이저 종류는 [4.5 더 깊게](05-깊게.md)에서 자세히 다룹니다. 지금은 **Adam이 안전한 선택**이라고만 아세요.

---

## 4. 학습 루프

여기가 핵심이에요. PyTorch의 학습 루프는 **거의 항상 같은 패턴**입니다.

```python
n_epochs = 10

for epoch in range(n_epochs):
    # 학습 모드
    model.train()
    
    epoch_loss = 0.0
    correct = 0
    total = 0

    for images, labels in train_loader:
        # 1. 데이터를 GPU로
        images, labels = images.to(device), labels.to(device)

        # 2. 그래디언트 초기화 (정말 중요!)
        optimizer.zero_grad()

        # 3. forward
        outputs = model(images)

        # 4. loss 계산
        loss = loss_fn(outputs, labels)

        # 5. backward (그래디언트 계산)
        loss.backward()

        # 6. step (가중치 업데이트)
        optimizer.step()

        # 통계
        epoch_loss += loss.item()
        _, predicted = outputs.max(1)
        total += labels.size(0)
        correct += predicted.eq(labels).sum().item()

    avg_loss = epoch_loss / len(train_loader)
    accuracy = correct / total
    print(f"Epoch {epoch+1}/{n_epochs}  Loss: {avg_loss:.4f}  Acc: {accuracy:.4f}")
```

이 6단계를 외우세요. **모든 PyTorch 학습 코드의 본질**입니다.

### 한 줄씩 뜯어보기

#### `model.train()`
모델을 학습 모드로. Dropout 같은 게 활성화돼요. (다음 글)

#### `optimizer.zero_grad()`
**가장 자주 잊는 줄!** PyTorch는 그래디언트를 자동으로 누적해요. 매 배치마다 초기화 안 하면 그래디언트가 계속 쌓여서 학습이 망가집니다.

#### `outputs = model(images)` (forward)
입력을 모델에 통과시켜서 출력을 얻음.

#### `loss = loss_fn(outputs, labels)`
출력과 정답을 비교해서 손실 계산.

#### `loss.backward()`
**자동 미분의 마법.** 모든 가중치의 그래디언트가 자동으로 계산돼서 `weight.grad` 속성에 저장됨.

#### `optimizer.step()`
계산된 그래디언트를 사용해서 가중치를 업데이트.

이게 한 사이클이에요. 이걸 모든 배치에 대해, 그리고 모든 epoch에 대해 반복합니다.

### 출력 예시

```
Epoch 1/10  Loss: 0.3247  Acc: 0.9051
Epoch 2/10  Loss: 0.1305  Acc: 0.9614
Epoch 3/10  Loss: 0.0883  Acc: 0.9737
Epoch 4/10  Loss: 0.0641  Acc: 0.9806
Epoch 5/10  Loss: 0.0478  Acc: 0.9856
...
```

Loss가 줄어들고 Accuracy가 올라가면 **학습이 잘 되고 있다는 뜻**이에요. 만약 Loss가 안 줄어들거나 NaN이 되면 뭔가 잘못된 거예요.

## 5. 평가

학습이 끝나면 test 데이터로 평가합니다.

```python
model.eval()    # 평가 모드 (Dropout 등 끔)

correct = 0
total = 0

with torch.no_grad():    # 평가 시엔 그래디언트 계산 X (메모리 절약)
    for images, labels in test_loader:
        images, labels = images.to(device), labels.to(device)
        outputs = model(images)
        _, predicted = outputs.max(1)
        total += labels.size(0)
        correct += predicted.eq(labels).sum().item()

accuracy = correct / total
print(f"테스트 정확도: {accuracy:.4f}")
```

```
테스트 정확도: 0.9778
```

**97.78%!** SVM의 91%보다 훨씬 좋아졌어요.

### 두 가지 핵심 메소드 요약

- **`model.eval()`**: 평가 모드. Dropout/BatchNorm이 다르게 동작.
- **`with torch.no_grad():`**: 그래디언트 계산 안 함. 평가 시 필수.

학습 시엔 `model.train()`, 평가 시엔 `model.eval() + torch.no_grad()`. 외워두세요.

## 전체 코드를 한 곳에

```python
import torch
import torch.nn as nn
from torch.utils.data import DataLoader
from torchvision import datasets, transforms


# 설정
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
batch_size = 64
n_epochs = 10
learning_rate = 0.001


# 데이터
transform = transforms.Compose([transforms.ToTensor()])
train_dataset = datasets.MNIST('./data', train=True, download=True, transform=transform)
test_dataset = datasets.MNIST('./data', train=False, download=True, transform=transform)

train_loader = DataLoader(train_dataset, batch_size=batch_size, shuffle=True)
test_loader = DataLoader(test_dataset, batch_size=batch_size, shuffle=False)


# 모델
class MNISTClassifier(nn.Module):
    def __init__(self):
        super().__init__()
        self.flatten = nn.Flatten()
        self.fc1 = nn.Linear(784, 256)
        self.fc2 = nn.Linear(256, 128)
        self.fc3 = nn.Linear(128, 10)

    def forward(self, x):
        x = self.flatten(x)
        x = torch.relu(self.fc1(x))
        x = torch.relu(self.fc2(x))
        return self.fc3(x)


model = MNISTClassifier().to(device)
loss_fn = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=learning_rate)


# 학습
for epoch in range(n_epochs):
    model.train()
    total_loss = 0
    correct = 0
    total = 0

    for images, labels in train_loader:
        images, labels = images.to(device), labels.to(device)

        optimizer.zero_grad()
        outputs = model(images)
        loss = loss_fn(outputs, labels)
        loss.backward()
        optimizer.step()

        total_loss += loss.item()
        _, predicted = outputs.max(1)
        total += labels.size(0)
        correct += predicted.eq(labels).sum().item()

    print(f"Epoch {epoch+1}/{n_epochs}  Loss: {total_loss/len(train_loader):.4f}  "
          f"Acc: {correct/total:.4f}")


# 평가
model.eval()
correct, total = 0, 0
with torch.no_grad():
    for images, labels in test_loader:
        images, labels = images.to(device), labels.to(device)
        outputs = model(images)
        _, predicted = outputs.max(1)
        total += labels.size(0)
        correct += predicted.eq(labels).sum().item()

print(f"\n테스트 정확도: {correct/total:.4f}")
```

이 코드를 그대로 Colab(GPU 모드)에서 돌려보세요. 약 1~2분이면 끝나고 97% 이상 정확도가 나올 거예요.
