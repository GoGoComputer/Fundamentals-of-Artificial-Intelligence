# 3.4 회귀 평가 지표

## "얼마나 잘 맞았어?"의 4가지 답

분류는 "맞췄나 틀렸나"가 명확했죠. 회귀는 답이 숫자라서 "얼마나 맞았나"가 좀 더 미묘해요. 예측이 100인데 실제가 110이라면? 90% 맞은 건가요? 10 차이만큼 틀린 건가요?

이걸 표현하는 방식이 4가지 있어요.

1. **MSE** (Mean Squared Error)
2. **RMSE** (Root Mean Squared Error)
3. **MAE** (Mean Absolute Error)
4. **R²** (R-squared, 결정계수)

각각 언제 어떤 걸 봐야 하는지 정리해 봅시다.

---

## 출발점: 잔차 (Residual)

모든 회귀 평가 지표는 **잔차**에서 시작합니다.

```
잔차 = 실제값 - 예측값
```

예: 실제 집값 10억인데 모델이 8억으로 예측 → 잔차 = 2억

샘플마다 잔차가 하나씩 있어요. 이걸 어떻게 종합하느냐가 지표의 차이입니다.

---

## 1. MSE (Mean Squared Error, 평균 제곱 오차)

가장 기본적인 지표예요.

```
MSE = (잔차₁² + 잔차₂² + ... + 잔차ₙ²) / n
```

각 잔차를 **제곱**해서 평균을 냅니다.

### 왜 제곱하나요?

두 가지 이유예요.

1. **양수와 음수가 상쇄되지 않게.** 그냥 평균을 내면 +5와 -5가 0이 돼요.
2. **큰 오차에 더 큰 페널티.** 1²=1, 2²=4, 10²=100. 큰 오차가 더 무겁게 평가됨.

### 코드

```python
from sklearn.metrics import mean_squared_error
mse = mean_squared_error(y_test, y_pred)
```

### 단점

값이 직관적이지 않아요. **단위가 (실제 단위)²** 이거든요. 집값이 억 단위면 MSE는 억²인 거죠. "MSE 24억²" 들으면 감이 안 와요.

→ 그래서 RMSE를 씁니다.

---

## 2. RMSE (Root Mean Squared Error, 평균 제곱근 오차)

MSE에 제곱근(√)을 씌운 거예요.

```
RMSE = √MSE
```

### 코드

```python
from sklearn.metrics import mean_squared_error
import numpy as np

rmse = np.sqrt(mean_squared_error(y_test, y_pred))

# 또는 sklearn 1.4+ 에서
rmse = mean_squared_error(y_test, y_pred, squared=False)

# 또는 root_mean_squared_error
from sklearn.metrics import root_mean_squared_error
rmse = root_mean_squared_error(y_test, y_pred)
```

### 장점

**단위가 원래 단위로 돌아옴.** 집값 RMSE 4.93이면 "평균적으로 약 5천 달러 정도 틀린다" 라고 말할 수 있어요.

### 가장 자주 쓰는 지표

회귀에서 단 하나만 본다면, **RMSE**입니다. 가장 표준적이에요.

---

## 3. MAE (Mean Absolute Error, 평균 절대 오차)

제곱 대신 **절댓값**을 씁니다.

```
MAE = (|잔차₁| + |잔차₂| + ... + |잔차ₙ|) / n
```

### 코드

```python
from sklearn.metrics import mean_absolute_error
mae = mean_absolute_error(y_test, y_pred)
```

### MAE vs RMSE — 차이가 뭐예요?

가장 중요한 차이는 **이상치(outlier) 처리**입니다.

| | MAE | RMSE |
|---|---|---|
| 큰 오차 처리 | 작게 평가 | 크게 평가 |
| 이상치 영향 | 적음 | 큼 |
| 직관성 | 매우 직관적 | 비교적 직관적 |

### 예시

샘플 5개의 잔차가 다음과 같다고 해 봐요.
- 잔차들: 1, 2, 3, 4, 50

```
MAE  = (1+2+3+4+50) / 5 = 12
RMSE = √((1+4+9+16+2500)/5) = √506 = 22.5
```

같은 데이터인데 RMSE가 거의 2배예요. **이상치(50) 하나가 RMSE를 크게 끌어올렸어요.**

### 언제 어떤 걸?

- **이상치를 무겁게 봐야 할 때** → RMSE
  - 큰 실수 한 번이 작은 실수 여러 번보다 치명적인 경우
  - 예: 의료 진단 (큰 오류는 사람 죽음)

- **이상치를 평등하게 봐야 할 때** → MAE
  - 모든 오차가 동등하게 중요한 경우
  - 예: 일반 수요 예측 (가끔 큰 실수는 어쩔 수 없음)

---

## 4. R² (R-squared, 결정계수)

이건 좀 다른 종류의 지표예요. **0~1 사이의 비율**로 나옵니다.

```
R² = 1 - (모델의 잔차 제곱합) / (평균 예측의 잔차 제곱합)
```

복잡해 보이지만 의미는 단순해요.

> **"가장 단순한 예측(평균값으로만 예측)에 비해, 우리 모델이 얼마나 더 잘 하는가?"**

### 해석

- **R² = 1.0**: 완벽한 예측
- **R² = 0.0**: 평균값으로 예측한 거랑 똑같음 (모델 의미 없음)
- **R² < 0**: 평균보다도 못함 (드물지만 가능)

일반적으로:
- R² > 0.9: 매우 좋음
- R² > 0.7: 좋음
- R² > 0.5: 보통
- R² < 0.3: 안 좋음

### 코드

```python
from sklearn.metrics import r2_score
r2 = r2_score(y_test, y_pred)
```

### 장점

- **단위 무관** — 비율로 나오니까 도메인이 달라도 비교 가능
- **직관적** — "모델이 데이터의 X%를 설명함"
- **모델 비교에 좋음**

### 한계

- **변수 추가하면 항상 R²가 늘어남** (의미 없는 변수도)
- 그래서 **Adjusted R²** 라는 변형도 있음

---

## 4가지 지표 한 번에 보기

```python
import numpy as np
from sklearn.metrics import (
    mean_squared_error,
    mean_absolute_error,
    r2_score,
)

mse = mean_squared_error(y_test, y_pred)
rmse = np.sqrt(mse)
mae = mean_absolute_error(y_test, y_pred)
r2 = r2_score(y_test, y_pred)

print(f"MSE:  {mse:.4f}")
print(f"RMSE: {rmse:.4f}")
print(f"MAE:  {mae:.4f}")
print(f"R²:   {r2:.4f}")
```

```
MSE:  24.29
RMSE: 4.93
MAE:  3.18
R²:   0.6688
```

### 어떻게 해석하나요?

- **RMSE 4.93**: "평균적으로 ~$4,930만큼 틀림"
- **MAE 3.18**: "보통은 ~$3,180만큼 틀림"
- **MAE < RMSE**: 일부 큰 오차가 있음 (이상치)
- **R² 0.67**: "데이터 변동의 67%를 설명"

→ **나쁘지 않은 모델, 하지만 이상치 처리 개선 여지가 있음**

## 시각화: 4가지 지표 비교

```python
import matplotlib.pyplot as plt

fig, axes = plt.subplots(1, 2, figsize=(15, 5))

# 1. 예측 vs 실제
axes[0].scatter(y_test, y_pred, alpha=0.5)
axes[0].plot([y_test.min(), y_test.max()],
             [y_test.min(), y_test.max()], 'r--', lw=2)
axes[0].set_xlabel('실제값')
axes[0].set_ylabel('예측값')
axes[0].set_title(f'예측 vs 실제\nRMSE={rmse:.2f}, R²={r2:.4f}')

# 2. 잔차 분포
residuals = y_test - y_pred
axes[1].hist(residuals, bins=30, color='steelblue', edgecolor='black')
axes[1].axvline(0, color='red', linestyle='--')
axes[1].set_xlabel('잔차')
axes[1].set_ylabel('빈도')
axes[1].set_title(f'잔차 분포\nMAE={mae:.2f}')

plt.tight_layout()
plt.show()
```

## 어느 지표를 봐야 하나요?

상황별 권장 지표를 정리하면:

| 상황 | 주요 지표 |
|------|---------|
| 일반 회귀 (모델 선택) | **RMSE** |
| 결과를 사람에게 설명할 때 | **MAE** (가장 직관적) |
| 모델 비교 (단위 다른 경우) | **R²** |
| 이상치를 절대 놓치면 안 됨 | **RMSE** |
| 이상치를 가볍게 취급 | **MAE** |
| 비즈니스 KPI에 포함 | **MAPE** (다음에 설명) |

---

## 보너스: 다른 지표들

알아두시면 가끔 유용한 지표들.

### MAPE (Mean Absolute Percentage Error)

```
MAPE = 평균(|잔차| / 실제값) × 100
```

**오차를 비율(%)로 표현**하는 지표예요.

```python
import numpy as np

mape = np.mean(np.abs((y_test - y_pred) / y_test)) * 100
print(f"MAPE: {mape:.2f}%")
```

비즈니스 보고서에 자주 써요. "평균 5% 정도 틀렸어요" 하면 임원들이 잘 이해하시거든요.

⚠️ **단점**: 실제값이 0이거나 매우 작으면 폭발해요. 0이 있는 데이터에선 못 씀.

### Median Absolute Error

평균 대신 중앙값을 쓰는 MAE예요. 이상치에 더 강해요.

```python
from sklearn.metrics import median_absolute_error
medae = median_absolute_error(y_test, y_pred)
```

### Max Error

가장 큰 단일 오차. "최악의 경우" 보고할 때.

```python
from sklearn.metrics import max_error
me = max_error(y_test, y_pred)
```

## 정리

```python
# 가장 자주 보는 4종 세트
import numpy as np
from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score

rmse = np.sqrt(mean_squared_error(y_test, y_pred))
mae = mean_absolute_error(y_test, y_pred)
r2 = r2_score(y_test, y_pred)

print(f"RMSE: {rmse:.4f}    ← 일반적으로 가장 많이 봄")
print(f"MAE:  {mae:.4f}    ← 직관적, 이상치에 강함")
print(f"R²:   {r2:.4f}     ← 0~1 비율, 모델 비교에 좋음")
```

**최소 권장: RMSE + MAE 같이 보세요.** 둘 차이가 크면 이상치 문제가 있는 것.
