# 1장. 파이썬 기초

> 이 장의 목표는 파이썬을 깊게 마스터하는 것이 아니라, 다음 장의 머신러닝 코드를 읽고 직접 바꿔 볼 수 있는 상태를 만드는 것입니다.
> 그래서 범위를 넓히기보다, 실제로 자주 쓰는 문법만 골라서 반복합니다.

머신러닝을 처음 배우는 분들이 흔히 하는 오해가 하나 있습니다. "일단 파이썬을 몇 달 완벽하게 공부한 뒤에 머신러닝을 시작해야 하지 않을까?" 하는 생각입니다. 그런데 실제로는 그렇게 하지 않아도 됩니다. 필요한 것은 파이썬 전체가 아니라, **데이터를 담고, 반복하고, 조건을 나누고, 함수를 호출하는 감각**입니다.

이 장은 바로 그 감각을 만드는 데 집중합니다. 따라서 문법의 모든 예외를 다루기보다, 다음 장에서 바로 다시 만나게 될 패턴을 우선적으로 익히게 됩니다.

## 이 장을 끝내면 무엇이 달라지나요?

이 장을 끝내면 단순히 문법을 외운 상태가 아니라, 다음과 같은 실질적인 변화가 생겨야 합니다.

- 파이썬 코드가 나와도 줄 단위로 겁먹지 않고 읽기 시작할 수 있다.
- 리스트와 딕셔너리로 데이터를 어떻게 담는지 감이 생긴다.
- 반복문과 조건문이 왜 머신러닝 코드의 대부분을 차지하는지 이해하게 된다.
- 함수 호출 구조를 보고 "입력은 뭐고 결과는 뭐구나"를 파악할 수 있다.
- 2장부터 나오는 `X_train`, `y_test`, `model.fit()` 같은 코드를 봐도 완전히 낯설지 않다.

즉, 이 장은 파이썬 자체의 종착점이 아니라 **머신러닝 코드로 넘어가기 위한 발판**입니다.

## 추천 학습 순서

이 장은 다음 순서로 공부하는 편이 가장 효과적입니다.

1. 이론 글을 먼저 읽으며 큰 그림을 잡습니다.
2. 바로 연결된 실습 파일을 실행하며 같은 개념을 손으로 확인합니다.
3. 코드 결과를 보기 전에 먼저 "어떤 결과가 나올지"를 예상해 봅니다.
4. 실행 후 예상과 다르면, 왜 달랐는지를 반드시 짚고 넘어갑니다.
5. 마지막에는 코드를 조금씩 망가뜨려 보며 에러 메시지까지 읽어 봅니다.

이 방식이 중요한 이유는 파이썬이 머리로만 읽어서는 빨리 익지 않기 때문입니다. 특히 변수 재할당, 인덱스 0부터 시작하기, `==`와 `=`의 차이, `if`와 들여쓰기 같은 요소는 **직접 여러 번 손으로 부딪혀 봐야** 감각이 생깁니다.

## 이미 파이썬을 조금 아는 분에게

이 장은 파이썬 입문 전체가 아니라 **머신러닝 입문에 필요한 최소 핵심만 압축한 장**입니다. 따라서 이미 아래 네 가지가 편하다면, 1장은 빠르게 훑고 2장으로 넘어가셔도 무방합니다.

- 리스트와 딕셔너리를 자유롭게 읽고 수정할 수 있다
- for문과 if문을 섞어서 간단한 로직을 작성할 수 있다
- 함수를 정의하고, 인자를 넘기고, 결과를 받을 수 있다
- `import numpy as np` 같은 기본 import 문법이 낯설지 않다

하지만 문법을 아는 것과 머신러닝 문맥에서 익숙한 것은 다르기 때문에, 파이썬 경험이 있어도 실습 파일은 한 번 정도 보는 편을 권합니다.
