## 1. print: 컴퓨터에게 말 시키기

프로그래밍을 배울 때 세상의 모든 사람이 처음 쓰는 한 줄이 있어요.

```python
print("Hello, World!")
```

이 한 줄을 Colab 셀에 입력하시고 `Shift + Enter`를 눌러 보세요.

```
Hello, World!
```

이렇게 나옵니다. 축하드립니다. 컴퓨터가 여러분의 명령에 처음으로 답을 했어요.

### print가 뭐예요?

**`print`는 "괄호 안에 있는 걸 화면에 보여줘"** 라는 뜻의 명령어입니다. 파이썬에 이미 만들어져 있는 함수예요. 우리는 그냥 가져다 쓰면 됩니다.

```python
print("안녕하세요")
print(2025)
print(3.14)
```

문자, 숫자, 소수점 다 됩니다. 결과는 이렇게 나와요.

```
안녕하세요
2025
3.14
```

### 한 번에 여러 개 출력하기

`print` 안에 콤마(`,`)로 여러 개를 넣으면, 띄어쓰기 한 칸으로 구분해서 한 줄에 출력해 줍니다.

```python
print("내 이름은", "철수", "입니다")
```

```
내 이름은 철수 입니다
```

### 따옴표는 왜 쓰는 거예요?

좋은 질문입니다. 차이를 보세요.

```python
print(2025)        # 숫자 2025를 출력
print("2025")      # 문자 "2025"를 출력
```

둘 다 화면에는 똑같이 `2025`가 나오지만, **컴퓨터에게는 완전히 다른 것**입니다.

- `2025` → 숫자. 다른 숫자랑 더하기, 곱하기 가능.
- `"2025"` → 문자. 다른 문자랑 이어 붙이기 가능.

```python
print(2025 + 1)        # 2026
print("2025" + "1")    # 20251 (이어 붙임)
```

이게 처음 보면 헷갈려요. **따옴표(`"`)로 감싼 건 "글자"고, 따옴표 없는 숫자는 "숫자"** 라는 것만 기억하세요.

### Colab만의 특별한 한 가지

Colab에서는 셀의 **마지막 줄**은 `print` 없이 그냥 적어도 자동으로 출력됩니다.

```python
2025
```

이렇게만 적고 실행해도 `2025`가 보여요. 아주 편한 기능이라 머신러닝 코드에서 데이터를 확인할 때 자주 쓰게 됩니다. 다만 **마지막 줄만** 그래요. 중간 줄은 그래도 `print`를 써야 합니다.

```python
2025         # 출력 안 됨
2026         # 출력 안 됨
2027         # ← 이것만 출력됨
```

## 2. 변수: 값에 이름표 붙이기

`print("Hello, World!")` 만 평생 쓸 수는 없잖아요. 이제 진짜 프로그래밍의 첫걸음을 내딛어 봅시다.

```python
name = "철수"
print(name)
```

```
철수
```

여기서 `name = "철수"` 이 한 줄이 일어난 일은 이거예요.

> **`"철수"`라는 글자에 `name`이라는 이름표를 붙여 둔다.**

이제부터 `name`이라고 쓰면 `"철수"`를 가리키게 됩니다.

```python
name = "철수"
age = 25
height = 175.5

print(name)        # 철수
print(age)         # 25
print(height)      # 175.5
```

### 이름표는 바뀔 수 있어요

`=`은 "같다"는 뜻이 아닙니다. **"오른쪽 값에 왼쪽 이름표를 붙여라"** 라는 명령이에요.

```python
name = "철수"
print(name)        # 철수

name = "영희"      # 같은 이름표를 다른 값에 다시 붙임
print(name)        # 영희
```

이름표가 옮겨 간 거예요. 처음에는 `"철수"`를 가리키던 `name`이 이제 `"영희"`를 가리킵니다.

### 변수 이름 짓는 규칙

변수 이름은 아무거나 막 지을 수 있는 게 아닙니다. 몇 가지 규칙이 있어요.

| 규칙 | OK 예시 | 안 됨 예시 |
|------|---------|-----------|
| 영문, 숫자, 밑줄(`_`)만 | `name`, `age2`, `my_name` | `이름` (한글은 가능하지만 권장 X), `my-name` |
| 숫자로 시작 X | `name1` | `1name` |
| 파이썬 예약어는 X | `my_for` | `for`, `if`, `class` 같은 단어들 |

그리고 **아주 중요한 권장사항**이 하나 있어요.

> **이름만 봐도 무엇이 들어 있는지 짐작이 가게 지으세요.**

```python
# ❌ 안 좋은 변수 이름
a = 25
x = "철수"
b = [85, 92, 78]

# ✅ 좋은 변수 이름
age = 25
student_name = "철수"
test_scores = [85, 92, 78]
```

본인이 한 달 뒤에 다시 봤을 때, 또는 동료가 처음 봤을 때, 이름만 보고 알 수 있게요.

### 변수에 변수를 담아도 됩니다

```python
a = 10
b = a       # b에 a의 값(=10)을 담음
print(b)    # 10

a = 20      # a를 바꿔도
print(b)    # b는 그대로 10
```

이 동작이 헷갈릴 수 있어요. `b = a`는 **그 순간의 a 값을 b에 복사**한 거예요. 나중에 a가 바뀌어도 b는 그대로입니다.

(주의: 리스트나 딕셔너리는 동작이 좀 달라요. 그건 4장에서 다룹니다.)

## 3. 주석: 컴퓨터가 무시하는 글

코드를 적다 보면 "이 줄은 왜 이렇게 했지?" 같은 설명을 남기고 싶을 때가 있어요. 그럴 때 쓰는 게 **주석(comment)** 입니다.

### 한 줄 주석: `#`

`#` 뒤에 적은 모든 글자는 **컴퓨터가 무시합니다.** 사람만 읽어요.

```python
# 이건 학생 이름입니다
name = "철수"

age = 25  # 이렇게 코드 옆에 적어도 됩니다
```

### 여러 줄 주석: `'''` 또는 `"""`

따옴표 세 개로 감싸면 여러 줄을 한꺼번에 주석으로 만들 수 있어요.

```python
'''
이 부분은 학생 정보를 담는 변수들입니다.
2026년 1학기 기준으로 작성되었어요.
'''
name = "철수"
age = 25
```

### 주석을 어떻게 적어야 하나요?

여기에는 의외로 깊은 이야기가 있어요. **좋은 주석은 "코드가 무엇을 하는지"가 아니라 "왜 이렇게 했는지"를 적습니다.**

```python
# ❌ 안 좋은 주석 (코드를 읽으면 이미 아는 내용)
age = 25  # age 변수에 25를 넣음

# ✅ 좋은 주석 (왜 이 값인지 설명)
age = 25  # 우리 회사의 평균 신입 사원 나이
```

머신러닝 코드를 짜다 보면 "왜 이 학습률을 0.001로 했지?" 같은 결정이 많이 나와요. 그런 결정의 **이유**를 주석으로 적어 두면, 한 달 뒤의 본인과 동료가 정말 고마워합니다.

## 4. 들여쓰기: 파이썬의 핏줄

이제 파이썬에서 가장 중요하고 가장 헷갈리는 한 가지를 알려 드릴게요. **들여쓰기(indentation)** 입니다.

대부분의 프로그래밍 언어는 코드 묶음을 중괄호 `{ }`로 표현해요.

```javascript
// 자바스크립트
if (age > 20) {
    console.log("성인입니다");
    console.log("축하해요");
}
```

그런데 **파이썬은 중괄호가 없습니다.** 대신 **들여쓰기로 묶음을 표현**해요.

```python
# 파이썬
if age > 20:
    print("성인입니다")
    print("축하해요")
```

`if age > 20:` 다음에 **공백 4칸** 들여쓰기 한 줄들이 "이 if문에 속한 줄"이라는 뜻입니다.

### 들여쓰기 칸 수

관례상 **공백 4칸**을 씁니다. 탭(Tab)을 쓰셔도 되긴 하는데, 같은 파일 안에서는 일관되게 한쪽만 쓰셔야 해요. 섞으면 에러납니다.

좋은 소식: **거의 모든 코드 에디터가 자동으로 4칸 들여쓰기**를 해 줍니다. Colab도 마찬가지예요. `:`로 끝나는 줄 다음에 Enter를 치면 자동으로 들여써집니다.

### 들여쓰기 안 맞으면 어떻게 되나요?

```python
if age > 20:
    print("성인입니다")
   print("들여쓰기가 한 칸 모자람")    # 에러!
```

**`IndentationError`** 라는 에러가 떠요. 처음 파이썬 배우실 때 가장 자주 보는 에러 중 하나입니다. 당황하지 마세요. **`:` 다음 줄들은 모두 같은 칸수로 들여써져야 한다**는 것만 기억하세요.

지금 단계에서는 if문이나 for문을 안 다루고 있어서 들여쓰기 쓸 일이 없지만, [5장 제어문](05-제어문.md)부터 본격적으로 나옵니다. 그 전에 미리 알려 드린 거예요.

## 5. 모듈 가져오기: import

마지막으로 한 가지 더. 머신러닝 코드에서 가장 많이 보게 되실 줄 중 하나가 이거예요.

```python
import numpy as np
import pandas as pd
from sklearn.svm import SVC
```

이게 뭐냐면, **다른 사람이 만든 코드 묶음을 가져다 쓰는** 명령이에요.

### 비유: 도서관에서 책 빌리기

파이썬은 본체에 기본 기능만 있어요. 그런데 세상에는 똑똑한 사람들이 만들어 놓은 추가 기능들이 어마어마하게 많습니다. 그걸 **모듈** 또는 **라이브러리**라고 불러요. `import`는 그 라이브러리를 빌려와서 쓰겠다는 선언입니다.

```python
import math    # 수학 함수가 들어 있는 모듈을 가져와

print(math.pi)         # 3.141592653589793
print(math.sqrt(16))   # 4.0 (제곱근)
```

### import의 4가지 형태

이 자료에서 자주 보시게 될 4가지 형태를 정리해 드릴게요.

**형태 1: 통째로 가져오기**
```python
import math
print(math.pi)
```

**형태 2: 별명 붙여서 가져오기 (`as`)**
```python
import numpy as np    # numpy를 np라는 짧은 이름으로
print(np.pi)
```
머신러닝에서는 라이브러리 이름이 길어서 거의 항상 짧은 별명을 붙여 씁니다. 관례적으로 다음 별명들을 써요.

| 라이브러리 | 별명 | 무엇? |
|------|------|------|
| `numpy` | `np` | 숫자 배열 |
| `pandas` | `pd` | 표 데이터 |
| `matplotlib.pyplot` | `plt` | 그래프 그리기 |
| `seaborn` | `sns` | 더 예쁜 그래프 |
| `tensorflow` | `tf` | 딥러닝 (구글) |

**형태 3: 일부만 가져오기 (`from ... import`)**
```python
from math import pi
print(pi)    # math. 안 붙여도 됨
```

**형태 4: 별명 + 일부**
```python
from sklearn.svm import SVC as SupportVectorClassifier
```

지금은 다 외울 필요 없습니다. **"아, `import`는 다른 사람 코드 가져다 쓰는 거구나"** 만 머리에 남기시면 충분해요.

> **Q. `print` 안에 따옴표 두 개를 다 써도 되나요?** (`"..."` vs `'...'`)
>
> 네, 둘 다 됩니다. `"안녕"`도 되고 `'안녕'`도 돼요. 다만 한 번 시작하면 같은 따옴표로 끝내야 해요. `"안녕'` 같은 건 안 됩니다.
>
> 그럼 왜 두 가지가 있냐면, **다른 따옴표를 글자 안에 쓰고 싶을 때** 편해요.
> ```python
> print("그가 '안녕'이라고 말했다")
> print('그가 "안녕"이라고 말했다')
> ```

> **Q. 변수 이름에 한글 써도 되나요?**
>
> 됩니다. `이름 = "철수"` 도 동작합니다. 하지만 **권장하지 않습니다.** 라이브러리들은 다 영어로 되어 있어서, 코드의 일관성이 깨져요. 영어가 익숙하지 않으셔도 변수 이름은 영어로 적는 습관을 들이세요.

> **Q. `print(a, b)` 와 `print(a + b)` 차이가 뭐예요?**
>
> ```python
> a = "안녕"
> b = "하세요"
> print(a, b)      # 안녕 하세요  ← 가운데 띄어쓰기 자동 추가
> print(a + b)     # 안녕하세요   ← 그냥 이어 붙임
> ```

## 실전에서 보는 패턴

처음 만남 단계에서는 거창한 프로젝트보다 "작게 실행하고 바로 확인"하는 루프가 중요합니다.

1. `print`로 현재 값을 눈으로 확인합니다.
2. 변수 이름을 바꿔 가며 의미가 코드에 드러나게 만듭니다.
3. 같은 코드를 숫자/문자 입력으로 각각 실행해 차이를 확인합니다.
4. 한 줄 실행 후 결과를 예측하고 맞는지 비교합니다.

## 자주 하는 실수와 해결

- 실수: 변수 이름을 의미 없이 짧게 쓴다.
    해결: `score`, `user_name`처럼 의미가 드러나는 이름을 사용합니다.
- 실수: 출력 결과를 확인하지 않고 다음으로 넘어간다.
    해결: 각 셀/줄 실행 후 기대값과 실제값을 비교합니다.
- 실수: 오타를 실력 부족으로 받아들인다.
    해결: 오타는 정상입니다. 에러 줄 번호 확인 -> 수정 -> 재실행 루틴을 습관화합니다.

## 이해를 더 단단하게 만드는 보강 설명

이 단원은 처음 읽을 때는 "아, 이런 개념이 있구나" 정도로 넘어가기 쉽지만, 실제로는 작은 차이가 나중에 큰 성능 차이로 이어집니다. 특히 **1.1 첫 만남: print와 변수** 같은 주제는 한 번에 완벽히 이해하려고 하기보다, 다음 세 가지를 반복하면서 이해를 깊게 만드는 편이 훨씬 효과적입니다.

첫째, 용어를 정의로 외우지 말고 상황으로 연결해 보세요. 예를 들어 "왜 이 개념이 필요한가"를 스스로 설명할 수 있어야 다음 단원에서 응용이 됩니다. 둘째, 코드가 나오면 결과를 먼저 예측한 뒤 실행해 보세요. 예측과 결과가 다를 때가 진짜 학습이 일어나는 순간입니다. 셋째, 같은 코드를 숫자나 파라미터만 바꿔 여러 번 돌려 보세요. 머신러닝과 딥러닝은 정답을 암기하는 과목이 아니라 **변화를 읽는 과목**이기 때문입니다.

또 하나 중요한 점은, 이 단원의 내용이 단독으로 존재하지 않는다는 사실입니다. 실제 프로젝트에서는 데이터 준비, 모델링, 평가, 배포가 끊기지 않고 이어집니다. 그래서 지금 배우는 **1.1 첫 만남: print와 변수**도 나중에 다른 단계와 반드시 연결됩니다. 이 연결을 의식하면서 읽으면, 단편 지식이 아니라 시스템 관점으로 이해하게 됩니다.

## 자주 하는 오해와 바로잡기

### 1) "이론만 이해하면 실습은 저절로 된다"
아닙니다. 이론 이해와 코드 실행 능력은 별개의 근육입니다. 둘을 같이 써야 합니다.

### 2) "예제 코드가 돌아갔으니 완전히 이해했다"
동작 확인은 시작일 뿐입니다. 파라미터를 바꿨을 때 결과가 어떻게 달라지는지까지 확인해야 이해가 완성됩니다.

### 3) "좋은 모델은 항상 하나다"
데이터 특성과 제약(시간, 메모리, 응답 속도)에 따라 최선의 선택은 달라집니다.

### 4) "지표 숫자만 높으면 끝이다"
지표는 해석이 필요합니다. 어떤 데이터에서, 어떤 분포에서, 어떤 기준으로 얻은 점수인지 함께 보아야 합니다.

### 5) "지금은 초반이니까 배포는 나중에 생각해도 된다"
초반부터 재현성과 저장 구조를 습관화해야 나중에 배포 단계에서 무너지지 않습니다.

## 실전 연결 시나리오

아래는 이 단원 내용을 실제 업무로 연결할 때 자주 쓰는 흐름입니다.

1. 문제를 한 문장으로 정의한다.  
예: "1.1 첫 만남: print와 변수 관점에서 입력으로부터 어떤 출력을 만들 것인가?"

2. 성공 기준을 지표로 정한다.  
예: 정확도/RMSE/F1/지연시간 같은 운영 가능한 숫자로 정의.

3. 가장 단순한 베이스라인을 만든다.  
복잡한 모델보다 먼저, 빠르게 재현 가능한 기본 모델을 세웁니다.

4. 한 번에 하나씩 개선한다.  
전처리, 모델, 튜닝, 임계값, 배치 크기 등은 동시에 바꾸지 않고 순차적으로 바꿉니다.

5. 결과를 로그와 함께 저장한다.  
실험 이름, 파라미터, 점수, 데이터 버전을 함께 남겨야 재현이 됩니다.

6. 실패 케이스를 분석한다.  
점수 평균보다 오분류/고오차 샘플을 분석할 때 개선 아이디어가 나옵니다.

7. 배포 가능 형태로 정리한다.  
모델 파일, 메타데이터, 입력 스키마, 테스트 스크립트까지 묶습니다.

## 복습 체크리스트

다음 질문에 답할 수 있으면 이 단원을 실전 수준으로 이해한 것입니다.

- 이 단원의 핵심 개념을 중학생에게 설명하듯 말할 수 있는가?
- 코드의 각 블록이 왜 필요한지 한 줄씩 설명할 수 있는가?
- 이 단원에서 가장 자주 틀리는 지점을 알고 있는가?
- 지표를 볼 때 어떤 함정이 있는지 알고 있는가?
- 베이스라인과 개선 모델의 차이를 표로 정리할 수 있는가?
- 재현 가능하게 실험을 저장하는 습관이 있는가?
- 실패 사례를 보고 다음 실험 가설을 만들 수 있는가?
- 이 단원 내용을 다음 장과 어떻게 연결할지 설명할 수 있는가?

## 확장 연습 과제

### 과제 A: 파라미터 감각 만들기
현재 예제에서 핵심 파라미터 1~2개만 바꿔 5회 실험하고, 결과를 표로 정리해 보세요.

### 과제 B: 실패 사례 분석
오분류 또는 오차가 큰 샘플 20개를 모아 공통 패턴을 찾아 보세요.

### 과제 C: 설명 가능 리포트 작성
"무엇을 했고, 왜 했고, 결과가 어땠고, 다음엔 무엇을 할지"를 1페이지로 정리해 보세요.

이 3개를 직접 해 보면 **1.1 첫 만남: print와 변수**의 이해도는 읽기만 했을 때보다 훨씬 빠르게 올라갑니다.

## 한 번 더 정리

1.1 첫 만남: print와 변수는 단순히 시험용 개념이 아니라, 실제 문제를 풀 때 반복해서 쓰게 되는 실전 도구입니다. 처음에는 낯설 수 있지만, 작은 실험을 반복하면 감각이 생기고, 감각이 생기면 코드와 지표를 읽는 속도가 급격히 빨라집니다. 지금 단계에서 중요한 것은 완벽한 암기가 아니라, **작게 실행하고, 결과를 읽고, 다시 수정하는 루프**를 몸에 익히는 것입니다. 이 루프를 익히면 다음 장으로 넘어갈수록 학습 속도가 오히려 빨라집니다.
