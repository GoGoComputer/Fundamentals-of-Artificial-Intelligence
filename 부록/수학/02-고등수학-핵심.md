# 2. 고등 수학 핵심

> 고등학교 수학 전체를 다루지 않습니다. ML에 진짜 필요한 4가지만 골랐어요.
> 지수, 로그, 시그마(Σ), 그리고 자연상수 e.

---

## 2.1 지수 (Exponent)

`2³`이라고 쓰면 "2를 3번 곱하기" 라는 뜻이에요.

```
2³ = 2 × 2 × 2 = 8
2⁵ = 2 × 2 × 2 × 2 × 2 = 32
10⁴ = 10 × 10 × 10 × 10 = 10,000
```

`2`는 **밑(base)**, `3`은 **지수(exponent)** 라고 해요.

### 지수의 법칙

ML에 자주 쓰이는 두 가지만 외우세요.

#### 법칙 1: 같은 밑끼리 곱하면 지수 더하기

```
2³ × 2² = 2³⁺² = 2⁵
```

#### 법칙 2: 같은 밑끼리 나누면 지수 빼기

```
2⁵ / 2² = 2⁵⁻² = 2³
```

### 지수가 음수면?

```
2⁻¹ = 1/2 = 0.5
2⁻² = 1/4 = 0.25
2⁻³ = 1/8 = 0.125
```

음수 지수는 **역수의 양수 지수**예요. 즉 `2⁻ⁿ = 1/2ⁿ`.

### 지수가 0이면?

```
2⁰ = 1
5⁰ = 1
1000⁰ = 1
```

**0이 아닌 어떤 수든 0제곱은 1이에요.** 약속이에요. (왜 그런지는 검색해 보세요.)

### 지수가 분수면?

```
2^(1/2) = √2 ≈ 1.414      (제곱근)
8^(1/3) = ∛8 = 2          (세제곱근)
16^(1/4) = ⁴√16 = 2       (네제곱근)
```

`x^(1/n) = ⁿ√x` 입니다.

### ML에서 어디?

- **지수 함수 활성화** (`exp(x)`): softmax의 핵심 부품
- **표준편차 계산**: 분산의 1/2제곱
- **거리 계산**: 차이의 제곱
- **학습률 감쇠**: `lr × 0.95^epoch` 같은 패턴

## 2.2 로그 (Logarithm)

지수의 **반대** 연산이에요. "어떤 수를 몇 제곱하면 이 값이 될까?" 를 묻는 거예요.

```
2³ = 8 이라면, log₂ 8 = 3
                ↑
              "2를 몇 제곱하면 8?" → 3

10⁴ = 10000 이라면, log₁₀ 10000 = 4
                                  ↑
                              "10을 몇 제곱하면 10000?" → 4
```

기호: `log_밑(값)`. 한국에서는 `log₂(8) = 3` 처럼 써요.

### 자주 쓰는 두 가지 로그

#### 1. 상용로그 (밑 10)

`log₁₀ x` 를 그냥 `log x` 로 적어요.

```
log 100 = 2     (10² = 100)
log 1000 = 3    (10³ = 1000)
log 1 = 0       (10⁰ = 1)
log 0.1 = -1    (10⁻¹ = 0.1)
```

#### 2. 자연로그 (밑 e ≈ 2.718)

`log_e x` 를 `ln x` 또는 그냥 `log x`로 (수학에선 ln, ML에선 log로 자주 씀).

```
ln e = 1
ln 1 = 0
ln(e²) = 2
```

### 로그의 법칙

#### 법칙 1: 곱은 합으로

```
log(a × b) = log a + log b
```

예: `log(2×3) = log 2 + log 3`

#### 법칙 2: 나눗셈은 차이로

```
log(a / b) = log a - log b
```

#### 법칙 3: 지수는 앞으로

```
log(aⁿ) = n × log a
```

예: `log(2⁵) = 5 × log 2`

### ML에서 왜 로그가 중요?

#### 1. Cross-Entropy 손실에 들어 있음

```
CE = -Σ yᵢ × log(pᵢ)
                 ↑
              log!
```

분류 모델의 표준 손실. 아래에서 직관 봅시다.

#### 2. 매우 작은 확률을 다루기 좋음

확률이 0.0000001 같은 작은 값이면 컴퓨터가 0과 헷갈려요. 로그를 씌우면 다루기 쉬운 값이 돼요.

```
0.0000001 → log(0.0000001) = -7
```

#### 3. 곱셈을 덧셈으로

여러 확률의 곱을 다룰 때, 로그를 씌우면 합으로 변환돼요. (법칙 1)

```
P(A) × P(B) × P(C) × ... 
→ log P(A) + log P(B) + log P(C) + ...
```

훨씬 다루기 쉬워요.

## 2.3 시그마 (Σ) — 합 기호

ML 수식에 **가장 자주** 나오는 기호예요. 그냥 **"다 더하라"** 는 뜻이에요.

```
∑ᵢ₌₁ⁿ aᵢ = a₁ + a₂ + a₃ + ... + aₙ
```

### 읽는 법

```
   n
   Σ aᵢ
  i=1
```

- **i=1** 부터 시작
- **n** 까지
- **aᵢ** 를 다 더하기

### 예시

```
∑ᵢ₌₁⁵ i = 1 + 2 + 3 + 4 + 5 = 15

∑ᵢ₌₁³ i² = 1² + 2² + 3² = 1 + 4 + 9 = 14

∑ᵢ₌₁⁴ 2i = 2 + 4 + 6 + 8 = 20
```

### 코드로 쓰면

```python
# 시그마 = for문의 합
total = 0
for i in range(1, 5):
    total += i
# = 1 + 2 + 3 + 4 = 10

# 또는
sum(range(1, 5))    # 더 간단
```

### 인덱스 변경

```
∑ⱼ₌₁ⁿ xⱼ
```

`i` 대신 `j` 써도 같은 뜻이에요. 그냥 변수 이름이 바뀐 것뿐.

### 두 개 시그마 (이중 합)

```
∑ᵢ ∑ⱼ aᵢⱼ = (모든 i, j 조합의 aᵢⱼ를 다 더함)

예시: 2x2 행렬
A = [[1, 2],
     [3, 4]]

∑ᵢ ∑ⱼ Aᵢⱼ = 1 + 2 + 3 + 4 = 10
```

### ML에서

거의 모든 손실 함수에 시그마가 들어 있어요.

```
MSE = (1/n) Σᵢ (yᵢ - ŷᵢ)²
            ↑
          시그마 = 모든 샘플에 대해 더함
          
CE = -Σᵢ yᵢ × log(pᵢ)
      ↑
    시그마 = 모든 클래스에 대해 더함
```

읽으면: **"평균 제곱 오차 = 1/샘플수 × (모든 샘플의 차이의 제곱 합)."**

이제 이 식이 외계어가 아니라 한국어로 읽히기 시작했죠?

## 2.4 자연상수 e

`e ≈ 2.71828...` 이라는 무리수예요. **수학에서 가장 중요한 상수 중 하나**.

### 왜 e가 특별해요?

`y = eˣ` 라는 함수의 기울기가 **자기 자신과 같아요.**

즉, `y = eˣ` 면 어떤 점에서든 `미분 = eˣ` 그대로. 이게 **이 세상에서 유일한 함수의 성질**이에요.

이 성질 덕에 미분 계산이 매우 단순해져요. 그래서 ML에서도 **자연 지수와 자연로그가 표준**입니다.

### exp(x) 표기

`eˣ`를 `exp(x)` 라고 쓰기도 해요. 더 컴퓨터 친화적이에요.

```python
import math
math.exp(1)    # = e ≈ 2.718
math.exp(0)    # = 1
math.exp(2)    # ≈ 7.389
```

### ML에서 e가 어디?

#### 1. Sigmoid 함수

```
σ(x) = 1 / (1 + e⁻ˣ)
```

이진 분류의 표준 활성화 함수. 출력이 0~1.

#### 2. Softmax 함수

```
softmax(zᵢ) = e^zᵢ / Σⱼ e^zⱼ
```

다중 분류의 표준 출력. 합이 1인 확률들.

#### 3. 학습률 감쇠

```
lr(t) = lr₀ × e^(-α × t)
```

지수적 감쇠 스케줄러.

#### 4. Gaussian (정규분포)

```
p(x) = (1/√(2πσ²)) × e^(-(x-μ)²/(2σ²))
                       ↑
                       e!
```

5장에서 다룹니다.

## 2.5 함수의 합성과 역함수

### 합성 함수

함수를 **연속으로** 적용하는 거예요.

```
f(x) = 2x + 1
g(x) = x²

g(f(x)) = g(2x + 1) = (2x + 1)²
```

`g(f(x))`는 "x를 f에 넣고, 그 결과를 g에 넣어라" 는 뜻.

### ML에서?

신경망이 사실 **합성 함수의 연속 적용**이에요.

```
신경망 = activation(W₃ × activation(W₂ × activation(W₁ × x + b₁) + b₂) + b₃)
       └─ 층 1 ─┘
                  └────── 층 2 ──────┘
                                       └─────── 층 3 ────────┘
```

함수 안에 함수 안에 함수... 이렇게 깊게 합성된 거예요. 그래서 **체인룰**(다음 장)이 필요해요.

### 역함수

`f`가 입력 → 출력이라면, **역함수 f⁻¹** 은 출력 → 입력.

```
f(x) = 2x + 3
f(5) = 13

f⁻¹(13) = 5
```

`f⁻¹(y) = (y - 3) / 2`

### ML에서?

- **로그 = 지수의 역함수** (이걸 활용해 cross-entropy를 계산)
- **softmax → log-softmax** (로그 적용해 안정화)
- **inverse_transform** (정규화 후 원래 단위로 되돌리기)

## 2.6 부등식 (다시) — sigmoid 같은 함수의 의미

기본 부등식은 1장에서 봤죠. 여기서는 ML에 나오는 활용을 짧게.

### 0 ≤ p ≤ 1 의 의미

확률은 항상 이 범위에 있어요. Sigmoid 함수가 정확히 이걸 만들어 줘요.

```
sigmoid(아무 실수 x) → 0과 1 사이의 값

x = -100  → 0.000...
x = -1    → 0.27
x = 0     → 0.5
x = 1     → 0.73
x = 100   → 0.999...
```

큰 음수는 0에 가깝게, 큰 양수는 1에 가깝게. 0이 정확히 0.5.

이 성질 때문에 sigmoid는 **이진 분류의 출력**에 표준이에요.

## 2.7 정리

```python
# 지수 - 같은 수의 반복 곱셈
2 ** 3 == 8

# 로그 - 지수의 역
import math
math.log2(8) == 3
math.log(math.e) == 1   # 자연로그

# 시그마 - 다 더하기
sum([i**2 for i in range(1, 4)]) == 14   # 1 + 4 + 9

# 자연상수 e
math.exp(0) == 1
math.exp(1) == math.e
```

### ML 식 다시 읽기

이제 이런 식들이 한국어로 읽혀야 해요.

```
softmax(zᵢ) = e^zᵢ / Σⱼ e^zⱼ
```
"i번째 출력의 소프트맥스 = e의 zᵢ제곱 나누기 (모든 j에 대한 e의 zⱼ제곱의 합)."

```
CE = -(1/n) Σᵢ yᵢ × log(pᵢ)
```
"교차엔트로피 = 마이너스 1/n 곱하기 (모든 i에 대한 (실제값 i × 예측 확률 i의 로그)의 합)."
