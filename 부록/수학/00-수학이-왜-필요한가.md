# 0. 수학이 왜 필요한가

## "ML 코드만 짤 줄 알면 안 되나요?"

당연히 됩니다. 솔직히 처음 6개월~1년은 수학 거의 몰라도 충분해요. `model.fit()` 부르면 모델이 학습되니까요.

근데 어느 시점에 막혀요. 다음 같은 상황에서요.

- "왜 학습률을 낮춰야 한다는 거야?"
- "왜 가중치를 정규화하면 과적합이 줄어드는 거야?"
- "왜 cross-entropy를 쓰는 거야? 그냥 차이의 절댓값 합치면 안 돼?"
- "역전파가 정확히 뭔데?"

이 모든 질문의 답에는 **수학**이 들어 있어요. 수학을 모르면 영원히 "남이 한 대로만 따라가는" 상태에 머물러요. 그게 답답해지는 순간이 옵니다.

이 자료가 있는 이유예요. **수학이 무서워서 ML을 포기하지 않으셨으면 좋겠어요.**

---

## ML/DL에 진짜로 쓰이는 수학 다섯 갈래

수학은 어마어마하게 큰 분야예요. 그 중 ML/DL이 진짜로 쓰는 건 다음 5가지밖에 안 돼요.

### 1. 함수 (Function)

> "입력을 받아서 출력을 내놓는 무언가."

ML 모델 자체가 거대한 함수예요. `model(x) = y`. 이게 함수가 아니면 뭐겠어요.

→ [1장](01-중학수학-복습.md)에서 함수의 기초

### 2. 미분 (Differentiation)

> "함수가 어디로 변하는지 (기울기) 알기."

신경망 학습은 **손실 함수의 기울기를 따라 내려가는** 과정이에요. 미분 모르면 학습 원리를 못 봐요.

→ [3장](03-미분-입문.md)에서 미분과 편미분, 체인룰

### 3. 선형대수 (Linear Algebra)

> "여러 숫자를 한꺼번에 다루는 도구."

신경망의 모든 연산이 **벡터와 행렬의 곱**으로 표현돼요. 입력 → 곱 → 더하기 → 활성화 → 출력. 이 "곱하기"가 선형대수예요.

→ [4장](04-선형대수-입문.md)에서 벡터, 행렬, 내적

### 4. 확률 / 통계 (Probability / Statistics)

> "불확실한 것을 다루기."

분류 모델의 출력은 **확률**이에요. 학습 데이터의 분포, 손실의 기댓값 — 다 확률이에요.

→ [5장](05-확률통계-입문.md)에서 평균/분산, 정규분포, 베이즈

### 5. 최적화 (Optimization)

> "가장 좋은 답을 찾기."

학습이라는 게 결국 **손실을 최소화하는 가중치를 찾는 것**이에요. 그게 최적화.

→ [6장](06-ML에-적용하기.md)에서 경사하강법, 손실함수

## 본문에 나오는 수식들 미리 보기

본문에서 만났던 (또는 만나게 될) 수식들을 미리 봅시다. 무서워 보이지만, 이 자료를 다 보시면 다 읽을 수 있게 돼요.

### 손실 함수 (Loss Function)

```
MSE = (1/n) × Σᵢ (y_true,ᵢ - y_pred,ᵢ)²
```

읽으면: **"평균 제곱 오차 = (1/샘플 수) × 시그마 (실제값 - 예측값)의 제곱."**

→ Σ (시그마)는 [2장](02-고등수학-핵심.md), MSE는 [6장](06-ML에-적용하기.md)에서.

### 경사하강법

```
w_new = w_old - η × ∂L/∂w
```

읽으면: **"새 가중치 = 기존 가중치 - 학습률 × 손실의 가중치에 대한 편미분."**

→ ∂ (편미분)는 [3장](03-미분-입문.md), 경사하강법은 [6장](06-ML에-적용하기.md).

### Softmax

```
softmax(zᵢ) = e^zᵢ / Σⱼ e^zⱼ
```

읽으면: **"입력 z의 i번째 값에 대한 소프트맥스 = e의 zᵢ제곱 / 모든 j에 대한 e의 zⱼ제곱의 합."**

→ e (자연상수)는 [2장](02-고등수학-핵심.md), softmax는 [6장](06-ML에-적용하기.md).

### Cross-Entropy

```
CE = -Σᵢ yᵢ × log(pᵢ)
```

읽으면: **"교차 엔트로피 = -(시그마) (실제 라벨) × log(예측 확률)."**

→ log는 [2장](02-고등수학-핵심.md), CE는 [6장](06-ML에-적용하기.md).

### 신경망 한 층

```
y = σ(Wx + b)
```

읽으면: **"출력 = 활성화함수( 가중치 행렬 × 입력 벡터 + 편향 )."**

→ Wx는 행렬 곱이라 [4장](04-선형대수-입문.md), σ는 [6장](06-ML에-적용하기.md).

## 수식을 읽는 법 — 첫 팁

수식을 두려워하지 않게 되는 가장 큰 팁: **수식은 영어 문장처럼 왼쪽에서 오른쪽으로 읽는다.**

```
y = w × x + b
```

이걸 읽으면: **"y는 w와 x의 곱에 b를 더한 것."**

```
∂L/∂w
```

이걸 읽으면: **"L의 w에 대한 편미분."**

수식의 각 기호는 **단어 하나**에 해당해요. 단어를 모르면 읽을 수 없을 뿐이죠. 단어를 배우러 가는 게 이 자료입니다.

### 수학 기호 미리 보기 (사전처럼 쓰세요)

| 기호 | 이름 | 의미 |
|------|------|------|
| `Σ` | 시그마 | 다 더해라 |
| `Π` | 파이 (대문자) | 다 곱해라 |
| `∂` | 라운드 d | 편미분 |
| `∇` | 나블라 | 그래디언트 (편미분 모음) |
| `∫` | 인테그랄 | 적분 |
| `∞` | 인피니티 | 무한대 |
| `∈` | 인 | ~의 원소 |
| `→` | 어로우 | ~로 가다, ~에 매핑되다 |
| `e` | 자연상수 | ≈ 2.718... |
| `π` | 파이 | ≈ 3.141... |
| `λ` | 람다 | 정규화 강도 등에 자주 씀 |
| `η` | 에타 | 학습률에 자주 씀 |
| `μ` | 뮤 | 평균에 자주 씀 |
| `σ` | 시그마 (소문자) | 표준편차 / 활성화함수 |

이 기호들이 무서웠던 분, 곧 친해지실 거예요.

## 한 가지 명심할 것

**수학을 잘 한다고 ML을 잘 하는 건 아니에요.** 거꾸로도 마찬가지고요.

수학자가 ML 엔지니어가 되려면 코드와 데이터를 배워야 하고, ML 엔지니어가 깊이 가려면 수학을 배워야 해요. 이 둘은 다른 능력이에요. **양쪽을 적절히 알면 되지, 한쪽만 잘하면 됩니다.**

이 자료의 목표는 **양쪽을 적절히** 만드는 것. 수학 박사 만들려는 게 아닙니다.
