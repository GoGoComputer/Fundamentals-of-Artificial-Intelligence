# 3. 미분 입문

> ML/DL 학습의 핵심 수학입니다. 신경망이 학습되는 모든 순간 미분이 일어나요.
> 진짜 깊이는 안 들어갑니다. **직관과 결과**만 이해하면 ML에는 충분해요.

---

## 3.1 기울기 (Slope) — 미분의 직관

### 1차 함수의 기울기

`y = 2x + 3` 의 기울기는 2였죠. ([1장](01-중학수학-복습.md))

기울기의 진짜 의미: **"x가 1 늘 때 y가 얼마나 늘어나는가."**

```
x = 0 → y = 3
x = 1 → y = 5  (y가 2 증가)
x = 2 → y = 7  (또 y가 2 증가)
x = 3 → y = 9  (또 2)

→ x가 1 늘 때마다 y는 항상 2 증가 → 기울기 = 2
```

직선이라 어디서나 기울기가 똑같아요.

### 곡선은?

`y = x²` 같은 곡선은 어떨까요?

```
x = 1 → y = 1
x = 2 → y = 4   (y가 3 증가)
x = 3 → y = 9   (y가 5 증가)
x = 4 → y = 16  (y가 7 증가)
```

x가 1 늘었을 때 y의 증가량이 다르죠. **곡선은 위치에 따라 기울기가 달라요.**

### 그럼 곡선의 "한 점에서의 기울기"는?

직관: **그 점에 살짝 닿는 직선**(접선)의 기울기.

```
y = x² 의 그래프와 x=2 에서의 접선

       y
       ↑
    9  ┤        ●
    7  ┤       /
    5  ┤   ────╳────  ← 접선 (x=2에서)
    3  ┤      /
    1  ┤    ╱
       └────────────→ x
           0  1  2  3
```

이 접선의 기울기를 그 점에서의 **순간 변화율** 또는 **미분값** 이라고 해요.

## 3.2 미분 (Derivative)

`y = f(x)` 라는 함수의 미분을 다음과 같이 써요.

```
f'(x)        ← 프라임 표기
df/dx        ← 라이프니츠 표기
dy/dx        ← 같은 의미
```

`f'(x)` 자체가 또 다른 함수예요. 각 x에 대한 기울기를 알려주는 함수.

### 가장 기본 — 다항식의 미분 공식

이 두 가지만 외우면 ML에는 거의 충분해요.

#### 공식 1: 거듭제곱

```
f(x) = xⁿ  →  f'(x) = n × xⁿ⁻¹
```

예시:
- `f(x) = x²` → `f'(x) = 2x¹ = 2x`
- `f(x) = x³` → `f'(x) = 3x²`
- `f(x) = x⁵` → `f'(x) = 5x⁴`
- `f(x) = x` → `f'(x) = 1` (=`1×x⁰ = 1`)
- `f(x) = 5` (상수) → `f'(x) = 0`

#### 공식 2: 합과 상수

```
[f(x) + g(x)]' = f'(x) + g'(x)
[c × f(x)]'    = c × f'(x)        (c는 상수)
```

예시:
- `f(x) = 3x² + 2x + 5` 의 미분?
  - `(3x²)' = 3 × 2x = 6x`
  - `(2x)' = 2`
  - `(5)' = 0`
  - 합쳐서 `f'(x) = 6x + 2`

### 실제 값 구하기

`f(x) = x²` 의 `x = 3` 에서의 기울기?

`f'(x) = 2x`
`f'(3) = 2 × 3 = 6`

→ x=3 에서 곡선의 기울기는 6.

### ML에서 자주 쓰는 미분 공식 몇 개 더

| 함수 | 미분 |
|------|------|
| `x` | `1` |
| `x²` | `2x` |
| `xⁿ` | `n × xⁿ⁻¹` |
| `eˣ` | `eˣ` (자기 자신!) |
| `log x` (자연로그) | `1/x` |
| `c` (상수) | `0` |

이 정도만 외우셔도 ML 코드 99%를 이해할 수 있어요.

## 3.3 왜 ML에 미분이 필요한가

여기가 핵심이에요.

### 학습 = "손실을 최소화하는 가중치 찾기"

```
손실 L(w)
│
│ \                    /
│  \                  /
│   \                /
│    \              /
│     \____________/
│           ↑
│      여기를 찾고 싶음
└──────────────────────→ w
```

이 골짜기의 가장 낮은 점을 찾는 게 학습이에요.

### 어떻게 찾을까? — 기울기를 보고 내려가기

[1.5 산에서 내려오기 비유](../../04-딥러닝-분류/이론/04a-경사하강법-깊게.md)에서 봤죠.

> 안개 낀 산에서 발 디딘 곳의 기울기를 보고, 더 낮은 쪽으로 한 발자국. 그걸 반복.

기울기가 양수면 → 왼쪽으로 (더 낮은 쪽)
기울기가 음수면 → 오른쪽으로 (더 낮은 쪽)

```
              기울기 +
                ↓
                /
               /
              /
   ──────────╳──────  ← 어떤 점에서
            /
           /
          /
       기울기 - (반대편)
```

그래서 학습은 **"기울기의 반대 방향으로 한 걸음 가기"** 를 반복합니다.

### 경사하강법 식

```
w_new = w_old - 학습률 × 기울기
                          ↑
                    f'(w) = ∂L/∂w
```

이게 신경망 학습의 핵심 식이에요. **모든 신경망이 이 식을 매 step마다 실행**합니다.

`-` 부호인 이유: 기울기 반대 방향으로 가야 하니까.

### 예시로 보기

`L(w) = (w - 3)²` 라는 손실 함수가 있다고 해 봐요. 최솟값은 `w = 3` 일 때 `L = 0`.

미분: `L'(w) = 2(w - 3)`

학습률 = 0.1

처음에 `w = 0` 에서 시작:

```
step 1: w = 0
        L'(0) = 2(0-3) = -6
        w_new = 0 - 0.1×(-6) = 0.6

step 2: w = 0.6
        L'(0.6) = 2(0.6-3) = -4.8
        w_new = 0.6 - 0.1×(-4.8) = 1.08

step 3: w = 1.08
        L'(1.08) = 2(1.08-3) = -3.84
        w_new = 1.08 + 0.384 = 1.464

step 4: ...
...
```

`w`가 점점 3에 가까워져요. 기울기가 줄어들수록 한 발자국도 작아지면서. 이게 **수렴**이에요.

이걸 코드로 직접 해볼 수도 있어요.

```python
w = 0.0
lr = 0.1

for step in range(50):
    grad = 2 * (w - 3)
    w = w - lr * grad
    
    if step % 5 == 0:
        loss = (w - 3) ** 2
        print(f"step {step}: w={w:.4f}, loss={loss:.4f}")
```

```
step 0: w=0.6000, loss=5.7600
step 5: w=2.0144, loss=0.9712
step 10: w=2.6776, loss=0.1040
step 15: w=2.9000, loss=0.0100
...
```

진짜로 `w → 3` 으로 수렴해요. 미분의 마법이에요.

## 3.4 편미분 (Partial Derivative)

### 입력이 여러 개면?

지금까지는 `f(x)` — 입력 하나. 그런데 ML에선 **수백만 개의 가중치**가 있죠.

`L(w₁, w₂, w₃, ..., wₙ)` 같은 함수예요.

이런 다변수 함수에서 **하나의 변수에 대한 기울기**를 보는 게 **편미분**이에요.

### 표기법

```
∂L/∂wᵢ
```

`∂` (라운드 d) 는 "편미분이다" 라는 표시. 일반 미분 `d` 와 구분하려고 모양이 다른 거예요.

읽으면: **"L의 wᵢ에 대한 편미분."**

### 직관

다른 변수들은 **고정**해 두고, 한 변수만 움직였을 때 함수가 어떻게 변하는지.

예시: `L(w₁, w₂) = w₁² + 3w₂² + w₁w₂`

`w₂` 를 상수처럼 취급하고 `w₁` 에 대해 미분:
```
∂L/∂w₁ = 2w₁ + w₂
```
(`3w₂²` 는 w₁ 입장에서 상수라 미분하면 0, `w₁w₂` 는 w₂를 상수처럼 보고 `w₂`)

`w₁` 을 상수처럼 취급하고 `w₂` 에 대해 미분:
```
∂L/∂w₂ = 6w₂ + w₁
```

### ML에서

신경망의 모든 가중치 각각에 대해 편미분을 계산해요. 100만 개 가중치면 100만 개의 편미분.

다 모은 걸 **그래디언트(gradient)** 라고 부르고, 기호로는 `∇L` (나블라 L).

```
∇L = (∂L/∂w₁, ∂L/∂w₂, ..., ∂L/∂wₙ)
```

### PyTorch가 해주는 것

직접 손으로 100만 개 편미분을 계산하면 평생 못 끝내요. 다행히 PyTorch가 자동으로 해줘요.

```python
loss.backward()    # 모든 가중치의 편미분 자동 계산!

# 이게 뭐 하는 거냐면, 모든 파라미터에 .grad 속성이 채워져요
for p in model.parameters():
    print(p.grad)    # ← 편미분 값
```

이게 **autograd (자동 미분)** 의 정체예요. ML 엔지니어가 미분을 직접 안 해도 되는 이유.

## 3.5 체인룰 (Chain Rule)

신경망에서 가장 중요한 한 가지. **합성 함수의 미분.**

### 1변수 체인룰

```
y = f(g(x))   (합성 함수)

dy/dx = f'(g(x)) × g'(x)
```

말로: **"바깥 함수의 미분 × 안쪽 함수의 미분."**

### 예시

`y = (x² + 1)³` 의 미분?

이걸 직접 펼쳐 미분하면 어려워요. 체인룰로 풀면:

```
g(x) = x² + 1
f(u) = u³

y = f(g(x))

g'(x) = 2x
f'(u) = 3u²

dy/dx = f'(g(x)) × g'(x)
      = 3(x² + 1)² × 2x
      = 6x(x² + 1)²
```

체인룰이 없으면 사실상 ML 학습은 불가능했어요.

### 신경망에서

신경망은 합성 함수의 합성 함수의 합성 함수... ([2장](02-고등수학-핵심.md))

```
출력 = activation(W₃ × activation(W₂ × activation(W₁ × x + b₁) + b₂) + b₃)
       └─ 마지막 ─┘                                              
                    └────── 두 번째 ────────┘                    
                                              └─── 첫 번째 ───┘
```

손실 `L`을 가중치 `W₁` 에 대해 편미분하려면? **체인룰을 깊은 층 수만큼 적용**해야 해요.

```
∂L/∂W₁ = ∂L/∂y × ∂y/∂h₃ × ∂h₃/∂h₂ × ∂h₂/∂h₁ × ∂h₁/∂W₁
         └────── 출력층 ──────┘  └── 중간층들 ──┘  └ 첫 층 ┘
```

이걸 **역전파(Backpropagation)** 라고 해요. 체인룰의 ML 버전이에요.

### 다행히도

다시 말하지만, **PyTorch의 `.backward()`가 다 해줍니다.** 우리가 손으로 계산할 필요 없어요.

```python
loss.backward()
# 위의 어마어마한 체인룰 계산이 한 번에 끝남
```

체인룰의 직관만 이해하시면 충분해요. **"각 층의 미분을 곱해 가며 거꾸로 전파"** 라는 그림.

## 3.6 그래디언트 (Gradient) — 한 단어로

지금까지 배운 걸 한 단어로 정리하면 **그래디언트**예요.

> **그래디언트** = 함수의 기울기 (편미분의 모음).
> 가장 빠르게 함수가 증가하는 방향을 가리킴.

### 표기

```
∇L  (나블라 L)

또는 명시적으로:
∇L = (∂L/∂w₁, ∂L/∂w₂, ..., ∂L/∂wₙ)
```

### 학습 식 다시 보기

```
w_new = w_old - η × ∇L
                ↑     ↑
           학습률   그래디언트
```

이제 이게 다 읽혀요.
- `η` (에타) = 학습률
- `∇L` = 손실의 그래디언트
- 그래디언트의 **반대 방향**으로 학습률만큼 가기 → 손실 줄임

이게 **모든 신경망 학습의 본질**입니다. 다른 모든 것 (Adam, Momentum 등)은 이 식의 변형이에요.

## 3.7 정리

```python
# 미분 — 함수의 기울기
# d(x²)/dx = 2x
# d(x³)/dx = 3x²
# d(eˣ)/dx = eˣ
# d(log x)/dx = 1/x

# 편미분 — 한 변수에 대해서만
# ∂(w₁² + w₂²)/∂w₁ = 2w₁

# 체인룰 — 합성 함수의 미분
# d(f(g(x)))/dx = f'(g(x)) × g'(x)

# 그래디언트 — 모든 편미분의 모음
# ∇L = (∂L/∂w₁, ∂L/∂w₂, ...)

# 경사하강법 — 학습의 본질
# w_new = w_old - lr × ∇L

# PyTorch가 다 해줌
loss.backward()
```

### ML 식 다시 읽기

```
w ← w - η × ∂L/∂w
```

읽으면: **"새로운 w는 기존 w에서 (학습률 × L의 w에 대한 편미분)을 뺀 값."**

이제 외계어 아니죠?

1. `f(x) = 3x² + 2x` 일 때, `f'(x) = ?`
2. `f'(2) = ?` (위 함수)
3. `f(x, y) = x²y + y³` 일 때, `∂f/∂x = ?` 그리고 `∂f/∂y = ?`
4. `y = (x² + 1)²` 의 미분은? (체인룰)
5. 손실이 음수 기울기인 곳에 있을 때, 가중치를 어느 방향으로 움직여야 손실이 줄까요?

<details>
<summary>정답</summary>

2. `f'(2) = 12 + 2 = 14`
3. `∂f/∂x = 2xy`, `∂f/∂y = x² + 3y²`
4. `2(x² + 1) × 2x = 4x(x² + 1)`
5. **양의 방향**(오른쪽). 기울기가 음수라는 건 함수가 감소 중이라는 뜻이고, 그 방향으로 가야 더 낮아지니까. 식으로는 `w - lr × (-)` = `w + 양수`.
