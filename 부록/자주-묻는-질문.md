# 자주 묻는 질문 (FAQ)

학생들이 가장 자주 물어보는 질문들과 솔직한 답변입니다.

## 시작하기 전

### Q. 수학을 못해도 되나요?

**솔직히:** 없어도 시작은 됩니다. 깊이 가시려면 도움이 됩니다.

이 자료는 수식보다 직관과 코드 위주예요. 처음 시작하실 땐 수학 없이도 충분히 배우실 수 있어요.

다만 **언젠가는** 다음 정도는 알면 좋아요:
- **선형대수**: 행렬 곱, 벡터, 차원
- **미분**: 기울기 = 미분 (편미분, 체인룰)
- **확률**: 평균, 분산, 정규분포

이건 ML 시작 후에 필요할 때 천천히 배우셔도 늦지 않아요.

### Q. 어느 언어로 배워야 해요? Python? R?

**Python.** 다른 옵션 거의 없어요. 2026년 기준 ML의 사실상 표준이에요.

R은 통계학과 데이터 분석에 강하지만, 산업에서는 Python이 압도적이에요.

### Q. 학력이 부족해도 ML 엔지니어가 될 수 있나요?

**됩니다.** 학력보다 포트폴리오가 훨씬 중요해요.

GitHub에 본인이 만든 ML 프로젝트 3~5개 올려두시면, 학력보다 강력한 무기가 됩니다. 캐글(Kaggle)에서 메달 받으시는 것도 좋아요.

### Q. 나이가 많은데 늦지 않았나요?

**전혀요.** ML은 새로운 분야라 평균 연령이 다른 분야보다 낮긴 해요. 하지만 30대, 40대에 ML로 전향하신 분 정말 많아요. 본인의 도메인 지식과 결합하면 오히려 강점이 됩니다.

## 배우는 동안

### Q. 진도가 너무 안 나가요. 정상인가요?

**정상입니다.** 진짜 정상이에요.

ML은 익숙해지는 데 시간이 걸려요. 새 개념이 머리에 들어오기 전에 다른 새 개념이 들어오는 것 같죠. 그래도 한 달, 두 달 지나면 어느새 익숙해져 있어요.

**팁:**
- 한 번에 다 이해하려 하지 마세요
- 막히면 일단 넘기고, 며칠 후 다시 보세요
- 손으로 코드를 쳐보세요 (복사 붙여넣기 금지)

### Q. 코드를 따라 쳤는데 에러가 나요. 어떡하죠?

**기뻐하세요.** 에러는 배움의 기회예요.

[질문하는 법](../00-시작하기-전에/04-질문하는-법.md)을 다시 읽어 보세요. 4단계 (에러 메시지 읽기 → 검색 → AI에게 묻기 → 사람에게 묻기)를 차근차근 거쳐 보세요.

### Q. 책이나 강의를 더 봐야 할까요?

**일단 이 자료를 끝까지 보세요.** 그리고 본인 프로젝트 1~2개를 해 보세요. 그 후에 다른 자료를 찾으세요.

처음에 자료 여러 개를 동시에 보면 헷갈려요. 하나를 끝까지 봐서 큰 그림을 잡고, 그다음 깊게 들어가시는 게 좋아요.

### Q. 노트북 환경 뭐가 좋아요? Jupyter? Colab? VSCode?

- **배우는 단계: Colab** (설치 필요 없고 GPU 무료)
- **본격적인 프로젝트: VSCode + Jupyter** 확장
- **공유: Jupyter Notebook (.ipynb)**

회사에서는 보통 VSCode 또는 PyCharm + Jupyter 확장을 씁니다.

### Q. ChatGPT나 Claude 같은 AI를 학습에 써도 돼요?

**적극 쓰세요.** 단, 다음만 주의:

- ✅ 코드 설명 요청
- ✅ 에러 디버깅 도움
- ✅ 개념 설명
- ❌ 답을 그냥 복사 붙여넣기 (배움 X)
- ❌ 본인이 이해 못 한 코드를 그대로 쓰기

AI는 정말 좋은 학습 도구예요. 하지만 **본인이 이해할 때까지 묻고 또 물으세요.**

## 모델과 데이터

### Q. 데이터가 너무 적어요. 어떡하죠?

**상황에 따라:**

- 100개 미만 → 너무 적어요. 더 모으세요.
- 100~1000개 → 단순한 모델 (Linear, Logistic, KNN). 딥러닝 X.
- 1000~10000개 → ML 모델 (Random Forest, XGBoost). 딥러닝은 어려움.
- 10000~100000개 → ML 위주, 딥러닝 시도 가능.
- 100000+ → 딥러닝도 적극 고려.

데이터 늘리는 방법:
- 더 모으기 (가장 좋음)
- Data Augmentation (이미지 회전, 자르기 등)
- Synthetic Data (생성형 모델로 가짜 데이터)
- Semi-supervised Learning (라벨 없는 데이터 활용)

### Q. 모델 정확도가 낮아요. 어떻게 올리죠?

다음 순서로 시도해 보세요.

1. **데이터부터 점검**
   - 노이즈, 이상치, 결측값
   - 데이터 누수 (test가 train에 들어감)
   - 라벨이 정확한가

2. **베이스라인 모델 확인**
   - Random Forest 같은 단순 강력 모델 시도
   - 그게 잘 나오면 거기서 시작

3. **특성 엔지니어링**
   - 새 특성 추가 (조합, 변환)
   - 안 좋은 특성 제거

4. **하이퍼파라미터 튜닝**
   - GridSearch, RandomizedSearch, Optuna

5. **다른 모델 시도**
   - XGBoost, LightGBM, CatBoost
   - 딥러닝 (데이터 충분하면)

6. **앙상블**
   - 여러 모델의 예측을 평균/투표

### Q. Train 정확도는 높은데 Test가 낮아요.

**과적합입니다.**

해결:
1. 데이터 더 모으기 (가장 좋음)
2. 모델 단순화 (작은 모델, 얕은 트리)
3. 정규화 추가 (L1, L2, Dropout)
4. Early Stopping
5. Cross-Validation으로 정확한 평가

### Q. 어떤 모델을 써야 할지 모르겠어요.

**황금 법칙:** 단순한 것부터 시작.

```
1. 표 데이터, 작음 (~1만 개)
   → Logistic Regression / Linear Regression 베이스라인
   → 그 다음 Random Forest

2. 표 데이터, 큼 (10만 개+)
   → XGBoost / LightGBM

3. 이미지
   → Pre-trained CNN (ResNet, EfficientNet) + Fine-tuning
   → 정말 데이터 많으면 from scratch

4. 텍스트
   → Pre-trained Transformer (BERT, RoBERTa)
   → 또는 LLM API (GPT, Claude)

5. 시계열
   → 단순한 건 ARIMA, Prophet
   → 복잡한 건 LSTM, Transformer
```

### Q. 학습이 너무 오래 걸려요.

여러 가지 시도:
1. **데이터 줄이기** (실험 단계엔 일부만)
2. **모델 작게 하기**
3. **GPU 쓰기**
4. **batch_size 늘리기** (메모리 허용하면)
5. **Mixed Precision (FP16)**
6. **early stopping**
7. **simpler 모델 (Linear → RF → XGBoost → DL)**

## 진로

### Q. ML 엔지니어 vs 데이터 사이언티스트 — 뭐가 달라요?

| | 데이터 사이언티스트 | ML 엔지니어 |
|---|---|---|
| **주력** | 분석, 모델링, 인사이트 | 시스템, 배포, 인프라 |
| **도구** | Jupyter, sklearn, pandas | Docker, K8s, Airflow |
| **결과물** | 리포트, 노트북 | API, 파이프라인 |
| **요구 스킬** | 통계, 도메인 지식 | 소프트웨어 엔지니어링 |

회사마다 정의가 달라요. 작은 회사는 한 사람이 둘 다.

### Q. 어디서부터 취업 준비를 시작해야 해요?

1. **포트폴리오 만들기** (가장 중요)
   - GitHub에 ML 프로젝트 3~5개
   - 캐글 메달 (가능하면)
   - 블로그 (학습 정리)

2. **기본기 다지기**
   - 이 자료 같은 걸로 ML 기본
   - Python, SQL, Git

3. **인턴십 / 신입 공고 보기**
   - 큰 회사보단 작은 회사가 진입 쉬움
   - 인턴 → 정규직 트랙 이용

4. **네트워킹**
   - 컨퍼런스 (Pycon, ML 컨퍼런스)
   - 스터디 그룹
   - 멘토 찾기

### Q. 어떤 회사에 가야 ML 일을 할 수 있어요?

ML 채용이 활발한 회사 유형:

- **빅테크**: 카카오, 네이버, 삼성, LG, 쿠팡 등
- **AI 스타트업**: Upstage, 뤼튼, 솔루드 등
- **금융**: 은행, 보험, 카드사 (사기 탐지, 신용 평가)
- **전자상거래**: 마켓컬리, 무신사 등 (추천)
- **자율주행**: 현대모비스, 라이드플럭스 등
- **의료 AI**: Lunit, 뷰노 등

기억할 점: **모든 회사가 점점 ML이 필요해져요.** 본인이 일하고 싶은 도메인의 회사를 찾으세요.

## 도구와 환경

### Q. PyTorch vs TensorFlow — 뭘 배워야 해요?

**PyTorch.** 2026년 기준 압도적입니다.

- 학계: 95%+ PyTorch
- 산업: 50%+ PyTorch (점점 증가)

TensorFlow도 여전히 큰 회사 (Google 등)에서 씀. PyTorch 익히면 TensorFlow는 며칠이면 옮길 수 있어요.

### Q. Mac M1/M2/M3에서 GPU를 쓸 수 있어요?

**됩니다.** PyTorch가 MPS (Metal Performance Shaders)를 지원해요.

```python
device = torch.device('mps' if torch.backends.mps.is_available() else 'cpu')
```

다만:
- NVIDIA CUDA보다 호환성이 약간 떨어져요
- 일부 모델/라이브러리가 MPS를 지원 안 함
- 큰 모델이라면 클라우드 GPU가 더 나음

### Q. 클라우드 GPU 어디서 빌려요?

| 서비스 | 장점 | 시간당 가격 (대략) |
|--------|------|------------------|
| **Colab Pro** | 가장 쉬움 | 정액제 ($10/월) |
| **Vast.ai** | 가장 저렴 | $0.3~1.0 |
| **Runpod** | 사용 쉬움 | $0.4~2.0 |
| **AWS / GCP / Azure** | 큰 회사용 | $1~10 |
| **Lambda Labs** | ML 특화 | $0.5~2.0 |

학습 단계엔 **Colab Pro** 또는 **Vast.ai** 추천.

### Q. 회사에 ML 인프라가 없어요. 뭐부터 해야 해요?

작게 시작하세요.

1. Git + GitHub
2. Colab 또는 본인 PC + Python venv
3. 첫 모델 만들기
4. Jupyter Notebook 공유 (HTML로 export)
5. (필요해지면) MLflow 도입
6. (서비스화 결정되면) Docker + 클라우드

처음부터 거대한 인프라 만들지 마세요. 필요한 만큼만 도입.

## 윤리와 책임

### Q. AI가 일자리를 뺏는다는데 진짜인가요?

**부분적으로 진실.** AI가 단순 반복 작업을 대체하고 있어요. 하지만 **AI가 못하는 일을 하는 사람**의 가치는 오히려 올라갑니다.

ML을 배우는 분이라면, AI가 일자리를 뺏는 쪽이 아니라 **AI를 도구로 쓰는 쪽**에 서게 됩니다.

### Q. 내가 만든 모델이 차별을 한다면 어떡해요?

진지한 문제예요. 다음을 항상 점검:

1. **데이터 편향 확인** — 학습 데이터에 특정 그룹 비율이 적은가?
2. **그룹별 성능 비교** — 성별, 인종, 나이별 정확도 차이?
3. **반복적 모니터링** — 배포 후에도 계속 봐야 함
4. **문서화** — 어떤 한계가 있는지 명시

회사라면 윤리 검토 위원회나 법무팀과 협업.

### Q. ChatGPT가 거짓말을 해요. 왜요?

LLM은 **다음 단어를 예측**하는 모델이에요. 진실 여부를 알지 못해요. 그럴듯한 단어들을 이어 붙이는 거죠.

이걸 **할루시네이션(Hallucination)** 이라고 합니다. 해결 방법:
- 사실 확인 (Fact-checking)
- RAG (Retrieval-Augmented Generation)로 출처 제공
- 신뢰할 수 있는 도구 호출 (Tool use)

LLM의 답은 항상 **참고만**. 중요한 결정엔 인간이 검증.
