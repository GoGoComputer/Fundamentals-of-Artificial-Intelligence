## 1장 — 파이썬 기초

> **소요 시간:** 5~7시간 (하루 30분이면 2주)
> **선수 지식:** 없음

### 무엇을 배우나요?
- `print`, 변수, 자료형
- 리스트 / 튜플 / 셋 / 딕셔너리
- 조건문(`if`)과 반복문(`for`, `while`)
- 함수 만들기와 쓰기

### 끝나면 할 수 있는 것
- 간단한 계산기를 함수로 만들 수 있다.
- 100명의 학생 점수가 든 리스트에서 평균 / 최고점 / 최저점을 구할 수 있다.
- 다음 장의 머신러닝 코드를 보고 "아, 이건 함수 호출이구나" 정도는 알 수 있다.

### 왜 이 정도만 배우나요?
파이썬은 깊이 들어가면 끝이 없어요(데코레이터, 메타클래스, async, ...). 하지만 머신러닝을 시작하는 데에는 **파이썬의 20%만 알아도 80%의 일을 할 수 있습니다.** 그 20%만 압축해서 다룹니다.

## 2장 — 머신러닝 분류

> **소요 시간:** 6~8시간
> **선수 지식:** 1장

### 무엇을 배우나요?
- "분류(Classification)"가 무엇인지
- 데이터를 학습용/검증용/평가용으로 나누는 이유
- 정규화(scaling)는 왜 하는가
- SVM, 랜덤 포레스트, 로지스틱 회귀 — 알고리즘 3종 비교
- 정확도, 정밀도, 재현율, F1 — 어떤 지표를 언제 봐야 하는가
- 하이퍼파라미터 튜닝 (Grid Search vs Random Search)

### 실습 데이터
- MNIST 손글씨 숫자 (0~9)
- (선택) 타이타닉 생존 예측

### 끝나면 할 수 있는 것
- 새로운 분류 데이터셋을 받아 처음부터 끝까지 모델을 학습시킬 수 있다.
- "이 모델이 좋은 모델인가?"를 정확도 외에 여러 각도로 따져볼 수 있다.
- 동료 개발자가 보낸 분류 코드 PR을 리뷰할 수 있다.

## 3장 — 머신러닝 회귀

> **소요 시간:** 5~7시간
> **선수 지식:** 1장, 2장

### 무엇을 배우나요?
- "회귀(Regression)"가 무엇인지 — 분류와의 차이
- 선형 회귀의 직관 (수식 없이)
- 정규화(Regularization): Lasso(L1), Ridge(L2), ElasticNet
- 트리 기반 회귀: Random Forest, Gradient Boosting, XGBoost
- MSE, RMSE, MAE, R² — 회귀 지표들

### 실습 데이터
- Boston Housing (집값 예측)

### 끝나면 할 수 있는 것
- 연속된 값을 예측해야 하는 문제(매출, 온도, 수요량 등)에 모델을 붙일 수 있다.
- 선형 모델의 가중치(weight)를 보고 "어떤 변수가 중요했나"를 설명할 수 있다.
- 정규화가 왜 필요한지 다른 사람에게 설명할 수 있다.

[📖 3장 시작 →](03-머신러닝-회귀/)

## 4장 — 딥러닝 분류

> **소요 시간:** 8~10시간
> **선수 지식:** 1, 2장

### 무엇을 배우나요?
- 인공 신경망의 직관 (perceptron부터 MLP까지)
- 활성화 함수, 손실 함수, 옵티마이저가 무엇인가
- PyTorch로 첫 신경망 만들기
- GPU를 어떻게 쓰는지 (Colab GPU 활용)
- 학습 루프(forward → loss → backward → step)
- 학습 결과 시각화 (loss curve, confusion matrix)

### 실습 데이터
- MNIST 손글씨 숫자 (다시 한 번, 이번엔 신경망으로)

### 끝나면 할 수 있는 것
- PyTorch로 자기만의 신경망을 정의하고 학습시킬 수 있다.
- "왜 ML 모델로는 안 되고 DL이 필요한가?"를 본인의 언어로 답할 수 있다.
- Colab에서 GPU를 켜고 학습 속도를 10배 끌어올릴 수 있다.

## 5장 — 딥러닝 회귀

> **소요 시간:** 6~8시간
> **선수 지식:** 1, 2, 3, 4장

### 무엇을 배우나요?
- 회귀를 신경망으로 풀 때 무엇이 달라지는가
- **과적합(overfitting)** — 모든 ML 엔지니어의 평생의 적
- 과적합과 싸우는 세 가지 무기:
  - Early Stopping
  - Dropout
  - L2 Regularization (weight decay)
- 학습/검증 곡선 읽는 법

### 실습 데이터
- Boston Housing (다시, 이번엔 신경망으로)

### 끝나면 할 수 있는 것
- 학습이 잘 되는지 안 되는지를 곡선만 보고 진단할 수 있다.
- 과적합이 일어났을 때 어떤 조치를 어떤 순서로 취해야 할지 안다.
- 모델 체크포인트를 저장하고 불러올 수 있다.

## 6장 — 프로덕션으로 가는 길

> **소요 시간:** 5~10시간 (얼마나 깊이 들어가시느냐에 따라)
> **선수 지식:** 1~5장 (또는 ML 실무 경험)

### 무엇을 배우나요?
- 학습된 모델을 어떻게 "서비스"로 만드는가
- 모델 저장 / 불러오기 / 버전 관리
- FastAPI로 모델 서빙 API 만들기
- Docker로 패키징
- 배포된 모델의 성능을 어떻게 모니터링하는가
- "데이터가 변했을 때(drift)" 어떻게 알아채는가
- 재학습 파이프라인의 기초

### 끝나면 할 수 있는 것
- 노트북 안의 모델을 외부에서 호출할 수 있는 API로 바꿀 수 있다.
- 회사 인프라에 모델을 안전하게 배포할 수 있는 청사진을 그릴 수 있다.
- "MLOps"라는 단어를 들었을 때 무엇을 이야기하는지 안다.
