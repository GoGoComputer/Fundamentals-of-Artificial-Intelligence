## 분류 코드와 회귀 코드의 차이

[4장](../../04-딥러닝-분류/)에서 분류 신경망을 만들었어요. 회귀 신경망과 비교해 봅시다.

### 분류 (4장)

```python
class Classifier(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 256)
        self.fc2 = nn.Linear(256, 10)    # ← 클래스 수만큼

    def forward(self, x):
        x = torch.relu(self.fc1(x))
        return self.fc2(x)               # ← 활성화 X (CrossEntropy가 처리)


loss_fn = nn.CrossEntropyLoss()          # ← 분류용 손실
```

### 회귀 (이 장)

```python
class Regressor(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(13, 64)     # 보스턴이라 입력 13
        self.fc2 = nn.Linear(64, 1)      # ← 출력 1개 (숫자 하나)

    def forward(self, x):
        x = torch.relu(self.fc1(x))
        return self.fc2(x)               # ← 활성화 X (그냥 숫자)


loss_fn = nn.MSELoss()                   # ← 회귀용 손실
```

차이가 보이시죠?

| | 분류 | 회귀 |
|---|---|---|
| 마지막 층 | `Linear(..., n_classes)` | `Linear(..., 1)` |
| 출력 | 각 클래스 점수 | 예측 숫자 |
| 손실 | `CrossEntropyLoss` | `MSELoss` (또는 L1, Huber) |
| 라벨 | int | float |

**나머지는 다 같아요.** 학습 루프 패턴, 옵티마이저, GPU 사용 — 모두 동일.

## 보스턴 데이터로 회귀 신경망

3장에서 머신러닝으로 풀었던 보스턴 집값 예측을 신경망으로 다시 풀어 봅시다.

### 데이터 준비

```python
import numpy as np
import pandas as pd
import torch
from torch.utils.data import TensorDataset, DataLoader
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler


# 데이터 (3장 코드 재사용)
data_url = "http://lib.stat.cmu.edu/datasets/boston"
raw_df = pd.read_csv(data_url, sep=r"\s+", skiprows=22, header=None)
data = np.hstack([raw_df.values[::2, :], raw_df.values[1::2, :2]])
target = raw_df.values[1::2, 2]


# 분할
X_train, X_test, y_train, y_test = train_test_split(
    data, target, test_size=0.2, random_state=42
)


# 정규화 (X와 y 둘 다!)
scaler_X = StandardScaler().fit(X_train)
scaler_y = StandardScaler().fit(y_train.reshape(-1, 1))

X_train_s = scaler_X.transform(X_train)
X_test_s = scaler_X.transform(X_test)
y_train_s = scaler_y.transform(y_train.reshape(-1, 1)).flatten()
y_test_s = scaler_y.transform(y_test.reshape(-1, 1)).flatten()


# 텐서로
X_train_t = torch.tensor(X_train_s, dtype=torch.float32)
X_test_t = torch.tensor(X_test_s, dtype=torch.float32)
y_train_t = torch.tensor(y_train_s, dtype=torch.float32)
y_test_t = torch.tensor(y_test_s, dtype=torch.float32)


# DataLoader
train_dataset = TensorDataset(X_train_t, y_train_t)
test_dataset = TensorDataset(X_test_t, y_test_t)

train_loader = DataLoader(train_dataset, batch_size=32, shuffle=True)
test_loader = DataLoader(test_dataset, batch_size=32, shuffle=False)
```

### ⚠️ 회귀에서 주의: y도 정규화하세요

분류에서 y는 카테고리(정수)라 정규화 안 했어요. 하지만 **회귀에서 y는 숫자**라, **타겟의 스케일이 학습에 큰 영향**을 줘요.

타겟이 큰 값(예: 100,000)이면 손실이 너무 커서 학습이 불안정해질 수 있어요.

**해결: y도 정규화** (StandardScaler).

예측 후에 다시 원래 단위로 되돌리는 걸 잊지 마세요.

```python
y_pred = model(X_test_t)               # 정규화된 단위
y_pred_original = scaler_y.inverse_transform(y_pred.numpy().reshape(-1, 1))
```

## 모델 정의

```python
import torch.nn as nn


class BostonRegressor(nn.Module):
    def __init__(self, input_dim=13):
        super().__init__()
        self.fc1 = nn.Linear(input_dim, 64)
        self.fc2 = nn.Linear(64, 32)
        self.fc3 = nn.Linear(32, 1)        # 출력 1

    def forward(self, x):
        x = torch.relu(self.fc1(x))
        x = torch.relu(self.fc2(x))
        return self.fc3(x).squeeze()        # (B, 1) → (B,)


device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
model = BostonRegressor(input_dim=X_train.shape[1]).to(device)
```

`.squeeze()`로 마지막 차원을 없애줘요. `(B, 1)` → `(B,)`. y와 모양을 맞추기 위해서.

## 학습 루프

```python
import torch.optim as optim


loss_fn = nn.MSELoss()
optimizer = optim.Adam(model.parameters(), lr=0.001)

n_epochs = 100
train_losses = []
test_losses = []


for epoch in range(n_epochs):
    # === 학습 ===
    model.train()
    epoch_loss = 0
    
    for X_batch, y_batch in train_loader:
        X_batch, y_batch = X_batch.to(device), y_batch.to(device)
        
        optimizer.zero_grad()
        y_pred = model(X_batch)
        loss = loss_fn(y_pred, y_batch)
        loss.backward()
        optimizer.step()
        
        epoch_loss += loss.item()
    
    train_loss = epoch_loss / len(train_loader)

    # === 평가 ===
    model.eval()
    test_loss = 0
    
    with torch.no_grad():
        for X_batch, y_batch in test_loader:
            X_batch, y_batch = X_batch.to(device), y_batch.to(device)
            y_pred = model(X_batch)
            test_loss += loss_fn(y_pred, y_batch).item()
    
    test_loss /= len(test_loader)

    train_losses.append(train_loss)
    test_losses.append(test_loss)
    
    if (epoch + 1) % 10 == 0:
        print(f"Epoch {epoch+1}/{n_epochs}  "
              f"Train Loss: {train_loss:.4f}  Test Loss: {test_loss:.4f}")
```

**4장의 분류 코드와 거의 같죠?** 차이는:
- `loss_fn`이 MSELoss
- accuracy 계산 안 함

## 평가: 원래 단위로 되돌리기

```python
import numpy as np
from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score


# 전체 test 예측
model.eval()
all_preds = []
all_targets = []

with torch.no_grad():
    for X_batch, y_batch in test_loader:
        X_batch = X_batch.to(device)
        y_pred = model(X_batch).cpu().numpy()
        all_preds.extend(y_pred)
        all_targets.extend(y_batch.numpy())


# 정규화된 단위에서의 점수
all_preds = np.array(all_preds)
all_targets = np.array(all_targets)


# 원래 단위로 되돌리기
preds_original = scaler_y.inverse_transform(all_preds.reshape(-1, 1)).flatten()
targets_original = scaler_y.inverse_transform(all_targets.reshape(-1, 1)).flatten()


# 평가 (원래 단위)
rmse = np.sqrt(mean_squared_error(targets_original, preds_original))
mae = mean_absolute_error(targets_original, preds_original)
r2 = r2_score(targets_original, preds_original)

print(f"\n[평가 — 원래 단위 ($1000)]")
print(f"  RMSE: {rmse:.4f}")
print(f"  MAE:  {mae:.4f}")
print(f"  R²:   {r2:.4f}")
```

## ML 모델과 비교

3장에서 만들었던 모델들과 비교해 봅시다.

| 모델 | RMSE (Boston) |
|------|--------------|
| Linear Regression | ~4.93 |
| Random Forest | ~2.88 |
| XGBoost | ~2.69 |
| **신경망 (이 장)** | **~3.1** |

흥미롭죠? **신경망이 트리 모델(RF, XGBoost)보다 약간 못해요.**

이게 작은 정형 데이터의 특징이에요. **데이터가 적고 표 형태면, 트리 모델이 신경망보다 자주 더 좋습니다.**

신경망의 진가는:
- 데이터가 많을 때 (100만+)
- 데이터가 비정형일 때 (이미지, 음성, 텍스트)

이 두 조건을 만족할 때 빛납니다. 보스턴 데이터(506개)로는 신경망의 진면목을 볼 수 없어요. 이 장에서는 그냥 **연습 도구**로 쓰는 거예요.

## "그럼 회귀에 신경망 쓰지 마요?"

아니에요. 신경망 회귀를 쓰는 좋은 경우가 많아요.

- **데이터가 매우 많을 때** (수백만 row)
- **변수 간 복잡한 비선형 관계**
- **이미지/텍스트로부터 숫자 예측** (예: 사진으로 사람 나이 예측)
- **시계열 예측** (LSTM, Transformer)

**작은 표 데이터** → 트리 모델 위주
**큰 표 데이터 또는 비정형 데이터** → 신경망

이 가이드라인만 기억하세요.

## 이해를 더 단단하게 만드는 보강 설명

이 단원은 처음 읽을 때는 "아, 이런 개념이 있구나" 정도로 넘어가기 쉽지만, 실제로는 작은 차이가 나중에 큰 성능 차이로 이어집니다. 특히 **5.1 회귀를 신경망으로** 같은 주제는 한 번에 완벽히 이해하려고 하기보다, 다음 세 가지를 반복하면서 이해를 깊게 만드는 편이 훨씬 효과적입니다.

또 하나 중요한 점은, 이 단원의 내용이 단독으로 존재하지 않는다는 사실입니다. 실제 프로젝트에서는 데이터 준비, 모델링, 평가, 배포가 끊기지 않고 이어집니다. 그래서 지금 배우는 **5.1 회귀를 신경망으로**도 나중에 다른 단계와 반드시 연결됩니다. 이 연결을 의식하면서 읽으면, 단편 지식이 아니라 시스템 관점으로 이해하게 됩니다.

## 자주 하는 오해와 바로잡기

### 1) "이론만 이해하면 실습은 저절로 된다"
아닙니다. 이론 이해와 코드 실행 능력은 별개의 근육입니다. 둘을 같이 써야 합니다.

### 2) "예제 코드가 돌아갔으니 완전히 이해했다"
동작 확인은 시작일 뿐입니다. 파라미터를 바꿨을 때 결과가 어떻게 달라지는지까지 확인해야 이해가 완성됩니다.

### 3) "좋은 모델은 항상 하나다"
데이터 특성과 제약(시간, 메모리, 응답 속도)에 따라 최선의 선택은 달라집니다.

### 4) "지표 숫자만 높으면 끝이다"
지표는 해석이 필요합니다. 어떤 데이터에서, 어떤 분포에서, 어떤 기준으로 얻은 점수인지 함께 보아야 합니다.

### 5) "지금은 초반이니까 배포는 나중에 생각해도 된다"
초반부터 재현성과 저장 구조를 습관화해야 나중에 배포 단계에서 무너지지 않습니다.

## 한 번 더 정리

5.1 회귀를 신경망으로는 단순히 시험용 개념이 아니라, 실제 문제를 풀 때 반복해서 쓰게 되는 실전 도구입니다. 처음에는 낯설 수 있지만, 작은 실험을 반복하면 감각이 생기고, 감각이 생기면 코드와 지표를 읽는 속도가 급격히 빨라집니다. 지금 단계에서 중요한 것은 완벽한 암기가 아니라, **작게 실행하고, 결과를 읽고, 다시 수정하는 루프**를 몸에 익히는 것입니다. 이 루프를 익히면 다음 장으로 넘어갈수록 학습 속도가 오히려 빨라집니다.
