# 3.1 회귀란 무엇인가

## 분류와 회귀, 한 그림으로

분류와 회귀는 머신러닝의 양 날개예요.

```
┌─────────────────────────────────────────────┐
│           지도학습 (Supervised Learning)      │
│                                              │
│   ┌─────────────────┐  ┌─────────────────┐  │
│   │     분류         │  │     회귀         │  │
│   │ (Classification) │  │  (Regression)   │  │
│   │                  │  │                  │  │
│   │  답이 카테고리    │  │  답이 숫자       │  │
│   │  "강아지/고양이"  │  │  "5억 원"        │  │
│   │  "스팸/햄"       │  │  "27도"          │  │
│   │  "0~9 숫자"      │  │  "73%"           │  │
│   └─────────────────┘  └─────────────────┘  │
└─────────────────────────────────────────────┘
```

답의 형태만 다르고, 나머지는 거의 같아요.

---

## 회귀의 정의

> **회귀(Regression)** 는 입력을 받아 **연속된 숫자**를 예측하는 문제이다.

핵심은 **"연속된"** 이에요.

- ✅ 회귀: 1.5, 27.3, 100.0, -5.7 ... (실수 어떤 값이든)
- ❌ 분류: 0, 1, 2 ... 또는 "강아지", "고양이" (정해진 몇 개)

### 헷갈리는 경계 케이스

> "별점 1~5점을 예측하는 문제는 분류일까 회귀일까?"

좋은 질문입니다. 사실 **둘 다로 풀 수 있어요.**

- **분류로 풀면**: 5개 카테고리 분류 (1점, 2점, 3점, 4점, 5점)
- **회귀로 풀면**: 1~5 사이의 연속값 예측 (3.7점도 가능)

뭐가 더 좋은지는 상황 따라 달라요. 보통:
- 별점 자체가 의미 있으면 (4점과 5점은 큰 차이) → 회귀가 자연스러움
- 별점이 그냥 라벨이면 → 분류

> "확률을 예측하는 문제는 분류일까 회귀일까?"

이건 좀 더 미묘해요. 사실 **이진 분류 모델은 본질적으로 0~1의 확률을 예측**하는 거예요. (`predict_proba`) 그리고 0.5 기준으로 카테고리를 정합니다. 그래서 "확률 예측"은 분류의 부산물이라고 보는 게 맞아요.

순수한 회귀의 예: 가격, 온도, 시간, 거리, 비율, 점수, 카운트.

---

## 회귀 문제의 진짜 예시

### 1. 부동산 가격 예측
- **입력**: 평수, 방 개수, 지하철역 거리, 학군 등
- **출력**: 추정 매매가 (예: 6억 5천만 원)
- **활용**: 부동산 앱의 시세 예측

### 2. 매출 예측
- **입력**: 작년 매출, 마케팅 비용, 계절, 경쟁사 동향
- **출력**: 다음 분기 매출 (예: 12억 원)
- **활용**: 사업 계획 수립

### 3. 수요 예측
- **입력**: 과거 판매량, 가격, 프로모션, 날씨
- **출력**: 다음 주 판매량 (예: 1,250개)
- **활용**: 재고 관리

### 4. 의료 수치 예측
- **입력**: 환자의 검사 수치들
- **출력**: 향후 6개월 후의 어떤 지표 (예: 혈압 130)
- **활용**: 예방 의학

### 5. 광고 클릭률 (CTR) 예측
- **입력**: 사용자 정보, 광고 정보, 시간
- **출력**: 클릭 확률 (예: 0.034 = 3.4%)
- **활용**: 광고 추천 시스템

## 회귀가 어떻게 작동하나요?

분류는 "두 그룹을 가르는 선"을 찾았죠. 회귀는 **"데이터의 추세를 따르는 선"** 을 찾아요.

```
가격(억원)
   ↑
12 │           •
   │       •     •
10 │     •     •  •
   │   •        •
 8 │  •     •
   │•   •
 6 │      •
   └────────────────→ 평수(평)
       20  30  40  50

→ 데이터들의 가운데를 통과하는 선을 찾자!

가격(억원)
   ↑
12 │           •
   │       •  ●  •
10 │     ●  ●     •
   │   • ●        •
 8 │  ● ●   •
   │● ●●
 6 │   •
   └────────────────→
       20  30  40  50
   (●는 모델의 예측값)
```

이 선을 **회귀선(regression line)** 이라고 해요. 새 데이터(예: 35평 집)가 들어오면, 회귀선의 해당 값(예: 9.5억)을 예측값으로 답하는 거예요.

### 좋은 회귀선이란?

데이터에서 너무 멀리 떨어지지 않은 선이 좋은 선이에요. **모든 점에서 선까지의 거리(오차)의 합을 최소화**하는 선을 찾는 게 회귀의 목표입니다.

```
가격(억원)
   ↑
12 │           •
   │       •  │
10 │     •────●─── 선
   │   • │
 8 │  • ↕  ← 오차들
   │● │
 6 │   
   └────────────────→
```

이 오차들을 **잔차(residual)** 라고 부르고, 잔차의 제곱의 평균이 우리의 평가 지표인 **MSE(평균 제곱 오차)** 가 됩니다. (3.4에서 자세히)

## 회귀 알고리즘들

분류처럼 회귀에도 여러 알고리즘이 있어요. 모양만 다를 뿐이에요.

| 알고리즘 | 모양 | 특징 |
|---------|------|------|
| **선형 회귀** | 직선 | 가장 단순, 가장 빠름 |
| **다항 회귀** | 곡선 (2차, 3차...) | 약간의 비선형 |
| **Lasso, Ridge** | 직선 (정규화) | 과적합 방지, 특성 선택 |
| **SVR** | 곡선 (kernel) | 작은 데이터에 강함 |
| **랜덤 포레스트** | 계단 모양 | 거의 항상 잘 함 |
| **XGBoost / LightGBM** | 더 정교한 계단 | 정확도 최강 |
| **신경망 (DL)** | 매우 복잡한 곡선 | 데이터 많을 때 |

분류 챕터에서 봤던 4가지 모델(LogReg, KNN, SVM, RF)이 회귀에서도 거의 그대로 있어요. 이름만 살짝 바뀝니다.

| 분류 | 회귀 |
|------|------|
| `LogisticRegression` | `LinearRegression` |
| `KNeighborsClassifier` | `KNeighborsRegressor` |
| `SVC` | `SVR` |
| `RandomForestClassifier` | `RandomForestRegressor` |

## 회귀의 코드 패턴

분류랑 똑같아요. 정말 똑같습니다.

```python
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error

# 1. 모델
model = LinearRegression()

# 2. 학습
model.fit(X_train, y_train)

# 3. 예측
y_pred = model.predict(X_test)

# 4. 평가
mse = mean_squared_error(y_test, y_pred)
rmse = mse ** 0.5
print(f"RMSE: {rmse:.2f}")
```

차이가 보이세요? `accuracy_score` 가 `mean_squared_error`로 바뀌었을 뿐이에요. 알고리즘만 바뀌면 회귀도 분류처럼 같은 패턴으로 풀 수 있어요.

## 정리

```
┌────────────────────────────────────────────┐
│                회귀 (Regression)             │
│                                              │
│   입력 → [모델] → 연속된 숫자                 │
│                                              │
│   - 답이 숫자 (1.5, 27.3, ...)              │
│   - 모델은 "추세선"을 찾는다                 │
│   - 잔차의 제곱 합을 최소화                  │
│   - 평가 지표: MSE, RMSE, MAE, R²           │
└────────────────────────────────────────────┘
```

분류와 거의 같아요. 차이는 답의 형태와 평가 지표뿐.
