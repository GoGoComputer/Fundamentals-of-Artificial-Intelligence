# 3.1 회귀란 무엇인가

## 분류와 회귀, 한 그림으로

분류와 회귀는 머신러닝의 양 날개예요.

```
┌─────────────────────────────────────────────┐
│           지도학습 (Supervised Learning)      │
│                                              │
│   ┌─────────────────┐  ┌─────────────────┐  │
│   │     분류         │  │     회귀         │  │
│   │ (Classification) │  │  (Regression)   │  │
│   │                  │  │                  │  │
│   │  답이 카테고리    │  │  답이 숫자       │  │
│   │  "강아지/고양이"  │  │  "5억 원"        │  │
│   │  "스팸/햄"       │  │  "27도"          │  │
│   │  "0~9 숫자"      │  │  "73%"           │  │
│   └─────────────────┘  └─────────────────┘  │
└─────────────────────────────────────────────┘
```

답의 형태만 다르고, 나머지는 거의 같아요.

---

## 회귀의 정의

> **회귀(Regression)** 는 입력을 받아 **연속된 숫자**를 예측하는 문제이다.

핵심은 **"연속된"** 이에요.

- ✅ 회귀: 1.5, 27.3, 100.0, -5.7 ... (실수 어떤 값이든)
- ❌ 분류: 0, 1, 2 ... 또는 "강아지", "고양이" (정해진 몇 개)

### 헷갈리는 경계 케이스

> "별점 1~5점을 예측하는 문제는 분류일까 회귀일까?"

좋은 질문입니다. 사실 **둘 다로 풀 수 있어요.**

- **분류로 풀면**: 5개 카테고리 분류 (1점, 2점, 3점, 4점, 5점)
- **회귀로 풀면**: 1~5 사이의 연속값 예측 (3.7점도 가능)

뭐가 더 좋은지는 상황 따라 달라요. 보통:
- 별점 자체가 의미 있으면 (4점과 5점은 큰 차이) → 회귀가 자연스러움
- 별점이 그냥 라벨이면 → 분류

> "확률을 예측하는 문제는 분류일까 회귀일까?"

이건 좀 더 미묘해요. 사실 **이진 분류 모델은 본질적으로 0~1의 확률을 예측**하는 거예요. (`predict_proba`) 그리고 0.5 기준으로 카테고리를 정합니다. 그래서 "확률 예측"은 분류의 부산물이라고 보는 게 맞아요.

순수한 회귀의 예: 가격, 온도, 시간, 거리, 비율, 점수, 카운트.

---

## 회귀 문제의 진짜 예시

### 1. 부동산 가격 예측
- **입력**: 평수, 방 개수, 지하철역 거리, 학군 등
- **출력**: 추정 매매가 (예: 6억 5천만 원)
- **활용**: 부동산 앱의 시세 예측

### 2. 매출 예측
- **입력**: 작년 매출, 마케팅 비용, 계절, 경쟁사 동향
- **출력**: 다음 분기 매출 (예: 12억 원)
- **활용**: 사업 계획 수립

### 3. 수요 예측
- **입력**: 과거 판매량, 가격, 프로모션, 날씨
- **출력**: 다음 주 판매량 (예: 1,250개)
- **활용**: 재고 관리

### 4. 의료 수치 예측
- **입력**: 환자의 검사 수치들
- **출력**: 향후 6개월 후의 어떤 지표 (예: 혈압 130)
- **활용**: 예방 의학

### 5. 광고 클릭률 (CTR) 예측
- **입력**: 사용자 정보, 광고 정보, 시간
- **출력**: 클릭 확률 (예: 0.034 = 3.4%)
- **활용**: 광고 추천 시스템

## 회귀가 어떻게 작동하나요?

분류는 "두 그룹을 가르는 선"을 찾았죠. 회귀는 **"데이터의 추세를 따르는 선"** 을 찾아요.

```
가격(억원)
   ↑
12 │           •
   │       •     •
10 │     •     •  •
   │   •        •
 8 │  •     •
   │•   •
 6 │      •
   └────────────────→ 평수(평)
       20  30  40  50

→ 데이터들의 가운데를 통과하는 선을 찾자!

가격(억원)
   ↑
12 │           •
   │       •  ●  •
10 │     ●  ●     •
   │   • ●        •
 8 │  ● ●   •
   │● ●●
 6 │   •
   └────────────────→
       20  30  40  50
   (●는 모델의 예측값)
```

이 선을 **회귀선(regression line)** 이라고 해요. 새 데이터(예: 35평 집)가 들어오면, 회귀선의 해당 값(예: 9.5억)을 예측값으로 답하는 거예요.

### 좋은 회귀선이란?

데이터에서 너무 멀리 떨어지지 않은 선이 좋은 선이에요. **모든 점에서 선까지의 거리(오차)의 합을 최소화**하는 선을 찾는 게 회귀의 목표입니다.

```
가격(억원)
   ↑
12 │           •
   │       •  │
10 │     •────●─── 선
   │   • │
 8 │  • ↕  ← 오차들
   │● │
 6 │   
   └────────────────→
```

이 오차들을 **잔차(residual)** 라고 부르고, 잔차의 제곱의 평균이 우리의 평가 지표인 **MSE(평균 제곱 오차)** 가 됩니다. (3.4에서 자세히)

## 회귀 알고리즘들

분류처럼 회귀에도 여러 알고리즘이 있어요. 모양만 다를 뿐이에요.

| 알고리즘 | 모양 | 특징 |
|---------|------|------|
| **선형 회귀** | 직선 | 가장 단순, 가장 빠름 |
| **다항 회귀** | 곡선 (2차, 3차...) | 약간의 비선형 |
| **Lasso, Ridge** | 직선 (정규화) | 과적합 방지, 특성 선택 |
| **SVR** | 곡선 (kernel) | 작은 데이터에 강함 |
| **랜덤 포레스트** | 계단 모양 | 거의 항상 잘 함 |
| **XGBoost / LightGBM** | 더 정교한 계단 | 정확도 최강 |
| **신경망 (DL)** | 매우 복잡한 곡선 | 데이터 많을 때 |

분류 챕터에서 봤던 4가지 모델(LogReg, KNN, SVM, RF)이 회귀에서도 거의 그대로 있어요. 이름만 살짝 바뀝니다.

| 분류 | 회귀 |
|------|------|
| `LogisticRegression` | `LinearRegression` |
| `KNeighborsClassifier` | `KNeighborsRegressor` |
| `SVC` | `SVR` |
| `RandomForestClassifier` | `RandomForestRegressor` |

## 회귀의 코드 패턴

분류랑 똑같아요. 정말 똑같습니다.

```python
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error

# 1. 모델
model = LinearRegression()

# 2. 학습
model.fit(X_train, y_train)

# 3. 예측
y_pred = model.predict(X_test)

# 4. 평가
mse = mean_squared_error(y_test, y_pred)
rmse = mse ** 0.5
print(f"RMSE: {rmse:.2f}")
```

차이가 보이세요? `accuracy_score` 가 `mean_squared_error`로 바뀌었을 뿐이에요. 알고리즘만 바뀌면 회귀도 분류처럼 같은 패턴으로 풀 수 있어요.

## 헷갈리기 쉬운 포인트

회귀에서는 "맞음/틀림"보다 "얼마나 틀렸는가"가 핵심입니다.

- 평균 오차가 낮아도 특정 구간 오차가 크면 실전에서 문제가 됩니다.
- 단일 지표만 보면 모델 약점을 놓치기 쉽습니다.
- 예측값 단위가 비즈니스 의사결정 단위와 연결되어야 의미가 있습니다.

## 자주 하는 오해와 바로잡기

### 1) "이론만 이해하면 실습은 저절로 된다"
아닙니다. 이론 이해와 코드 실행 능력은 별개의 근육입니다. 둘을 같이 써야 합니다.

### 2) "예제 코드가 돌아갔으니 완전히 이해했다"
동작 확인은 시작일 뿐입니다. 파라미터를 바꿨을 때 결과가 어떻게 달라지는지까지 확인해야 이해가 완성됩니다.

### 3) "좋은 모델은 항상 하나다"
데이터 특성과 제약(시간, 메모리, 응답 속도)에 따라 최선의 선택은 달라집니다.

### 4) "지표 숫자만 높으면 끝이다"
지표는 해석이 필요합니다. 어떤 데이터에서, 어떤 분포에서, 어떤 기준으로 얻은 점수인지 함께 보아야 합니다.

### 5) "지금은 초반이니까 배포는 나중에 생각해도 된다"
초반부터 재현성과 저장 구조를 습관화해야 나중에 배포 단계에서 무너지지 않습니다.

## 실전 연결 시나리오

아래는 이 단원 내용을 실제 업무로 연결할 때 자주 쓰는 흐름입니다.

1. 문제를 한 문장으로 정의한다.  
예: "3.1 회귀란 무엇인가 관점에서 입력으로부터 어떤 출력을 만들 것인가?"

2. 성공 기준을 지표로 정한다.  
예: 정확도/RMSE/F1/지연시간 같은 운영 가능한 숫자로 정의.

3. 가장 단순한 베이스라인을 만든다.  
복잡한 모델보다 먼저, 빠르게 재현 가능한 기본 모델을 세웁니다.

4. 한 번에 하나씩 개선한다.  
전처리, 모델, 튜닝, 임계값, 배치 크기 등은 동시에 바꾸지 않고 순차적으로 바꿉니다.

5. 결과를 로그와 함께 저장한다.  
실험 이름, 파라미터, 점수, 데이터 버전을 함께 남겨야 재현이 됩니다.

6. 실패 케이스를 분석한다.  
점수 평균보다 오분류/고오차 샘플을 분석할 때 개선 아이디어가 나옵니다.

7. 배포 가능 형태로 정리한다.  
모델 파일, 메타데이터, 입력 스키마, 테스트 스크립트까지 묶습니다.

## 한 번 더 정리

3.1 회귀란 무엇인가는 단순히 시험용 개념이 아니라, 실제 문제를 풀 때 반복해서 쓰게 되는 실전 도구입니다. 처음에는 낯설 수 있지만, 작은 실험을 반복하면 감각이 생기고, 감각이 생기면 코드와 지표를 읽는 속도가 급격히 빨라집니다. 지금 단계에서 중요한 것은 완벽한 암기가 아니라, **작게 실행하고, 결과를 읽고, 다시 수정하는 루프**를 몸에 익히는 것입니다. 이 루프를 익히면 다음 장으로 넘어갈수록 학습 속도가 오히려 빨라집니다.
