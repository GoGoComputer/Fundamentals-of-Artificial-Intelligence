# 3.4 회귀 평가 지표

## "얼마나 잘 맞았어?"의 4가지 답

분류는 "맞췄나 틀렸나"가 명확했죠. 회귀는 답이 숫자라서 "얼마나 맞았나"가 좀 더 미묘해요. 예측이 100인데 실제가 110이라면? 90% 맞은 건가요? 10 차이만큼 틀린 건가요?

이걸 표현하는 방식이 4가지 있어요.

1. **MSE** (Mean Squared Error)
2. **RMSE** (Root Mean Squared Error)
3. **MAE** (Mean Absolute Error)
4. **R²** (R-squared, 결정계수)

각각 언제 어떤 걸 봐야 하는지 정리해 봅시다.

---

## 출발점: 잔차 (Residual)

모든 회귀 평가 지표는 **잔차**에서 시작합니다.

```
잔차 = 실제값 - 예측값
```

예: 실제 집값 10억인데 모델이 8억으로 예측 → 잔차 = 2억

샘플마다 잔차가 하나씩 있어요. 이걸 어떻게 종합하느냐가 지표의 차이입니다.

---

## 1. MSE (Mean Squared Error, 평균 제곱 오차)

가장 기본적인 지표예요.

```
MSE = (잔차₁² + 잔차₂² + ... + 잔차ₙ²) / n
```

수학 식으로 짧게 쓰면 이렇게 표현해요.

```
MSE = (1/n) × Σᵢ (yᵢ - ŷᵢ)²
```

`Σ` (시그마) 가 무서워 보이지만 그냥 **"다 더하라"** 는 뜻이에요. `Σᵢ aᵢ` = `a₁ + a₂ + a₃ + ...` 이렇게요. 위 두 식이 정확히 같은 말입니다.

읽으면: **"평균 제곱 오차 = (1/샘플 수) × 모든 (실제값 yᵢ - 예측값 ŷᵢ)의 제곱의 합."**

`ŷ` (와이햇)은 "예측값"의 표준 표기예요. 옷 위의 모자(hat)처럼 ^가 붙어 있다고 ŷ라고 부릅니다.

각 잔차를 **제곱**해서 평균을 냅니다.

### 왜 제곱하나요?

두 가지 이유예요.

1. **양수와 음수가 상쇄되지 않게.** 그냥 평균을 내면 +5와 -5가 0이 돼요.
2. **큰 오차에 더 큰 페널티.** 1²=1, 2²=4, 10²=100. 큰 오차가 더 무겁게 평가됨.

### 코드

```python
from sklearn.metrics import mean_squared_error
mse = mean_squared_error(y_test, y_pred)
```

### 단점

값이 직관적이지 않아요. **단위가 (실제 단위)²** 이거든요. 집값이 억 단위면 MSE는 억²인 거죠. "MSE 24억²" 들으면 감이 안 와요.

→ 그래서 RMSE를 씁니다.

---

## 2. RMSE (Root Mean Squared Error, 평균 제곱근 오차)

MSE에 제곱근(√)을 씌운 거예요.

```
RMSE = √MSE
```

### 코드

```python
from sklearn.metrics import mean_squared_error
import numpy as np

rmse = np.sqrt(mean_squared_error(y_test, y_pred))

# 또는 sklearn 1.4+ 에서
rmse = mean_squared_error(y_test, y_pred, squared=False)

# 또는 root_mean_squared_error
from sklearn.metrics import root_mean_squared_error
rmse = root_mean_squared_error(y_test, y_pred)
```

### 장점

**단위가 원래 단위로 돌아옴.** 집값 RMSE 4.93이면 "평균적으로 약 5천 달러 정도 틀린다" 라고 말할 수 있어요.

### 가장 자주 쓰는 지표

회귀에서 단 하나만 본다면, **RMSE**입니다. 가장 표준적이에요.

---

## 3. MAE (Mean Absolute Error, 평균 절대 오차)

제곱 대신 **절댓값**을 씁니다.

```
MAE = (|잔차₁| + |잔차₂| + ... + |잔차ₙ|) / n
```

### 코드

```python
from sklearn.metrics import mean_absolute_error
mae = mean_absolute_error(y_test, y_pred)
```

### MAE vs RMSE — 차이가 뭐예요?

가장 중요한 차이는 **이상치(outlier) 처리**입니다.

| | MAE | RMSE |
|---|---|---|
| 큰 오차 처리 | 작게 평가 | 크게 평가 |
| 이상치 영향 | 적음 | 큼 |
| 직관성 | 매우 직관적 | 비교적 직관적 |

### 예시

샘플 5개의 잔차가 다음과 같다고 해 봐요.
- 잔차들: 1, 2, 3, 4, 50

```
MAE  = (1+2+3+4+50) / 5 = 12
RMSE = √((1+4+9+16+2500)/5) = √506 = 22.5
```

같은 데이터인데 RMSE가 거의 2배예요. **이상치(50) 하나가 RMSE를 크게 끌어올렸어요.**

### 언제 어떤 걸?

- **이상치를 무겁게 봐야 할 때** → RMSE
  - 큰 실수 한 번이 작은 실수 여러 번보다 치명적인 경우
  - 예: 의료 진단 (큰 오류는 사람 죽음)

- **이상치를 평등하게 봐야 할 때** → MAE
  - 모든 오차가 동등하게 중요한 경우
  - 예: 일반 수요 예측 (가끔 큰 실수는 어쩔 수 없음)

---

## 4. R² (R-squared, 결정계수)

이건 좀 다른 종류의 지표예요. **0~1 사이의 비율**로 나옵니다.

```
R² = 1 - (모델의 잔차 제곱합) / (평균 예측의 잔차 제곱합)
```

수식으로 짧게 쓰면:

```
        Σ (yᵢ - ŷᵢ)²
R² = 1 - ─────────────
        Σ (yᵢ - ȳ)²
```

`ȳ` (와이바)는 **y의 평균** 이라는 뜻이에요. 분자는 **우리 모델의 잔차 제곱합**, 분모는 **"평균값으로만 예측했을 때"의 잔차 제곱합**.

복잡해 보이지만 의미는 단순해요.

> **"가장 단순한 예측(평균값으로만 예측)에 비해, 우리 모델이 얼마나 더 잘 하는가?"**

분자/분모가 같으면 R²=0 (모델이 평균이랑 똑같이 못함). 분자가 0이면 R²=1 (완벽). 분자가 분모보다 크면 R²가 음수 (평균보다 못함, 거의 사고 수준).

### 해석

- **R² = 1.0**: 완벽한 예측
- **R² = 0.0**: 평균값으로 예측한 거랑 똑같음 (모델 의미 없음)
- **R² < 0**: 평균보다도 못함 (드물지만 가능)

일반적으로:
- R² > 0.9: 매우 좋음
- R² > 0.7: 좋음
- R² > 0.5: 보통
- R² < 0.3: 안 좋음

### 코드

```python
from sklearn.metrics import r2_score
r2 = r2_score(y_test, y_pred)
```

### 장점

- **단위 무관** — 비율로 나오니까 도메인이 달라도 비교 가능
- **직관적** — "모델이 데이터의 X%를 설명함"
- **모델 비교에 좋음**

### 한계

- **변수 추가하면 항상 R²가 늘어남** (의미 없는 변수도)
- 그래서 **Adjusted R²** 라는 변형도 있음

## 4가지 지표 한 번에 보기

```python
import numpy as np
from sklearn.metrics import (
    mean_squared_error,
    mean_absolute_error,
    r2_score,
)

mse = mean_squared_error(y_test, y_pred)
rmse = np.sqrt(mse)
mae = mean_absolute_error(y_test, y_pred)
r2 = r2_score(y_test, y_pred)

print(f"MSE:  {mse:.4f}")
print(f"RMSE: {rmse:.4f}")
print(f"MAE:  {mae:.4f}")
print(f"R²:   {r2:.4f}")
```

```
MSE:  24.29
RMSE: 4.93
MAE:  3.18
R²:   0.6688
```

### 어떻게 해석하나요?

- **RMSE 4.93**: "평균적으로 ~$4,930만큼 틀림"
- **MAE 3.18**: "보통은 ~$3,180만큼 틀림"
- **MAE < RMSE**: 일부 큰 오차가 있음 (이상치)
- **R² 0.67**: "데이터 변동의 67%를 설명"

→ **나쁘지 않은 모델, 하지만 이상치 처리 개선 여지가 있음**

## 어느 지표를 봐야 하나요?

상황별 권장 지표를 정리하면:

| 상황 | 주요 지표 |
|------|---------|
| 일반 회귀 (모델 선택) | **RMSE** |
| 결과를 사람에게 설명할 때 | **MAE** (가장 직관적) |
| 모델 비교 (단위 다른 경우) | **R²** |
| 이상치를 절대 놓치면 안 됨 | **RMSE** |
| 이상치를 가볍게 취급 | **MAE** |
| 비즈니스 KPI에 포함 | **MAPE** (다음에 설명) |

---

## 보너스: 다른 지표들

알아두시면 가끔 유용한 지표들.

### MAPE (Mean Absolute Percentage Error)

```
MAPE = 평균(|잔차| / 실제값) × 100
```

**오차를 비율(%)로 표현**하는 지표예요.

```python
import numpy as np

mape = np.mean(np.abs((y_test - y_pred) / y_test)) * 100
print(f"MAPE: {mape:.2f}%")
```

비즈니스 보고서에 자주 써요. "평균 5% 정도 틀렸어요" 하면 임원들이 잘 이해하시거든요.

⚠️ **단점**: 실제값이 0이거나 매우 작으면 폭발해요. 0이 있는 데이터에선 못 씀.

### Median Absolute Error

평균 대신 중앙값을 쓰는 MAE예요. 이상치에 더 강해요.

```python
from sklearn.metrics import median_absolute_error
medae = median_absolute_error(y_test, y_pred)
```

### Max Error

가장 큰 단일 오차. "최악의 경우" 보고할 때.

```python
from sklearn.metrics import max_error
me = max_error(y_test, y_pred)
```

## 정리

```python
# 가장 자주 보는 4종 세트
import numpy as np
from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score

rmse = np.sqrt(mean_squared_error(y_test, y_pred))
mae = mean_absolute_error(y_test, y_pred)
r2 = r2_score(y_test, y_pred)

print(f"RMSE: {rmse:.4f}    ← 일반적으로 가장 많이 봄")
print(f"MAE:  {mae:.4f}    ← 직관적, 이상치에 강함")
print(f"R²:   {r2:.4f}     ← 0~1 비율, 모델 비교에 좋음")
```

**최소 권장: RMSE + MAE 같이 보세요.** 둘 차이가 크면 이상치 문제가 있는 것.

## 자주 하는 오해와 바로잡기

### 1) "이론만 이해하면 실습은 저절로 된다"
아닙니다. 이론 이해와 코드 실행 능력은 별개의 근육입니다. 둘을 같이 써야 합니다.

### 2) "예제 코드가 돌아갔으니 완전히 이해했다"
동작 확인은 시작일 뿐입니다. 파라미터를 바꿨을 때 결과가 어떻게 달라지는지까지 확인해야 이해가 완성됩니다.

### 3) "좋은 모델은 항상 하나다"
데이터 특성과 제약(시간, 메모리, 응답 속도)에 따라 최선의 선택은 달라집니다.

### 4) "지표 숫자만 높으면 끝이다"
지표는 해석이 필요합니다. 어떤 데이터에서, 어떤 분포에서, 어떤 기준으로 얻은 점수인지 함께 보아야 합니다.

### 5) "지금은 초반이니까 배포는 나중에 생각해도 된다"
초반부터 재현성과 저장 구조를 습관화해야 나중에 배포 단계에서 무너지지 않습니다.

복잡한 모델보다 먼저, 빠르게 재현 가능한 기본 모델을 세웁니다.

4. 한 번에 하나씩 개선한다.  
전처리, 모델, 튜닝, 임계값, 배치 크기 등은 동시에 바꾸지 않고 순차적으로 바꿉니다.

5. 결과를 로그와 함께 저장한다.  
실험 이름, 파라미터, 점수, 데이터 버전을 함께 남겨야 재현이 됩니다.

6. 실패 케이스를 분석한다.  
점수 평균보다 오분류/고오차 샘플을 분석할 때 개선 아이디어가 나옵니다.

7. 배포 가능 형태로 정리한다.  
모델 파일, 메타데이터, 입력 스키마, 테스트 스크립트까지 묶습니다.

## 복습 체크리스트

다음 질문에 답할 수 있으면 이 단원을 실전 수준으로 이해한 것입니다.

- 이 단원의 핵심 개념을 중학생에게 설명하듯 말할 수 있는가?
- 코드의 각 블록이 왜 필요한지 한 줄씩 설명할 수 있는가?

- 실패 사례를 보고 다음 실험 가설을 만들 수 있는가?
- 이 단원 내용을 다음 장과 어떻게 연결할지 설명할 수 있는가?

## 확장 연습 과제

### 과제 A: 파라미터 감각 만들기
현재 예제에서 핵심 파라미터 1~2개만 바꿔 5회 실험하고, 결과를 표로 정리해 보세요.

### 과제 B: 실패 사례 분석
오분류 또는 오차가 큰 샘플 20개를 모아 공통 패턴을 찾아 보세요.

### 과제 C: 설명 가능 리포트 작성
"무엇을 했고, 왜 했고, 결과가 어땠고, 다음엔 무엇을 할지"를 1페이지로 정리해 보세요.

## 한 번 더 정리

3.4 회귀 평가 지표는 단순히 시험용 개념이 아니라, 실제 문제를 풀 때 반복해서 쓰게 되는 실전 도구입니다. 처음에는 낯설 수 있지만, 작은 실험을 반복하면 감각이 생기고, 감각이 생기면 코드와 지표를 읽는 속도가 급격히 빨라집니다. 지금 단계에서 중요한 것은 완벽한 암기가 아니라, **작게 실행하고, 결과를 읽고, 다시 수정하는 루프**를 몸에 익히는 것입니다. 이 루프를 익히면 다음 장으로 넘어갈수록 학습 속도가 오히려 빨라집니다.
