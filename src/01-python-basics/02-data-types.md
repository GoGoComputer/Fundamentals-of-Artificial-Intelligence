# 1.2 자료형: 숫자, 문자, 참/거짓

## 자료형이 뭐예요?

지난 글에서 변수에 값을 담는 법을 배웠죠.

```python
name = "철수"
age = 25
height = 175.5
```

여기서 `"철수"`, `25`, `175.5`는 다 다른 종류의 값이에요. 이걸 **자료형(data type)** 이라고 부릅니다. 컴퓨터는 자료형을 알아야 그 값을 어떻게 다룰지 알 수 있어요.

> **비유:** 식당에서 "음식 주세요" 하고 주문하시면 음식점이 당황합니다. "한식 주세요", "중식 주세요" 처럼 종류를 알려줘야 해요. 컴퓨터도 마찬가지예요.

이 글에서는 파이썬의 **5가지 기본 자료형**을 배웁니다.

| 이름 | 영어 | 예시 | 무엇? |
|------|------|------|------|
| 정수 | `int` | `25`, `-7`, `0` | 소수점 없는 숫자 |
| 실수 | `float` | `3.14`, `-0.5`, `2.0` | 소수점 있는 숫자 |
| 문자열 | `str` | `"hello"`, `'안녕'` | 글자들의 모음 |
| 참거짓 | `bool` | `True`, `False` | 둘 중 하나 |
| 없음 | `NoneType` | `None` | "값이 없음"을 나타냄 |

하나씩 봅시다.

---

## 1. 정수 (int)

소수점 없는 숫자입니다. 양수, 음수, 0 다 됩니다.

```python
age = 25
year = 2026
debt = -50000
zero = 0
```

크기 제한이 없어요. 정말 큰 수도 다룰 수 있습니다.

```python
huge = 99999999999999999999999999999999999999
print(huge)
# 99999999999999999999999999999999999999
```

---

## 2. 실수 (float)

소수점이 있는 숫자입니다.

```python
pi = 3.14
height = 175.5
temperature = -0.5
```

소수점이 있으면 `.0`만 붙어도 실수예요.

```python
x = 2       # int (정수)
y = 2.0     # float (실수)
print(x, y)
```

### 정수 vs 실수, 왜 구분해요?

컴퓨터 내부에서 처리 방식이 달라요. 정수는 빠르고 정확하고, 실수는 좀 더 느리고 가끔 정확하지 않아요.

```python
print(0.1 + 0.2)
# 0.30000000000000004    ← !
```

엥? `0.3`이 나와야 할 것 같은데 이상한 숫자가 나오죠. 이건 컴퓨터가 실수를 표현하는 방식의 한계예요(2진법으로 0.1을 정확히 표현할 수가 없거든요). 너무 깊이 들어가지는 말고, **"실수 계산은 가끔 살짝 부정확할 수 있다"** 정도만 머리에 두세요.

머신러닝에서는 실수를 정말 많이 다룹니다. 모델의 가중치, 학습률, 확률값, 손실값 — 거의 다 실수예요.

---

## 3. 문자열 (str)

글자들의 모음입니다. 따옴표로 감싸요.

```python
name = "철수"
greeting = '안녕하세요'
sentence = "오늘은 2026년 5월 3일입니다."
```

따옴표는 큰 따옴표(`"`)와 작은 따옴표(`'`) 둘 다 됩니다. (지난 글에서 다뤘죠.)

### 여러 줄 문자열

따옴표 세 개로 감싸면 줄바꿈을 그대로 살릴 수 있어요.

```python
poem = """
하늘이 매우 푸르다
오늘은 좋은 날이다
"""
print(poem)
```

```

하늘이 매우 푸르다
오늘은 좋은 날이다

```

### 문자열 합치기 (+)

```python
first = "김"
last = "철수"
full = first + last
print(full)        # 김철수
```

### 문자열 반복 (*)

```python
print("야호! " * 3)
# 야호! 야호! 야호! 
```

신기하죠? 이게 처음에는 쓸데없어 보이는데 의외로 자주 씁니다.

```python
print("=" * 30)
# ==============================
```

구분선 그릴 때 좋아요.

### 문자열 길이 (len)

`len()` 함수는 문자열의 글자 수를 알려줘요.

```python
name = "Hello"
print(len(name))    # 5

korean = "안녕하세요"
print(len(korean))  # 5  (한글도 한 글자씩 셈)
```

### 문자열 안에 변수 넣기 (f-string)

이건 정말 자주 씁니다. 외워두세요.

```python
name = "철수"
age = 25
print(f"제 이름은 {name}이고, 나이는 {age}살입니다.")
# 제 이름은 철수이고, 나이는 25살입니다.
```

문자열 앞에 **`f`** 를 붙이고, 변수를 **`{ }`** 로 감싸면 변수 값이 그 자리에 들어가요. 이걸 **f-string**이라고 부릅니다 (formatted string의 줄임).

f-string 안에서 계산도 할 수 있어요.

```python
print(f"5년 후엔 {age + 5}살이 되겠네요.")
# 5년 후엔 30살이 되겠네요.
```

### 옛날 방식들 (참고만)

f-string이 나오기 전에는 이렇게 썼어요. 옛날 코드를 보실 수도 있으니 알아만 두세요.

```python
# 1. % 방식 (가장 오래됨)
print("이름은 %s, 나이는 %d" % (name, age))

# 2. .format() 방식
print("이름은 {}, 나이는 {}".format(name, age))

# 3. f-string (요즘 표준, 권장)
print(f"이름은 {name}, 나이는 {age}")
```

새 코드 쓰실 땐 무조건 **f-string**을 쓰세요.

## 4. 참거짓 (bool)

`True` 와 `False` 두 가지밖에 없는 자료형이에요.

```python
is_student = True
is_adult = False
```

⚠️ **주의:** 첫 글자가 대문자입니다. `true`, `false`는 안 돼요.

### 어디에 써요?

조건이 맞는지 안 맞는지를 표현할 때 씁니다.

```python
age = 25
is_adult = age >= 20
print(is_adult)    # True
```

`age >= 20`이라는 비교의 결과가 `True` 또는 `False`로 나와요. 다음 글의 비교 연산자에서 자세히 봅니다.

### bool은 사실 숫자입니다

이건 알아두면 가끔 편리한 트릭이에요. 파이썬에서 `True`는 1, `False`는 0으로 취급됩니다.

```python
print(True + True)        # 2
print(True + False)       # 1
print(True * 100)         # 100
```

머신러닝에서 "정답을 맞힌 개수"를 셀 때 이 트릭이 자주 쓰여요.

```python
correct = [True, True, False, True, False]
total_correct = sum(correct)
print(total_correct)    # 3
```

5문제 중 3개 맞혔다는 뜻이죠.

## 5. None: 값이 "없음"

`None`은 **"아무 값도 없다"** 라는 뜻의 특별한 값이에요.

```python
result = None
print(result)    # None
```

처음엔 "왜 이런 게 필요해?" 싶을 수 있어요. 예를 들어, 함수가 아직 결과를 안 만들었을 때, "결과가 0인 것"과 "아직 결과가 없는 것"을 구분해야 할 때가 있어요. 그런 상황에 씁니다.

```python
best_score = None    # 아직 시험을 안 봤음

# 시험 보고 나서
best_score = 95
```

머신러닝에서는 "아직 모델이 학습되지 않은 상태"를 표현할 때 이런 식으로 자주 씁니다.

## 자료형 확인하기: type()

값의 자료형이 무엇인지 알고 싶으면 `type()` 함수를 쓰세요.

```python
print(type(25))         # <class 'int'>
print(type(3.14))       # <class 'float'>
print(type("hello"))    # <class 'str'>
print(type(True))       # <class 'bool'>
print(type(None))       # <class 'NoneType'>
```

`<class 'int'>` 같은 결과가 좀 어색해 보이지만, **"이 값은 int 종류구나"** 라고 읽으시면 됩니다.

---

## 자료형 변환

자료형을 다른 자료형으로 바꿀 수도 있어요.

### 숫자 ↔ 문자열

가장 자주 쓰는 변환입니다.

```python
# 숫자 → 문자
age = 25
text = str(age)
print(text)         # "25"
print(type(text))   # <class 'str'>

# 문자 → 숫자
year = "2026"
num = int(year)
print(num)          # 2026
print(type(num))    # <class 'int'>
```

### 정수 ↔ 실수

```python
print(int(3.7))     # 3   (소수점 버림. 반올림 X !)
print(int(3.2))     # 3
print(int(-3.7))    # -3

print(float(5))     # 5.0
```

⚠️ 주의: `int()`는 **반올림이 아니라 소수점을 버립니다.** 반올림하려면 `round()`를 쓰세요.

```python
print(round(3.7))   # 4
print(round(3.2))   # 3
```

### 변환이 안 되는 경우

```python
int("hello")   # ValueError !
```

`"hello"`는 숫자로 바꿀 수가 없어요. 에러가 납니다. 사용자가 입력한 값을 숫자로 바꿀 때 이런 일이 자주 생기니 조심하세요.

### 왜 변환이 필요해요?

예를 들어, 사용자에게 입력을 받으면 그건 항상 **문자열**로 들어와요.

```python
year = input("태어난 연도를 입력하세요: ")
# 사용자가 1995 입력
print(year + 30)   # ❌ 에러! "1995" + 30은 불가능
print(int(year) + 30)   # ✅ 2025
```

머신러닝에서는 데이터를 불러올 때 자료형이 안 맞아서 변환해야 할 때가 정말 많아요. 익숙해지셔야 합니다.

## 정리: 한 페이지 요약

```python
# 5가지 자료형
age = 25                    # int (정수)
height = 175.5              # float (실수)
name = "철수"                # str (문자열)
is_adult = True             # bool (참거짓)
result = None               # NoneType (없음)

# 자료형 확인
type(age)          # <class 'int'>

# 자료형 변환
str(25)            # "25"
int("25")          # 25
float(5)           # 5.0
int(3.7)           # 3   (소수점 버림)
round(3.7)         # 4   (반올림)

# 문자열 합치기, 반복, 길이
"hello" + "world"  # "helloworld"
"하" * 3           # "하하하"
len("hello")       # 5

# f-string (외워두세요)
f"안녕 {name}, 나이 {age}"
```

## 자주 묻는 질문

> **Q. 자료형이 그렇게 중요한가요?**
>
> 네, 정말 중요합니다. 머신러닝 코드에서 일어나는 에러의 절반쯤은 자료형 문제예요. "이건 분명히 숫자인 줄 알았는데 알고 보니 문자열"이라거나, "정수가 와야 하는데 실수가 왔다"거나. `type()`으로 확인하는 습관을 들이세요.

> **Q. f-string에서 소수점 자릿수 조절할 수 있나요?**
>
> 됩니다. 자주 씁니다.
> ```python
> pi = 3.14159265
> print(f"{pi:.2f}")    # 3.14   (소수점 둘째 자리까지)
> print(f"{pi:.4f}")    # 3.1416
> ```
> 머신러닝에서 정확도를 보여줄 때 정말 자주 써요. 외워두세요.

> **Q. `True`와 `1`이 진짜로 같은 거예요?**
>
> 같다고 봐도 됩니다. `True == 1`은 `True`입니다. 다만 모양만 다른 거예요. 그래서 `True`를 숫자처럼 쓸 수 있는 거고요.
