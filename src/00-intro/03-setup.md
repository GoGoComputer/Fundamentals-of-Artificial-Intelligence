# 실습 환경 만들기

## 좋은 소식 하나 드릴게요

머신러닝을 배우려면 보통 이런 준비가 필요했습니다.

- 파이썬 설치 (버전 안 맞으면 곤란)
- 가상환경 설정 (venv, conda, ...)
- 라이브러리 설치 (numpy, pandas, scikit-learn, pytorch...)
- 의존성 충돌 해결
- GPU를 쓰려면 CUDA 설치
- 그러다 컴퓨터 한 번 망가뜨리고
- 다시 설치
- ...

그런데 이 모든 걸 **0초 만에 건너뛰는 방법**이 있습니다. **Google Colab**이에요.

## Google Colab이 뭔가요?

브라우저에서 파이썬을 실행할 수 있게 해 주는 구글의 서비스입니다. 핵심 특징은 이거예요.

- 🆓 **공짜**입니다 (유료 버전도 있지만 이 자료에서는 공짜로 충분)
- 🌐 **설치할 게 없습니다.** 브라우저만 있으면 끝.
- ⚡ **GPU도 공짜로 줍니다.** 무한정은 아니지만 학습 용도로는 충분.
- 📦 **주요 라이브러리가 이미 다 깔려 있어요.** numpy, pandas, scikit-learn, PyTorch, TensorFlow 등.

회사에서 진지하게 모델을 만드실 땐 결국 본인 컴퓨터/서버에 환경을 직접 구축하셔야 하지만, **배우는 단계에서는 Colab이 정답입니다.** 99%의 머신러닝 입문자가 Colab으로 시작하고, 또 그게 맞는 선택이에요.

## 5분 안에 첫 노트북 열기

### 1단계: 구글 계정 준비

### 2단계: Colab 열기

[https://colab.research.google.com](https://colab.research.google.com) 으로 가시거나, 구글 검색창에 "Colab"이라고 치셔도 됩니다.

처음 들어가시면 이런 화면이 보일 거예요.

```
┌──────────────────────────────────────┐
```

오른쪽 아래 또는 가운데의 **"새 노트(New notebook)"** 를 클릭하세요.

새 노트가 열리면 회색 박스(셀, cell이라고 부릅니다)가 하나 보일 거예요. 거기에 이렇게 입력해 보세요.


그리고 **`Shift + Enter`** 를 누르세요. (또는 셀 왼쪽의 ▶ 버튼을 눌러도 됩니다.)

처음 실행하실 땐 몇 초 걸립니다. 클라우드의 컴퓨터 한 대가 여러분을 위해 깨어나는 시간이거든요.

```

```

이 메시지가 셀 아래에 보이면 **성공입니다.** 진심으로 축하드립니다. 여러분은 방금 클라우드의 컴퓨터에 명령을 내렸어요.

## Colab의 기본 사용법

### 셀 두 종류

Colab 노트북은 두 종류의 셀로 이루어집니다.

| 종류 | 무엇? | 추가 단축키 |

| **코드 셀 (Code)** | 파이썬 코드를 실행 | `Ctrl + M, B`로 아래에 코드 셀 추가 |
| **텍스트 셀 (Text)** | 마크다운으로 설명을 적기 | `Ctrl + M, M`으로 아래에 텍스트 셀 추가 |

### 자주 쓰는 단축키

| 단축키 | 무엇? |

| `Alt + Enter` (맥은 `Option + Enter`) | 셀 실행 + 아래에 새 셀 |

### 노트북 저장하기

Colab 노트북은 자동으로 구글 드라이브에 저장됩니다. 위쪽 메뉴에서 `파일 → 저장`을 누르거나 `Ctrl + S`를 누르세요.

저장된 노트북은 [drive.google.com](https://drive.google.com) 에서 "Colab Notebooks" 폴더 안에 들어 있어요.

## GPU 켜는 법 (4장에서 필요합니다)

머신러닝 챕터(2, 3장)는 CPU만으로 충분하지만, 딥러닝 챕터(4, 5장)는 GPU가 있으면 학습 시간이 10배쯤 빨라집니다. Colab에서 GPU를 켜는 방법은 정말 간단해요.

1. 위쪽 메뉴에서 **`런타임(Runtime) → 런타임 유형 변경(Change runtime type)`** 클릭
2. **하드웨어 가속기**를 **"GPU"** 또는 **"T4 GPU"** 로 변경
3. **저장** 클릭

GPU가 잘 잡혔는지 확인하시려면 셀에 이렇게 적고 실행해 보세요.

```python
import torch
print("GPU 쓸 수 있나요?", torch.cuda.is_available())
print("어떤 GPU?", torch.cuda.get_device_name(0) if torch.cuda.is_available() else "없음")
```

이런 결과가 나오면 OK입니다.

```
GPU 쓸 수 있나요? True
어떤 GPU? Tesla T4
```

⚠️ **주의:** GPU를 쓰면 무료 사용량이 빨리 소진됩니다. 쓰지 않을 땐 `런타임 → 런타임 유형 변경`에서 다시 "CPU"로 돌려놓으시는 게 좋아요. 4장 들어가시기 전까지는 굳이 GPU 안 켜셔도 됩니다.

## (선택) 본인 컴퓨터에 환경 만들고 싶으신 분

회사에서 진지하게 ML을 하실 거라면, 결국 본인 컴퓨터에 환경을 만드실 일이 옵니다. 그때 권장하는 방법은 이거예요.

### 권장 스택 (2026년 기준)

```bash
# 1. uv 설치 (가장 빠른 파이썬 패키지 관리자)
curl -LsSf https://astral.sh/uv/install.sh | sh

# 2. 프로젝트 디렉토리 만들기
mkdir my-ml-project
cd my-ml-project

# 3. 파이썬 가상환경 + 의존성 한 방에
uv init
uv add numpy pandas scikit-learn matplotlib seaborn
uv add torch torchvision

# 4. Jupyter 노트북 환경
uv add jupyter
uv run jupyter notebook
```

`uv`는 파이썬 환경을 다루는 도구 중에 가장 빠르고 깔끔합니다. 옛날에 `pip + venv` 또는 `conda` 쓰셨던 분이라면 `uv`로 갈아타시는 걸 권합니다.

하지만 다시 한 번, **이 자료를 따라가는 데에는 Colab만 있으면 충분합니다.** 본인 컴퓨터 환경 구축은 나중에 필요할 때 하셔도 늦지 않아요.

## 자주 막히는 지점

### "GPU가 안 잡혀요"
- 무료 사용량이 일시적으로 소진된 것일 수 있어요. 몇 시간 후에 다시 시도하시거나, 그동안 CPU로 진행하세요.
- `런타임 → 세션 관리(Manage sessions)`에서 안 쓰는 세션을 모두 종료해 보세요.

### "셀 실행이 너무 오래 걸려요"
- 처음 실행하실 땐 1~2분 걸리는 게 정상입니다. 클라우드 컴퓨터가 깨어나는 시간이에요.
- 그래도 너무 오래 걸리면 `런타임 → 런타임 다시 시작`을 해 보세요.

### "모듈을 못 찾는다는 에러가 떠요"
- Colab에 기본으로 안 깔린 라이브러리는 직접 설치하셔야 해요. 셀에 이렇게 입력하시면 됩니다.

```python
!pip install torchinfo
```

`!` 로 시작하는 줄은 셸 명령어입니다. 노트북 안에서 터미널 명령을 쓸 수 있는 거예요.

## 헷갈리기 쉬운 포인트

환경 설정 글에서 자주 놓치는 핵심은 "완벽한 환경"이 아니라 "지속 가능한 실행 환경"입니다. 입문 단계에서는 속도와 안정성이 더 중요합니다.

- Colab의 목적은 설치 지식 과시가 아니라 학습 마찰 제거입니다.
- GPU는 항상 좋은 선택이 아닙니다. 필요 없는 단계에서 켜면 오히려 세션 제한과 대기 시간이 늘어납니다.
- 실행 성공 여부는 코드 난이도보다 커널 상태(재시작 필요, 패키지 설치 상태)의 영향을 자주 받습니다.
- 환경 문제를 해결하는 능력도 실무 역량입니다. 에러를 "내가 못해서"로 해석하지 마세요.

## 실전에서 보는 패턴

학습이 잘 이어지는 사람들은 환경을 다음처럼 관리합니다.

1. 시작 3분 점검: 런타임 타입, 커널 상태, 파일 저장 위치를 먼저 확인합니다.
2. 설치 로그 기록: `pip install`을 했다면 버전과 설치 이유를 노트 상단에 적습니다.
3. 세션 끊김 대비: 중요한 결과는 이미지/CSV/모델 파일로 즉시 저장합니다.
4. 재현 루틴 확보: "이 노트를 새 세션에서 다시 실행해도 되는가"를 주기적으로 테스트합니다.

## 자주 하는 실수와 해결

- 실수: 패키지 설치 셀을 중복 실행해 충돌을 만든다.
	해결: 설치 셀을 한 구역으로 모으고, 버전을 명시해 일관성을 유지합니다.
- 실수: GPU가 안 잡히면 학습 자체를 중단한다.
	해결: CPU로 가능한 범위(데이터 축소, epoch 감소)로 먼저 학습 흐름을 이어갑니다.
- 실수: 노트북 저장을 미루다가 세션이 끊긴다.
	해결: 단계 완료마다 자동 저장 여부와 출력 파일 존재를 확인합니다.
- 실수: 환경 오류를 무작정 재시작으로만 해결한다.
	해결: 오류 메시지와 마지막 변경점(설치/런타임 변경)을 먼저 기록한 후 조치합니다.

## 이해를 더 단단하게 만드는 보강 설명

이 단원은 처음 읽을 때는 "아, 이런 개념이 있구나" 정도로 넘어가기 쉽지만, 실제로는 작은 차이가 나중에 큰 성능 차이로 이어집니다. 특히 **실습 환경 만들기** 같은 주제는 한 번에 완벽히 이해하려고 하기보다, 다음 세 가지를 반복하면서 이해를 깊게 만드는 편이 훨씬 효과적입니다.

첫째, 용어를 정의로 외우지 말고 상황으로 연결해 보세요. 예를 들어 "왜 이 개념이 필요한가"를 스스로 설명할 수 있어야 다음 단원에서 응용이 됩니다. 둘째, 코드가 나오면 결과를 먼저 예측한 뒤 실행해 보세요. 예측과 결과가 다를 때가 진짜 학습이 일어나는 순간입니다. 셋째, 같은 코드를 숫자나 파라미터만 바꿔 여러 번 돌려 보세요. 머신러닝과 딥러닝은 정답을 암기하는 과목이 아니라 **변화를 읽는 과목**이기 때문입니다.

또 하나 중요한 점은, 이 단원의 내용이 단독으로 존재하지 않는다는 사실입니다. 실제 프로젝트에서는 데이터 준비, 모델링, 평가, 배포가 끊기지 않고 이어집니다. 그래서 지금 배우는 **실습 환경 만들기**도 나중에 다른 단계와 반드시 연결됩니다. 이 연결을 의식하면서 읽으면, 단편 지식이 아니라 시스템 관점으로 이해하게 됩니다.

## 자주 하는 오해와 바로잡기

### 1) "이론만 이해하면 실습은 저절로 된다"
아닙니다. 이론 이해와 코드 실행 능력은 별개의 근육입니다. 둘을 같이 써야 합니다.

### 2) "예제 코드가 돌아갔으니 완전히 이해했다"
동작 확인은 시작일 뿐입니다. 파라미터를 바꿨을 때 결과가 어떻게 달라지는지까지 확인해야 이해가 완성됩니다.

### 3) "좋은 모델은 항상 하나다"
데이터 특성과 제약(시간, 메모리, 응답 속도)에 따라 최선의 선택은 달라집니다.

### 4) "지표 숫자만 높으면 끝이다"
지표는 해석이 필요합니다. 어떤 데이터에서, 어떤 분포에서, 어떤 기준으로 얻은 점수인지 함께 보아야 합니다.

### 5) "지금은 초반이니까 배포는 나중에 생각해도 된다"
초반부터 재현성과 저장 구조를 습관화해야 나중에 배포 단계에서 무너지지 않습니다.

## 실전 연결 시나리오

아래는 이 단원 내용을 실제 업무로 연결할 때 자주 쓰는 흐름입니다.

1. 문제를 한 문장으로 정의한다.  
예: "실습 환경 만들기 관점에서 입력으로부터 어떤 출력을 만들 것인가?"

2. 성공 기준을 지표로 정한다.  
예: 정확도/RMSE/F1/지연시간 같은 운영 가능한 숫자로 정의.

3. 가장 단순한 베이스라인을 만든다.  
복잡한 모델보다 먼저, 빠르게 재현 가능한 기본 모델을 세웁니다.

4. 한 번에 하나씩 개선한다.  
전처리, 모델, 튜닝, 임계값, 배치 크기 등은 동시에 바꾸지 않고 순차적으로 바꿉니다.

5. 결과를 로그와 함께 저장한다.  
실험 이름, 파라미터, 점수, 데이터 버전을 함께 남겨야 재현이 됩니다.

6. 실패 케이스를 분석한다.  
점수 평균보다 오분류/고오차 샘플을 분석할 때 개선 아이디어가 나옵니다.

7. 배포 가능 형태로 정리한다.  
모델 파일, 메타데이터, 입력 스키마, 테스트 스크립트까지 묶습니다.

## 복습 체크리스트

다음 질문에 답할 수 있으면 이 단원을 실전 수준으로 이해한 것입니다.

- 이 단원의 핵심 개념을 중학생에게 설명하듯 말할 수 있는가?
- 코드의 각 블록이 왜 필요한지 한 줄씩 설명할 수 있는가?
- 이 단원에서 가장 자주 틀리는 지점을 알고 있는가?
- 지표를 볼 때 어떤 함정이 있는지 알고 있는가?
- 베이스라인과 개선 모델의 차이를 표로 정리할 수 있는가?
- 재현 가능하게 실험을 저장하는 습관이 있는가?
- 실패 사례를 보고 다음 실험 가설을 만들 수 있는가?
- 이 단원 내용을 다음 장과 어떻게 연결할지 설명할 수 있는가?

## 확장 연습 과제

### 과제 A: 파라미터 감각 만들기
현재 예제에서 핵심 파라미터 1~2개만 바꿔 5회 실험하고, 결과를 표로 정리해 보세요.

### 과제 B: 실패 사례 분석
오분류 또는 오차가 큰 샘플 20개를 모아 공통 패턴을 찾아 보세요.

### 과제 C: 설명 가능 리포트 작성
"무엇을 했고, 왜 했고, 결과가 어땠고, 다음엔 무엇을 할지"를 1페이지로 정리해 보세요.

이 3개를 직접 해 보면 **실습 환경 만들기**의 이해도는 읽기만 했을 때보다 훨씬 빠르게 올라갑니다.

## 한 번 더 정리

실습 환경 만들기는 단순히 시험용 개념이 아니라, 실제 문제를 풀 때 반복해서 쓰게 되는 실전 도구입니다. 처음에는 낯설 수 있지만, 작은 실험을 반복하면 감각이 생기고, 감각이 생기면 코드와 지표를 읽는 속도가 급격히 빨라집니다. 지금 단계에서 중요한 것은 완벽한 암기가 아니라, **작게 실행하고, 결과를 읽고, 다시 수정하는 루프**를 몸에 익히는 것입니다. 이 루프를 익히면 다음 장으로 넘어갈수록 학습 속도가 오히려 빨라집니다.
