# 4. 선형대수 입문

> 신경망의 모든 연산이 사실은 행렬 곱이에요. 이 장을 끝내면 `Wx + b` 같은 식이 익숙해집니다.

---

## 4.1 벡터 (Vector)

### 직관

**숫자 여러 개를 한 묶음으로** 다루는 것.

```
v = (1, 2, 3)
```

3개의 숫자를 묶었어요. 이걸 **3차원 벡터**라고 해요.

### 그림으로

2차원 벡터 `v = (3, 2)` 는 화살표예요.

```
       y
       ↑
    2  ┤────────●  v
       │       /
    1  ┤      /
       │     /
    0  ●───/───────→ x
       0   1   2   3
       원점에서 (3,2)로 가는 화살표
```

### 표기법

```
열벡터 (column vector):       행벡터 (row vector):
v = [1]                        v = [1 2 3]
    [2]
    [3]
```

수학에서는 보통 **열벡터**가 표준. ML 수식에서도 그래요.

### Python에서

```python
import numpy as np
v = np.array([1, 2, 3])
print(v.shape)    # (3,)   3차원 벡터
```

### ML에서

- **데이터 한 샘플** = 벡터 (예: 이미지 한 장의 모든 픽셀 값)
- **모델 출력** = 벡터 (예: 10개 클래스의 점수)
- **모델 가중치** = 벡터 (선형 모델의 경우)

MNIST 이미지 한 장은 28×28 = **784차원 벡터**예요.

---

## 4.2 벡터 연산

### 덧셈 (같은 위치끼리)

```
[1]   [4]   [5]
[2] + [5] = [7]
[3]   [6]   [9]
```

```python
v1 = np.array([1, 2, 3])
v2 = np.array([4, 5, 6])
print(v1 + v2)    # [5, 7, 9]
```

### 스칼라 곱 (모든 원소에 같은 수 곱)

```
    [1]   [2]
2 × [2] = [4]
    [3]   [6]
```

```python
print(2 * v1)    # [2, 4, 6]
```

### 내적 (Dot Product) — 매우 중요!

ML에서 정말 자주 만나요.

```
v · w = v₁×w₁ + v₂×w₂ + v₃×w₃ + ...

예:
[1] · [4]
[2]   [5]   = 1×4 + 2×5 + 3×6 = 4 + 10 + 18 = 32
[3]   [6]
```

**같은 위치끼리 곱해서 다 더함.** 결과는 숫자 하나(스칼라).

```python
v1 = np.array([1, 2, 3])
v2 = np.array([4, 5, 6])
print(np.dot(v1, v2))    # 32
print(v1 @ v2)           # 32 (같은 결과, @ 기호)
```

### 내적이 ML에 왜 중요?

선형 모델의 핵심 연산이 내적이에요.

```
선형 모델: y = w₁x₁ + w₂x₂ + ... + wₙxₙ + b
              ↑
           이게 내적!

→ y = w · x + b
```

훨씬 짧고 깔끔하죠. 모든 ML 모델 식이 이런 식으로 표기됩니다.

---

## 4.3 행렬 (Matrix)

### 직관

**숫자를 표(2차원)로 정리한 것.**

```
A = [1 2 3]
    [4 5 6]
```

이건 **2행 3열 (2×3) 행렬**이에요. 2개의 행, 3개의 열.

표기법: `Aᵢⱼ` 는 **i번째 행, j번째 열의 원소**.

```
A₁₁ = 1, A₁₂ = 2, A₁₃ = 3
A₂₁ = 4, A₂₂ = 5, A₂₃ = 6
```

### Python에서

```python
A = np.array([[1, 2, 3],
              [4, 5, 6]])
print(A.shape)    # (2, 3)
```

### 모양(shape)이 정말 중요

행렬을 다룰 때 가장 자주 만나는 에러가 **shape 안 맞음**이에요. 항상 모양을 확인하는 습관을.

```python
A.shape    # (2, 3)
v.shape    # (3,)
```

### ML에서

- **이미지 한 장** = 행렬 (28×28 픽셀이면 28×28 행렬)
- **데이터 묶음 (배치)** = 행렬 (N개 샘플 × D차원 = N×D 행렬)
- **신경망의 가중치** = 행렬

신경망이라는 게 결국 **행렬의 곱과 활성화의 연속**이에요.

---

## 4.4 행렬 곱 (Matrix Multiplication)

ML의 가장 핵심 연산. **반드시** 익혀야 해요.

### 규칙: (m×n) × (n×k) = (m×k)

**왼쪽 행렬의 열 수 = 오른쪽 행렬의 행 수** 여야 곱셈이 가능해요. 결과는 (왼쪽 행 수 × 오른쪽 열 수).

```
(2×3) × (3×4) = (2×4)   ✅
(2×3) × (4×3) = ?        ❌ 안 맞음
```

### 어떻게 계산?

결과의 (i, j) 원소 = **왼쪽의 i번째 행과 오른쪽의 j번째 열의 내적.**

예시: (2×2) × (2×2)

```
    A          B            결과
[1 2]  ×  [5 6]   =     [a₁₁ a₁₂]
[3 4]     [7 8]         [a₂₁ a₂₂]

a₁₁ = (1,2) · (5,7) = 1×5 + 2×7 = 19
a₁₂ = (1,2) · (6,8) = 1×6 + 2×8 = 22
a₂₁ = (3,4) · (5,7) = 3×5 + 4×7 = 43
a₂₂ = (3,4) · (6,8) = 3×6 + 4×8 = 50

→ [19 22]
   [43 50]
```

### Python에서

```python
A = np.array([[1, 2], [3, 4]])
B = np.array([[5, 6], [7, 8]])

print(A @ B)        # 행렬 곱
# [[19 22]
#  [43 50]]

# 또는
print(np.matmul(A, B))    # 같음
print(A.dot(B))           # 같음
```

⚠️ **`A * B` 는 행렬 곱이 아니라 원소별 곱**(같은 위치끼리)이에요. 매우 자주 헷갈려요.

```python
A * B
# [[5, 12],
#  [21, 32]]    ← 그냥 같은 위치끼리 곱한 것
```

행렬 곱은 **`@` 또는 `matmul`** 만 써야 해요.

### 행렬과 벡터의 곱

특별 케이스: (m×n) 행렬 × (n×1) 벡터 = (m×1) 벡터

```
A v
(2×3)(3×1) = (2×1)

[1 2 3]   [a]   [1a + 2b + 3c]
[4 5 6] × [b] = [4a + 5b + 6c]
          [c]
```

이게 신경망 한 층의 정확한 모습이에요.

## 4.5 신경망 = 행렬 곱 + 활성화

### 한 층 (Layer)

```
입력 x = [x₁, x₂, x₃]
가중치 W = [w₁₁ w₁₂ w₁₃]   (2×3 행렬)
          [w₂₁ w₂₂ w₂₃]
편향 b = [b₁, b₂]

출력 = activation(Wx + b)

       = activation([w₁₁x₁ + w₁₂x₂ + w₁₃x₃ + b₁])
                    [w₂₁x₁ + w₂₂x₂ + w₂₃x₃ + b₂]
```

`Wx + b` 가 신경망의 핵심 연산이에요. 본문에서 무한히 보게 될 거예요.

### 깊은 신경망

여러 층을 쌓으면:

```
h₁ = activation(W₁ x + b₁)
h₂ = activation(W₂ h₁ + b₂)
h₃ = activation(W₃ h₂ + b₃)
y = W₄ h₃ + b₄
```

각 단계가 행렬 곱 + 더하기 + 활성화. 이게 **신경망의 forward pass**예요.

### PyTorch로 보면

```python
self.fc1 = nn.Linear(3, 2)
# 내부적으로 W (2×3 행렬) + b (2 벡터)

x = torch.tensor([1.0, 2.0, 3.0])    # (3,)
h = self.fc1(x)                       # = Wx + b → (2,)

y = torch.relu(h)                      # 활성화
```

`nn.Linear(3, 2)` 가 사실은 **3차원 벡터를 받아 2차원 벡터를 만드는 행렬 곱**입니다.

### 배치 처리

여러 샘플을 한 번에 처리하면, 입력이 **벡터의 배치 = 행렬**이 돼요.

```
배치 입력 X: (N, D)        N개 샘플, D차원
가중치 W: (H, D)
편향 b: (H,)

출력 = X · Wᵀ + b          ← (N, H)
```

(여기서 `Wᵀ`는 W의 **전치**(transpose), 행과 열을 바꾼 것.)

```python
X = torch.randn(64, 784)         # 배치 64, 784차원
output = self.fc(X)               # (64, 256) 같은 결과
```

이게 PyTorch가 묵묵히 하는 일이에요.

## 4.6 자주 만나는 행렬 연산들

### 전치 (Transpose) — 행과 열 바꾸기

```
A = [1 2 3]            Aᵀ = [1 4]
    [4 5 6]                  [2 5]
                             [3 6]
```

`A`가 (2×3) → `Aᵀ`는 (3×2).

```python
A = np.array([[1, 2, 3], [4, 5, 6]])
print(A.T)
# [[1, 4],
#  [2, 5],
#  [3, 6]]
```

### 단위 행렬 (Identity)

대각선만 1, 나머지는 0인 행렬.

```
I = [1 0 0]
    [0 1 0]
    [0 0 1]
```

`AI = IA = A` — 곱해도 자기 자신.

```python
np.eye(3)
```

### 영행렬 (Zero matrix)

모두 0.

```
O = [0 0 0]
    [0 0 0]
```

```python
np.zeros((2, 3))
```

### 역행렬 (Inverse)

`A · A⁻¹ = I` 가 되는 행렬.

ML에선 거의 직접 안 써요 (계산이 비싸서). 다만 선형 회귀의 닫힌 해(closed-form solution)에 등장.

## 4.7 텐서 (Tensor) — 더 일반적인 것

| 차원 | 이름 | 예시 |
|------|------|------|
| 0차원 | 스칼라 | `5` |
| 1차원 | 벡터 | `[1, 2, 3]` |
| 2차원 | 행렬 | `[[1,2],[3,4]]` |
| 3차원 | 3-tensor | 컬러 이미지 (H × W × 3) |
| 4차원 | 4-tensor | 이미지 배치 (N × H × W × C) |
| n차원 | n-tensor | ... |

**텐서**는 이 모든 걸 통칭하는 단어예요. PyTorch 객체 이름이 `Tensor` 인 이유.

```python
import torch

scalar = torch.tensor(5)              # 0차원
vector = torch.tensor([1, 2, 3])      # 1차원
matrix = torch.tensor([[1,2],[3,4]])   # 2차원
image = torch.randn(3, 224, 224)      # 3차원 (컬러 이미지)
batch = torch.randn(64, 3, 224, 224)   # 4차원 (배치)

print(scalar.shape, vector.shape, matrix.shape, image.shape, batch.shape)
```

이미지 처리 (CNN) 코드에서는 4차원 텐서가 표준이에요.

## 4.8 정리

```python
import numpy as np
import torch

# 벡터
v = np.array([1, 2, 3])

# 행렬
M = np.array([[1, 2], [3, 4]])

# 내적
np.dot(v1, v2)
v1 @ v2

# 행렬 곱
A @ B
np.matmul(A, B)

# 원소별 곱 (행렬 곱 아님!)
A * B

# 전치
A.T

# 모양 확인
A.shape

# 신경망 한 층
y = activation(W @ x + b)
```

### ML 식 다시 읽기

```
y = σ(Wx + b)
```

읽으면: **"출력 = 활성화함수(가중치 행렬 × 입력 벡터 + 편향 벡터)."**

```
y = softmax(W₂ × ReLU(W₁ × x + b₁) + b₂)
```

읽으면: **"출력 = softmax( W₂ × ReLU( W₁ × x + b₁ ) + b₂ )."**
"입력 x를 W₁로 변환하고, b₁ 더하고, ReLU 통과시키고, 다시 W₂로 변환하고, b₂ 더하고, softmax로 확률 만들기."

신경망 한 글자 한 글자가 다 의미가 있죠.
