## 2a.1 극한 (Limit)

### 직관

`x`가 어떤 값 `a`에 **한없이 가까워질 때**, `f(x)`가 어떤 값에 가까워지는가?

```
lim_{x→a} f(x) = L
```

읽으면: **"x가 a로 갈 때 f(x)의 극한은 L."**

### 단순한 예

```
f(x) = x + 1

lim_{x→2} (x + 1) = 3
                    ↑
        그냥 x = 2 대입한 값
```

이런 경우는 그냥 대입하면 돼요. 흥미로운 건 다음.

### 0/0 형태 — 진짜 극한이 필요한 경우

```
f(x) = (x² - 1) / (x - 1)
```

`x = 1` 을 대입하면 `0/0` — 정의 안 됨!

근데 약분하면:

```
(x² - 1) / (x - 1) = (x+1)(x-1) / (x-1) = x + 1   (x ≠ 1)
```

그래서:

```
lim_{x→1} (x² - 1)/(x - 1) = lim_{x→1} (x + 1) = 2
```

`x = 1`에서는 정의되지 않지만, 극한은 2.

### 무한대 극한

`x → ∞` 으로 보내기.

```
lim_{x→∞} (1/x) = 0
lim_{x→∞} (1 + 1/x)^x = e   ← 자연상수 e의 정의!
```

두 번째가 신기하죠. `e ≈ 2.718...` 의 진짜 정의가 이 극한이에요.

### 극한이 ML에 왜 중요?

#### 미분의 정의가 극한이에요

```
f'(x) = lim_{h→0} [f(x+h) - f(x)] / h
```

읽으면: **"f의 미분 = h가 0으로 갈 때 (변화량/h) 의 극한."**

기울기를 한 점에서 정확히 보려면 — `h` 를 무한히 작게 — 그게 극한.

[3장](03-미분-입문.md)의 모든 미분 공식이 이 극한에서 도출돼요.

#### 컴퓨터에서?

물론 컴퓨터는 진짜 극한을 못 해요. 대신 매우 작은 h(예: 1e-7)로 근사. 이게 **수치 미분(numerical differentiation)**.

```python
def numerical_gradient(f, x, h=1e-7):
    return (f(x + h) - f(x - h)) / (2 * h)
```

PyTorch는 수치 미분 안 쓰고 **자동 미분 (autograd)**으로 정확히 계산해요.

## 2a.2 삼각함수 (Trigonometric Functions)

### 단위원 정의

각도 `θ`에 대해 단위원 위의 점 `(cos θ, sin θ)`.

```
       y
       ↑
      1 ┤        ●
        │      / │
        │  1 /   │ sin θ
        │   /    │
        │  / θ   │
        ┼─────────────→ x
        │ cos θ
       -1
```

각도가 변하면 점이 원 위를 돌아요.

### 라디안 (Radian)

각도의 단위를 도(°) 대신 **라디안**으로 표현. 더 수학적.

```
360° = 2π 라디안 ≈ 6.28
180° = π 라디안 ≈ 3.14
90°  = π/2 라디안 ≈ 1.57
60°  = π/3 라디안
45°  = π/4 라디안
30°  = π/6 라디안
```

ML 코드에서는 보통 라디안. `math.sin(math.pi/2)` = 1.

### 그래프

```
sin x — 파동 모양

       y
     1 ┤   ╭─╮            ╭─╮
       │  ╱   ╲          ╱   ╲
     0 ┼╱─────╲────────╱─────╲────→ x
       │       ╲      ╱       ╲
    -1 ┤        ╲────╯
       0   π/2   π   3π/2   2π

cos x — 같은 모양, π/2 만큼 왼쪽으로 이동
```

### 미분

신경망에서도 가끔 등장.

```
(sin x)' = cos x
(cos x)' = -sin x
(tan x)' = 1/cos²x = sec²x
```

### 항등식

```
sin² x + cos² x = 1            ← 가장 유명
sin(a + b) = sin a cos b + cos a sin b
cos(a + b) = cos a cos b - sin a sin b
```

### ML에서?

#### 1. Positional Encoding (Transformer)

Transformer 모델의 위치 정보 표현.

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))
```

위치마다 다른 sin/cos 값으로 위치 정보 인코딩.

#### 2. 회전 행렬

```
회전 행렬 R(θ) = [cos θ  -sin θ]
                [sin θ   cos θ]
```

이미지 데이터 증강 (회전), 컴퓨터 비전 등.

#### 3. 푸리에 변환

신호 처리, 음성 인식의 기초. 신호를 sin/cos의 합으로 분해.

## 2a.3 순열 (Permutation) — 순서가 중요

서로 다른 `n`개에서 `r`개를 **순서대로** 뽑는 경우의 수.

```
nPr = n! / (n-r)!
    = n × (n-1) × (n-2) × ... × (n-r+1)
```

### 예시

5명 중 3명을 1등, 2등, 3등으로 줄세우기:

```
5P3 = 5! / 2! = 5 × 4 × 3 = 60
```

### 팩토리얼 (!)

```
n! = n × (n-1) × (n-2) × ... × 2 × 1

5! = 120
0! = 1   (약속)
```

### Python

```python
import math
math.factorial(5)        # 120
math.perm(5, 3)          # 60   (Python 3.8+)
```

## 2a.4 조합 (Combination) — 순서 없음

`n`개에서 `r`개를 **순서 상관없이** 뽑는 경우의 수.

```
nCr = n! / (r! × (n-r)!)
    = nPr / r!
```

### 예시

5명 중 3명을 뽑기 (순서 X):

```
5C3 = 5! / (3! × 2!) = 120 / 12 = 10
```

순열과 다른 점: 1등, 2등, 3등 구분 없이 단지 3명을 고르기만.

### Python

```python
math.comb(5, 3)          # 10   (Python 3.8+)
```

### ML에서?

#### 1. 데이터 분할의 가능 경우 수

100개 중 20개를 test set으로 뽑는 경우 = `100C20`. 어마어마하게 큰 수.

#### 2. 베이즈 모델, 이항 분포

```
P(X = k) = nCk × p^k × (1-p)^(n-k)   ← 이항 분포
```

#### 3. 파라미터 공간 탐색

`d` 차원 데이터에서 `k`개 특성 부분집합 = `dCk`.

## 2a.5 이항 분포 (Binomial Distribution)

베르누이 시행을 `n`번 반복할 때, 성공한 횟수의 분포.

```
P(X = k) = nCk × p^k × (1-p)^(n-k)
```

여기서:
- `n`: 시행 횟수
- `p`: 한 번 성공 확률
- `k`: 성공한 횟수

### 예시

동전을 10번 던질 때 앞면이 정확히 5번 나올 확률 (p=0.5):

```
P(X=5) = 10C5 × 0.5^5 × 0.5^5
       = 252 × (1/32) × (1/32)
       = 252 / 1024
       ≈ 0.246
```

### 평균과 분산

```
평균: E[X] = np
분산: Var(X) = np(1-p)
```

위 예: `E = 10×0.5 = 5`, `Var = 10×0.5×0.5 = 2.5`.

### ML에서?

이진 분류의 라벨이 베르누이, 다중 시행의 합이 이항. 그리고:

```
n이 커지면 이항 분포 → 정규 분포에 수렴
```

이게 **중심극한정리(CLT)** 의 한 예. 다음 단원에서 자세히.

## 2a.6 확률 변수와 기댓값

### 확률 변수 (Random Variable)

**값이 확률적으로 결정되는 변수.** 보통 대문자 `X`, `Y`로 표기.

```
주사위를 던질 때 나오는 눈 = X
X의 가능한 값: {1, 2, 3, 4, 5, 6}
각 확률: P(X=k) = 1/6 (k=1~6)
```

### 기댓값 (Expected Value)

확률 변수가 **평균적으로 가지는 값**. `E[X]` 또는 `μ`.

```
E[X] = Σ x × P(X = x)        ← 이산형
     = ∫ x × p(x) dx           ← 연속형 (적분, 다음 장)
```

### 예시: 주사위

```
E[X] = 1×(1/6) + 2×(1/6) + ... + 6×(1/6)
     = (1+2+3+4+5+6)/6
     = 3.5
```

평균값 3.5. 한 번에는 안 나오지만, 많이 던지면 평균이 3.5에 가까워짐.

### 분산 (Variance) — 다시

확률 변수 버전:

```
Var(X) = E[(X - μ)²]
       = E[X²] - μ²
```

### 기댓값의 성질

```
E[aX + b] = aE[X] + b           ← 선형성
E[X + Y] = E[X] + E[Y]          ← X, Y 독립 아니어도 성립
E[XY] = E[X]E[Y]                 ← X, Y 독립일 때만
```

선형성이 ML에서 매우 유용. 손실 함수 분석할 때 자주 등장.

## 2a.7 정규분포 — 다시 한 번 깊게

[5장](05-확률통계-입문.md)에서 봤지만, 식을 풀어 보면:

```
p(x) = (1 / √(2πσ²)) × exp(-(x-μ)² / (2σ²))
       └────────────┘    └────────────────┘
       정규화 상수         지수 부분 (종 모양)
```

각 부분 의미:

- `(x - μ)²`: 평균에서 떨어진 거리의 제곱
- `/ (2σ²)`: 표준편차로 나눔 (스케일)
- `exp(-...)`: 지수 함수, 멀어질수록 빠르게 작아짐
- `1 / √(2πσ²)`: 전체 면적이 1이 되도록 맞추는 상수

### 표준화

`X ~ N(μ, σ²)` 인 데이터에 대해:

```
Z = (X - μ) / σ      ~ N(0, 1)
```

이게 **StandardScaler**가 하는 일이에요. 어떤 정규분포든 표준정규분포로 변환.

### ML에서

#### 가우시안 함수

방사기저함수(RBF) 커널, 가우시안 노이즈, VAE의 잠재 분포 등 어디든 등장.

#### 가우시안 분포 가정

선형 회귀의 MSE 손실은 **잔차가 가우시안 분포** 라고 가정한 것.

```
MSE 최소화 ⇔ 잔차가 N(0, σ²) 인 가우시안 가정의 MLE
```

(증명은 깊이 가지만, 사실만 알아두세요.)

## 2a.8 로피탈 정리 (참고)

`0/0` 또는 `∞/∞` 형태의 극한을 풀 때 유용.

```
lim f(x)/g(x) = lim f'(x)/g'(x)   (조건 만족 시)
```

예시:

```
lim_{x→0} sin(x)/x = ?
   대입하면 0/0
   
   미분: cos(x)/1 = cos(0)/1 = 1
   
   ∴ lim_{x→0} sin(x)/x = 1
```

ML에서는 직접 쓸 일 거의 없지만, 책에서 보면 알아보면 됩니다.

## 2a.9 정리

```python
import math
import numpy as np

# 극한
# lim_{x→0} (1+1/x)^x = e

# 삼각함수
math.sin(math.pi/2)    # 1
math.cos(math.pi/2)    # 0  (사실 거의 0)

# 순열, 조합
math.factorial(5)      # 120
math.perm(5, 3)        # 60
math.comb(5, 3)        # 10

# 이항 분포 샘플링
np.random.binomial(n=10, p=0.5, size=1000)

# 표준화
data_std = (data - np.mean(data)) / np.std(data)
```

### ML 식 다시 읽기

```
f'(x) = lim_{h→0} [f(x+h) - f(x)] / h
```
"f의 미분 = h가 0으로 갈 때 (변화율)의 극한."

```
PE(pos, 2i) = sin(pos / 10000^(2i/d))
```
"위치 pos의 2i번째 차원의 인코딩 = sin(pos를 10000^(2i/d)로 나눈 값)."

```
P(X = k) = nCk × p^k × (1-p)^(n-k)
```
"X가 k일 확률 = (n에서 k 뽑는 경우의 수) × p의 k제곱 × (1-p)의 (n-k)제곱."
